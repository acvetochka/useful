# Використання для автоматизації
Скрипти можна використовувати для автоматизації певних завдань. Автоматизація використовується для заміни повторюваних ручних дій на ті, що відбуваються автоматично. Людям властиво помилятися. Вони можуть втомлюватися, помилятися, не виконувати інструкції, бути непослідовними у виконанні роботи тощо. На противагу цьому, автоматизовані процеси виконують інструкції точно так, як вони закодовані, у послідовний спосіб. Вони можуть працювати 24 години на добу, щодня, не втомлюючись. Для багатьох завдань, які підходять для автоматизації, використання автоматизації може бути більш економічно ефективним, ніж людська праця.


## Автоматизація може бути використана, зокрема, в наступних сферах
- Автоматичний таймінг і регулювання світлофорів

- Повторювані завдання з високим ризиком людської помилки

- Надсилання команд на комп'ютер

- Виявлення та видалення дублікатів даних

- Надсилання автоматизованих електронних листів, які персоналізуються шляхом витягування окремих імен з бази даних і вставляння їх в електронний лист

- Оновлення великої кількості дозволів на доступ до файлів

- Звітування про системні дані, такі як використання диска або пам'яті

- Встановлення програмного забезпечення

- Створення звітів

- Розгортання файлу або комп'ютерної програми на всіх комп'ютерах у мережі компанії

- Використання системи управління конфігурацією для розгортання виправлень програмного забезпечення після того, як людина розробила систему

- Наповнення сайту електронної комерції товарами

- Налаштування домашнього каталогу та прав доступу для користувачів


## Автоматизація не завжди є доречним або повним рішенням
Автоматизація не може виконувати всю людську роботу. Завдання, які вимагають людської творчості, соціальних зв'язків, психології, гнучкості, винахідливості, оцінки та/або складної аналітичної роботи, не є хорошими кандидатами для повної автоматизації. Іноді автоматизація може бути використана для виконання однієї або декількох підзадач великого набору завдань, але для їх виконання необхідне втручання людини. Нижче наведено кілька прикладів завдань, які не можуть або не повинні бути повністю автоматизовані:

- Завдання, які потребують людської оцінки та аналітичних навичок:

  - Проектування системи управління конфігурацією

  - Дослідження та усунення всіх проблем кінцевого користувача

  - Написання комп'ютерної програми

  - Створення нового стартап-бізнесу

- Завдання, що вимагають людської креативності та/або естетичних якостей:

  - Створення привабливої веб-сторінки (ШІ може це зробити, але проста автоматизація - ні)

  - Весільна фотографія

  - Стрижки та укладки

- Елементи, які не можуть бути автоматизовані через базову фізику:

  - Усунення несправностей або ремонт машин, які не можуть увімкнутися або завантажитися

- Завдання, які потребують людської взаємодії, психології та/або навичок оцінювання:

  - Співбесіда та найм нових працівників

  - Обслуговування клієнтів (чат-боти не можуть задовольнити всі потреби клієнтів)

- Функції, які не можна повністю автоматизувати з міркувань вартості та безпеки:

  - Процес каси в продуктових магазинах, включаючи пакування продуктів у пакети

  - Завдання, які дешевше виконувати вручну


## Штучний інтелект
Важливо розуміти, що базова автоматизація - це не те саме, що штучний інтелект. Автоматизація використовується для того, щоб чітко проінструктувати машину про те, як виконати завдання. Штучний інтелект (ШІ) передбачає навчання обчислювальної машини виконувати більш складні завдання за допомогою процесу, який називається машинним навчанням. Цей процес готує програмне забезпечення ШІ до виконання нових завдань без необхідності для людини програмувати чіткі інструкції для кожного завдання. Хоча ШІ часто використовується для автоматизації людських завдань, автоматизація ШІ набагато складніша, ніж базова автоматизація.

# Зауваження щодо синтаксису та кодових блоків
Під час написання коду дуже важливо використовувати правильний синтаксис. Навіть невелика помилка, наприклад, пропущена дужка або зайва кома, може призвести до синтаксичної помилки, і код взагалі не виконається. Якщо ваш код призводить до помилки або виключення, зверніть особливу увагу на синтаксис і стежте за незначними помилками. У довгому коді на виявлення одного неправильного символу можуть піти години, тому важливо пам'ятати про синтаксис під час написання коду.

## Поширені синтаксичні помилки:
- Орфографічні помилки

- Неправильні відступи

- Відсутні або неправильні ключові символи:

  - Типи дужок - ( криві ), [ квадратні ], { фігурні }

  - Типи лапок - "прямі подвійні" або "прямі одинарні", "фігурні подвійні" або "фігурні одинарні

  - Символи введення блоку, такі як двокрапка - :

- Невідповідність типів даних

- Відсутні, неправильно використані або неправильно розміщені зарезервовані слова Python

- Використання неправильного регістру (верхній/нижній регістр) - Python є чутливою до регістру мовою

Якщо ваш синтаксис правильний, але скрипт має неочікувану поведінку або вивід, це може бути пов'язано з семантичною проблемою. Синтаксис - це словниковий запас, граматика, орфографія і пунктуація коду. Семантика - це значення і логіка закодованих тверджень. Можна мати синтаксично правильний код, який успішно виконується, але не робить того, що ми хочемо.

## Поширені семантичні помилки:
- Створення функціонального коду, але отримання ненавмисного результату

- Погані логічні структури при проектуванні коду

Працюючи з блоками коду у вправах цього курсу, пам'ятайте про синтаксичні та семантичні (логічні) помилки, а також про загальний результат вашого коду. Те, що ви виправили помилку, ще не означає, що код буде мати бажаний ефект при виконанні! Після того, як ви виправили помилку в коді, не забудьте натиснути кнопку "Виконати", щоб перевірити свою роботу.

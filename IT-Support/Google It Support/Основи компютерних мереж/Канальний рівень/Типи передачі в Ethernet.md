# Типи передачі даних в Ethernet
## Вступ

Раніше ми говорили про передачу даних між пристроями.  
Тепер важливо зрозуміти:
> 📦 як саме дані доставляються в мережі

У Ethernet існує 3 типи передачі:
- Unicast
- Multicast
- Broadcast

## 📍 1. Unicast (одноадресна передача)
**📌 Визначення:**

`Unicast` — це:
> передача даних від одного пристрою до одного конкретного

**⚙️ Як визначається:**
- використовується MAC-адреса отримувача
- якщо:
  - найменш значущий біт першого октету = 0  
    👉 це unicast

**📡 Як працює:**
- кадр надсилається в мережу
- всі пристрої його “бачать”
- але:
  - тільки один (з потрібною MAC) обробляє

**🧠 Простими словами:**
> як лист конкретній людині

## 👥 2. Multicast (багатоадресна передача)
**📌 Визначення:**

`Multicast` — це:
> передача даних групі пристроїв

**⚙️ Як визначається:**
- найменш значущий біт = 1

**📡 Як працює:**
- кадр надсилається в мережу
- кожен пристрій вирішує:
  - приймати чи ні

**💡 Особливість:**
- пристрої можуть бути налаштовані:
  - слухати певні multicast-адреси

**🧠 Простими словами:**
> як повідомлення групі (наприклад, чат)

## 📢 3. Broadcast (широкомовна передача)
**📌 Визначення:**

`Broadcast` — це:
> передача даних всім пристроям у мережі

**⚙️ Як визначається:**
- спеціальна MAC-адреса:
```
FF:FF:FF:FF:FF:FF
```

**📡 Як працює:**
- кожен пристрій:
  - отримує
  - обробляє

**💡 Для чого використовується:**
- пошук інших пристроїв
- службові запити (наприклад ARP)

**🧠 Простими словами:**
> як оголошення для всіх

## 📊 Порівняння
| Тип       | Кому йде | Хто обробляє      |
| --------- | -------- |
| Unicast   | одному   | тільки він        |
| Multicast | групі    | ті, хто підписані |
| Broadcast | всім     | всі               |

## ⚠️ Важливий нюанс

У старих мережах (з хабами):
- всі отримують всі кадри

У сучасних (з комутаторами):
- трафік більш ізольований
- але broadcast все одно доходить до всіх у сегменті

## 🧾 Висновок
- Ethernet підтримує 3 типи передачі
- MAC-адреса визначає тип
- різниця — в тому, хто обробляє дані

## 📌 Головна ідея

> Не всі дані в мережі призначені всім —  
важливо розуміти, кому саме вони адресовані

# Клієнти та сервери в мережі
## Вступ

Усі мережеві пристрої (хаби, комутатори, маршрутизатори) потрібні для того, щоб:

> 📡 комп’ютери могли обмінюватися даними — незалежно від відстані

У мережі такі пристрої називаються:
- вузли (nodes)

Але щоб зрозуміти, як саме відбувається обмін, потрібно знати дві ключові ролі:
- сервер
- клієнт
## 📡 Що таке сервер і клієнт

**📌 Сервер (Server)**

Сервер — це:

> щось, що надає дані іншим

**📌 Клієнт (Client)**

Клієнт — це:
> щось, що запитує і отримує дані

**🧠 Простими словами:**

- сервер = “дає”
- клієнт = “просить”

## 🔄 Важливий нюанс: це не тільки комп’ютери
❗ Не лише пристрої

Сервером або клієнтом може бути не тільки комп’ютер, а й:

- програма
- сервіс
- процес

👉 навіть на одному комп’ютері:

- браузер → клієнт
- веб-сервер → сервер

**💡 Приклад:**
- ти відкриваєш сайт  
  → браузер = клієнт  
  → веб-сервер = сервер

## ⚖️ Один вузол може бути і тим, і тим
**📌 Важливо:**

Більшість пристроїв:
- не є “чисто” сервером або клієнтом
- можуть виконувати обидві ролі

**🔁 Приклад:**

📧 Поштовий сервер:
- сервер → для користувачів (віддає пошту)
- клієнт → для DNS (запитує IP-адреси)

**🧠 Висновок:**

> Роль залежить від того, що саме зараз робить система

## 🧭 Основна роль у мережі

Попри те, що вузол може бути і тим, і тим:
- зазвичай він має основне призначення

**📊 Приклади:**
| Пристрій        | Основна роль |
| --------------- | ------------ |
| Веб-сервер      | сервер       |
| Поштовий сервер | сервер       |
| ПК користувача  | клієнт       |

**💡 Пояснення:**
- навіть якщо ПК іноді “віддає” дані  
  → його головна роль — отримувати
- навіть якщо сервер щось запитує  
  → його головна роль — надавати

**🧠 Простими словами**

> Назва “сервер” або “клієнт” — це не строгий статус,
а основна функція в мережі

## 🔗 Як це працює разом
📦 Типовий сценарій:
1. клієнт (браузер) → надсилає запит
1. сервер → обробляє
1. сервер → надсилає відповідь
1. клієнт → отримує дані

## 🧾 Висновок
- сервер → надає дані
- клієнт → отримує дані
- роль може змінюватися
- але зазвичай є основна функція

## 📌 Головна ідея

> У мережі важливо не “хто ти”, а “що ти робиш зараз”

# П’ятирівнева мережева модель (Network Model)

## Вступ

Щоб зрозуміти, як працюють комп’ютерні мережі, потрібно розглядати всі компоненти разом:

- фізичні з’єднання (кабелі, роз’єми)
- пристрої (комп’ютери, маршрутизатори)
- протоколи (правила обміну даними)

Для пояснення взаємодії використовується мережева модель із 5 рівнів, де кожен рівень виконує свою функцію.

## 🧱 1. Фізичний рівень (Physical Layer)
**📌 Що це:**

Найнижчий рівень, який описує фізичну передачу даних

**🔧 Що входить:**
- мережеві кабелі (Ethernet, оптика)
- роз’єми (RJ-45 тощо)
- електричні / світлові сигнали

**📡 Основна функція:**

Передача бітів (0 і 1) у вигляді сигналів

**🧠 Простими словами:**

> Це “дроти і сигнал” — як саме дані фізично передаються

## 🔗 2. Канальний рівень (Data Link Layer)

(також: Network Interface / Network Access Layer)

**📌 Що це:**

Рівень, який визначає, як пристрої спілкуються в межах однієї мережі

**🔧 Основні протоколи:**
- Ethernet (найпоширеніший)
- Wi-Fi (бездротові технології)

**📡 Основна функція:**
- інтерпретація сигналів з фізичного рівня
- передача даних між пристроями в одній мережі
  
**🧠 Простими словами:**

> Визначає “правила спілкування” між сусідніми пристроями

## 🌐 3. Мережевий рівень (Network Layer)

(також: Internet Layer)

**📌 Що це:**

Рівень, який дозволяє різним мережам взаємодіяти між собою

**🔧 Основний протокол:**
- IP

**📡 Основна функція:**
- маршрутизація (визначення шляху)
- доставка даних між різними мережами

**📦 Ключове поняття:**
Інтернет = мережа мереж

**🧠 Простими словами:**

> Відповідає за шлях: як дані доходять з точки A до точки B через багато мереж

## 🚚 4. Транспортний рівень (Transport Layer)
**📌 Що це:**

Рівень, який визначає, яка саме програма отримає дані

**🔧 Основні протоколи:**
- TCP
- UDP

**⚖️ TCP vs UDP:**
| Протокол | Особливість                                |
| -------- | ------------------------------------------ |
| TCP      | Надійна передача (контроль, підтвердження) |
| UDP      | Швидка, але без гарантії доставки          |

**📡 Основна функція:**  
- доставка даних до конкретного додатку (браузер, пошта)

**🧠 Простими словами:**

> Визначає “кому саме віддати дані” всередині комп’ютера

## 🖥️ 5. Прикладний рівень (Application Layer)
**📌 Що це:**

Найближчий до користувача рівень

**🔧 Приклади протоколів:**
- HTTP (веб)
- SMTP (email)
- FTP (файли)

**📡 Основна функція:**
- взаємодія між програмами користувача та мережею

**🧠 Простими словами:**

> Це те, що ти реально використовуєш (браузер, пошта, додатки)

## 🧠 Додатково: клієнт і сервер
**📌 Модель взаємодії:**
- Клієнт → робить запит
- Сервер → відповідає

**💡 Важливо:**
- один пристрій може бути і клієнтом, і сервером
- одночасно можуть працювати кілька програм (браузер + пошта)

## 📦 Аналогія: доставка посилки
| Рівень       | Аналогія                        |
| ------------ | ------------------------------- |
| Фізичний     | дороги і вантажівки             |
| Канальний    | рух між перехрестями            |
| Мережевий    | вибір маршруту                  |
| Транспортний | доставка конкретному отримувачу |
| Прикладний   | вміст пакета                    |

## 🧾 Висновок
- Кожен рівень виконує свою окрему роль
- Разом вони забезпечують повну передачу даних
- Важливо розуміти різницю між:
  - IP → доставка між пристроями
  - TCP/UDP → доставка до програм

# Таблиця маршрутизації (Routing Table)
## Вступ

Маршрутизація працює завдяки:
> 📌 таблиці маршрутизації

## 📌 Визначення

`Таблиця маршрутизації` — це:
> список правил, який каже маршрутизатору  
> куди відправити пакет

**🧠 Простими словами:**
> це “навігатор” для роутера

## 🖥️ Історичний факт
- перші маршрутизатори = звичайні комп’ютери
- таблиці:
  - заповнювались вручну 😅

**📌 Сьогодні:**
- навіть твій комп’ютер має:
  - свою routing table

## 🧱 Основні поля таблиці

Типова таблиця має 4 головні стовпці 👇

### 1️⃣ Destination Network (мережа призначення)
**📌 Що це:**
- мережа, куди потрібно доставити пакет

**📡 Формат:**
```
192.168.1.0/24
```
**🧠 Що означає:**
- всі IP цієї мережі

### 2️⃣ Next Hop (наступний вузол)
**📌 Що це:**
> куди передати пакет далі

**📡 Варіанти:**
- IP іншого маршрутизатора
- або:
  ```
  directly connected
  ```

**🧠 Простими словами:**
> “кому передати далі”

### 3️⃣ Metric / Hop Count (відстань)
**📌 Що це:**
- “наскільки далеко” мережа

**📡 Важливо:**
- менше = краще
- використовується для вибору шляху

**🧠 Простими словами:**
> “який маршрут коротший”

### 4️⃣ Interface (інтерфейс)
**📌 Що це:**
- через який порт відправити

**🧠 Простими словами:**
> “через який вихід вийти”

## 🔄 Як router використовує таблицю
📌 Алгоритм:
1. отримує пакет
2. дивиться IP призначення
3. шукає мережу в таблиці
4. вибирає маршрут
5. відправляє

## 🌐 Default Route (дуже важливо)
**📌 Що це:**
> маршрут “на всі випадки”

**📡 Приклад:**
```
0.0.0.0/0
```

**🧠 Простими словами:**
> “якщо не знаю — відправ туди”

## 🔥 Чому це важливо
📌 У мережі:
- існує багато шляхів
- маршрут може змінюватися

⚠️ Причини:
- поломка роутера
- перевантаження
- нові маршрути

## 🔁 Динаміка

Маршрутизатор:
- отримує оновлення
- змінює маршрути
- обирає найкращий шлях

## 🌍 Масштаб Інтернету
📌 Факт:
- великі маршрутизатори:
  - мають мільйони записів 😱

🧠 І при цьому:
- перевіряють таблицю для кожного пакета

## 🧾 Висновок
- routing table:
  - основа маршрутизації
- містить:
  - куди
  - через кого
  - як далеко
  - через який інтерфейс

## 📌 Головна ідея
> Маршрутизатор не “думає” —  
він просто дивиться в таблицю і виконує правило

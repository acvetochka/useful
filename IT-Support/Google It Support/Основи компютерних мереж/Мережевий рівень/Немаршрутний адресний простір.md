# Немаршрутний адресний простір

## 📜 Історія проблеми IPv4
📌 Факт
- IPv4 = 32 біти
- кількість адрес:
  ```
  2³² = 4 294 967 296
  ```

👉 звучить багато… але:
- населення Землі > 7 млрд
- сервери, телефони, IoT 😅

**⚠️ Проблема**
> IP-адрес не вистачає

## 📄 Що таке RFC
**📌 Визначення**

`RFC (Request for Comments)` — це:
> документи, які описують стандарти Інтернету

**📘 Важливий документ**

👉 RFC 1918 (1996 рік)

## 🧩 Рішення: приватні IP-адреси
**📌 Ідея**
> не всі пристрої повинні бути доступні з Інтернету

**📌 Визначення**

`Приватні IP (private / non-routable)` — це:
> адреси, які НЕ маршрутизуються в Інтернеті

**🧠 Простими словами**
- працюють всередині мережі
- “невидимі” для Інтернету

## 📦 Діапазони приватних IP
🔹 1. Велика мережа
```
10.0.0.0/8
```
👉 дуже багато адрес

🔹 2. Середня
```
172.16.0.0/12
```
🔹 3. Маленька (найпопулярніша)
```
192.168.0.0/16
```
👉 домашні мережі (Wi-Fi)

## 🧠 Важлива властивість

👉 ці адреси:
- ❌ не маршрутизуються в Інтернеті
- ✅ працюють всередині мережі

## 🌐 Чому це геніально

👉 ті самі IP можуть використовувати всі:
- у тебе вдома: 192.168.1.1
- у сусіда: теж 192.168.1.1

👉 і це нормально 😄

## 🔄 Як тоді працює Інтернет?

Тут з’являється:

### 🔧 NAT (Network Address Translation)
**📌 Ідея NAT**
> один публічний IP  
→ для всієї мережі

**🧠 Приклад**
- твій ПК: 192.168.1.100
- роутер: 84.123.45.67 (публічний IP)

👉 для Інтернету:
> всі запити виглядають як від 84.123.45.67

## 🔗 Як це пов’язано з маршрутизацією
- IGP → працює з приватними IP
- EGP → працює тільки з публічними IP

## ⚠️ Важливо

👉 маршрутизатори в Інтернеті:
- ігнорують приватні IP
- НЕ передають їх

## 🧠 Головна ідея
> приватні IP = внутрішнє життя мережі  
> публічні IP = життя в Інтернеті

## 📊 Чому це вирішило проблему

Без приватних IP:
- кожному пристрою потрібен унікальний IP

З приватними IP:
- тисячі пристроїв → 1 публічний IP

## 🧾 Висновок
- IPv4 не вистачає
- RFC 1918 ввів приватні IP
- NAT дозволяє їм працювати з Інтернетом

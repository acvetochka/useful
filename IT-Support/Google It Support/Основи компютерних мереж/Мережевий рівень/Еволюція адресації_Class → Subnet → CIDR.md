# Еволюція адресації: Class → Subnet → CIDR

## 🔴 1. Проблема класів (A / B / C)
**📌 Як було раніше:**
- Class A → дуже велика мережа
- Class B → середня
- Class C → маленька

**❗ Проблеми:**
1. Негнучкі розміри
   
    | Клас | Хостів    |
    | ---- | --------- |
    | A    | ~16 млн 😱 |
    | B    | ~65 тис   |
    | C    | 254       |

    👉 Реальність:
    - 254 → часто мало
    - 65 000 → занадто багато

    💥 Наслідок:

    компанії брали:
    - багато Class C мереж

2. таблиці маршрутизації ставали величезними

## 🧠 Приклад проблеми

Компанія має 2 мережі:
  ```
  192.168.1.0/24
  192.168.2.0/24
  ```
👉 маршрутизатору треба:
- 2 записи

## 🚀 2. Рішення — CIDR
**📌 Що таке:**

→ CIDR

**🔑 Головна ідея:**
> ❌ більше НЕ використовуємо класи  
> ✅ використовуємо тільки маску (/xx)

**🔧 Що змінюється**

**❌ Було:**
- Network ID
- Subnet ID
- Host ID

**✅ Стало:**
- Network (включає subnet)
- Host

**🧠 Простими словами:**
> CIDR “об’єднує” мережу і підмережу в одне

## 🔢 Приклад
Було:
```
9.100.100.100
маска: 255.255.255.0
```
CIDR:
```
9.100.100.100/24
```

**📌 Що це означає:**
- 24 біти → мережа
- 8 біт → хости

## 🔥 Головна перевага CIDR
**📌 Гнучкість**

тепер можна робити:
- /23
- /25
- /27
- будь-що

## 💡 Найважливий приклад 
**❌ Без CIDR:**

- 2 мережі:
  ```
  /24 + /24
  ```
- 👉 хости:
  ```
  254 + 254 = 508
  ```
- 👉 маршрути:
  - 2 записи ❌

**✅ З CIDR:**

об’єднуємо:
```
/23
```

**📊 Рахуємо:**
```
32 - 23 = 9 біт хоста
```
```
2⁹ = 512 адрес
```
```
512 - 2 = 510 хостів
```

**🎯 Результат:**
|          | Було | Стало |
| -------- | ---- | ----- |
| хости    | 508  | 510   |
| маршрути | 2    | 1     |

**🧠 Висновок:**
> CIDR = менше записів + більше ефективності

## 📡 Що таке “точка розмежування”
📌 Термін:
> demarcation point

🧠 Простими словами:
> межа між мережами

## 🧾 Висновок
- класи:
  - були першою спробою
  - але неефективні
- subnetting:
  - покращило ситуацію
- CIDR:
  - зробив систему гнучкою

## 📌 Головна ідея
> CIDR дозволяє створювати мережі будь-якого розміру  
і значно спрощує маршрутизацію

## 🔥 Дуже важливо запам’ятати

👉 CIDR = маска визначає все

👉 не існує більше:

- “це клас A/B/C”

👉 є тільки:

- /24, /27, /23 і т.д.

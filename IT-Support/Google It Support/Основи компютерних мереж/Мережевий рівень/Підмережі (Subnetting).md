# Підмережі (Subnetting)
## Вступ

У реальних мережах одна велика мережа:

- ❌ незручна в управлінні
- ❌ погано масштабується

👉 тому її ділять на менші частини

**📌 Визначення:**

Підмережа (subnet) — це:
> частина великої мережі, виділена логічно

**🧠 Простими словами:**
> subnetting = “розділити одну велику мережу на менші”

**❗ Чому це потрібно**  
**📌 Проблема без підмереж:**
- занадто багато пристроїв
- складна маршрутизація
- перевантаження мережі

**💡 Приклад:**

мережа класу A:
- ~16 мільйонів адрес 😱

👉 неможливо ефективно керувати

## 🔗 Як працює маршрутизація
**📦 Приклад:**

IP:
```
9.100.100.100
```

**📡 Що відбувається:**
1. Інтернет-маршрутизатори:
    - бачать мережу → 9.0.0.0
    - направляють трафік туди
2. Далі:
    - працює маршрутизатор шлюзу (gateway)

**📌 Gateway**

Маршрутизатор шлюзу — це:
> точка входу/виходу для мережі

**🧠 Простими словами:**
> gateway = “вхідні двері в мережу”

## ⚠️ Проблема великої мережі

Навіть після цього:
- одна мережа = занадто велика
- один маршрутизатор:
  - не впорається

## 🔀 Рішення: підмережі
**📌 Що відбувається:**

велика мережа:
```
9.0.0.0
```
👉 ділиться на:
- 9.1.0.0
- 9.2.0.0
- 9.3.0.0
- і т.д.

**📡 Кожна підмережа:**  
має:
- свій діапазон IP
- свій gateway

**🧠 Простими словами:**
> замість одного великого міста → багато районів

## 🧱 Роль підмереж
**📌 Підмережі дозволяють:**
- зменшити навантаження
- краще організувати мережу
- ізолювати трафік
- спростити адміністрування

## 🔢 Маска підмережі (Subnet Mask)
**📌 Що це:**

визначає:
> яка частина IP — мережа  
яка — хост

**💡 (поки базово)**

наприклад:
```
255.255.255.0
```
👉 означає:
- перші 3 октети → мережа
- останній → хост

## 🚀 CIDR (сучасний підхід)
**📡 Повна назва:**

→ CIDR

**📌 Що дає:**
- більше гнучкості
- не прив’язаний до класів A/B/C

**🧠 Простими словами:**
> CIDR = “розбиваємо мережу як хочемо”

## ⚠️ Практичне значення
**📌 Важливо для IT:**
- неправильна підмережа =
  - ❌ немає зв’язку
  - ❌ проблеми з доступом

**💡 Часта проблема:**
- пристрої в різних підмережах
- але без правильного маршруту

## 🧾 Висновок
- підмережі:
  - ділять великі мережі
- кожна підмережа:
  - має свій gateway
- використовуються:
  - маски
  - CIDR

## 📌 Головна ідея
> Підмережі роблять великі мережі керованими,  
розбиваючи їх на логічні частини

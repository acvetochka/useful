# Протоколи зовнішнього шлюзу (EGP)
## 📌 Визначення

`EGP (Exterior Gateway Protocols)` — це:
> протоколи, які використовуються для обміну маршрутами  
між різними автономними системами (AS)

**🧠 Простими словами**
- IGP → всередині компанії
- EGP → між компаніями / провайдерами

## 🌐 Чому це важливо

👉 Інтернет = це:
> мережа з мереж (мережа автономних систем)

**📌 Роль EGP**
- зв’язує різні AS
- дозволяє передавати дані по всьому світу
- забезпечує роботу Інтернету

## 🏢 Що таке автономна система (AS)

`Autonomous System (AS)` — це:
> група мереж під одним управлінням

**📡 Приклади:**
- інтернет-провайдер
- Google, Amazon
- велика корпорація

## 🔢 ASN — номер автономної системи
**📌 Визначення**

`ASN (Autonomous System Number)` — це:
> унікальний ідентифікатор автономної системи

**📊 Формат**
- 32-бітне число
- записується як одне число

👉 приклад:
```
19604
```

**🧠 Важливо**
- IP → ідентифікує пристрій
- ASN → ідентифікує всю мережу

## 🏛️ Хто це все контролює
🌐 Internet Assigned Numbers Authority

<details><summary>Internet Assigned Numbers Authority</summary>

`Internet Assigned Numbers Authority (IANA)` — організація, що відповідає за глобальну координацію унікальних ідентифікаторів Інтернету, зокрема адрес IP, доменів верхнього рівня та номерів протоколів. Її діяльність є критично важливою для стабільного функціонування Інтернету в усьому світі.

**Ключові факти**
- **Заснована:** 1988 р.
- **Поточний оператор:** Internet Corporation for Assigned Names and Numbers
- **Основні функції:** управління DNS, IP-адресами, параметрами протоколів
- **Штаб-квартира:** Лос-Анджелес, США
- **Ключовий документ:** Угода про функції IANA (2016 р.)

**Історія та роль у формуванні Інтернету**  
IANA виникла в рамках досліджень мереж ARPANET під керівництвом Джона Постела в Університеті Південної Каліфорнії. З розвитком Інтернету її завдання — централізоване керування технічними ідентифікаторами — стали основою глобальної сумісності мереж. У 1998 р. операції IANA було передано ICANN, що забезпечило відкриту, багатосторонню модель управління.

**Основні функції**  
IANA координує три основні області:
1. Адресація IP: розподіл блоків IPv4 та IPv6 між регіональними реєстраторами (наприклад, RIPE NCC, ARIN).
2. Доменна система імен (DNS): управління кореневою зоною та делегування доменів верхнього рівня (.com, .org, країнових).
3. Параметри протоколів: ведення реєстрів для стандартів, розроблених Internet Engineering Task Force.

**Управління та перехід від урядового контролю**  
До 2016 р. нагляд за IANA здійснював уряд США через NTIA. Після багаторічних консультацій функції було передано до спільноти зацікавлених сторін під егідою ICANN. Цей перехід зміцнив міжнародну легітимність та незалежність системи управління Інтернетом.

**Значення**  
IANA забезпечує унікальність адрес та назв в Інтернеті, підтримуючи його єдність і функціональну сумісність. Без централізованої координації IANA глобальна мережа могла б розпастися на несумісні частини, що суттєво обмежило б комунікацію та інновації.

</details>


**📌 Функції:**
- розподіл IP-адрес
- видача ASN
- координація глобального Інтернету

**❗ Чому це критично**

Без цього:
> кожен використовував би будь-які IP → хаос 😅

## 🌍 Як працює Інтернет (на високому рівні)
**📌 Основна ідея**

👉 головна задача:
> доставити пакет до потрібної автономної системи

**🔁 Процес**
1. маршрутизатор бачить IP
2. визначає:
  - до якої AS він належить
3. передає:
  - до “краю” цієї AS
4. далі вже працює IGP

## 🔗 Взаємодія IGP + EGP
📌 Логіка:

- EGP:
  > знаходить яку AS потрібно досягти

- IGP:
  > знаходить як дістатися всередині AS

## 🧠 Аналогія
- EGP = міжнародна доставка 🌍
- IGP = доставка всередині міста 🏙️

## ⚠️ Важливе зауваження

👉 Деталі EGP (особливо BGP):

- дуже складні
- зазвичай:
  - для ISP
  - великих компаній

## 📌 Головна ідея
> Інтернет працює завдяки тому,  
що автономні системи “домовляються” між собою через EGP

## 🧾 Висновок
- EGP = зв’язок між мережами
- ASN = ID мережі
- IANA = глобальний координатор
- разом → формують Інтернет

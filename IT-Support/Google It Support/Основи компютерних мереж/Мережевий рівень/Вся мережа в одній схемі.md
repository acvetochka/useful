# Вся мережа в одній схемі

![alt text](network/image-23.png)

## 🧠 Як читати цю схему (знизу вверх)

### 🔌 1. Physical (Фізичний рівень)
- кабелі / Wi-Fi
- передає біти (0 і 1)

### 🔗 2. Data Link (Ethernet)
- MAC-адреси
- кадри (frames)
- ARP (IP → MAC)

### 🌐 3. Network (IP)
- IP-адреси
- маршрутизація
- підмережі, CIDR

### 🚚 4. Transport
- TCP → надійно
- UDP → швидко

### 🌍 5. Application
- HTTP, HTTPS, SMTP
- те, чим ти реально користуєшся

## 📦 Як рухається запит (реальний сценарій)
**🖥️ Ти відкриваєш сайт:**
```
google.com
```
**🔁 Відбувається так:**
1. Application
    - браузер створює запит
2. Transport
    - TCP додає порт
3. Network
    - додається IP (наприклад 8.8.8.8)
4. Data Link
    - додається MAC (через ARP)
5. Physical
    - все це → у вигляді сигналів

## 🌍 Далі по Інтернету
**🔹 У твоїй мережі**
- пакет → router (default route)

**🔹 Router**
- дивиться IP
- шукає маршрут

**🔹 Інтернет**
- багато router'ів
- працює:
  - IGP (всередині мереж)
  - BGP (між мережами)

**🔹 Сервер**
- отримує пакет
- відповідає назад

## 🔐 Де тут NAT

👉 у твоєму роутері:
```
192.168.1.100 → 84.xxx.xxx.xxx
```

## 🧠 Як все пов’язано
**📦 Інкапсуляція (дуже важливо)**
```
[Ethernet frame]
  └── [IP packet]
        └── [TCP segment]
              └── [Data]
```

## 🔑 Головна ідея всієї схеми
> Дані “загортаються” в шари →  
проходять через мережу →  
“розгортаються” на іншому кінці

## ⚡ Супер-коротко (1 рядок)

> Інтернет =  
біти → кадри → пакети → маршрути → домовленості (BGP)

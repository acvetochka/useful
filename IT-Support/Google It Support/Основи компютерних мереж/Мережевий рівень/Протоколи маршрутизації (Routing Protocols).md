# Протоколи маршрутизації (Routing Protocols)
##  Вступ

Маршрутизатори “знають дороги” не самі по собі.

👉 Вони обмінюються інформацією через:
> протоколи маршрутизації

## 🧠 Що це таке

`Протоколи маршрутизації` — це:
> спеціальні протоколи, через які маршрутизатори  
обмінюються інформацією про мережі

**📡 Навіщо вони потрібні**
- оновлюють routing table
- знаходять найкращий шлях
- реагують на зміни в мережі

## 🧩 Типи протоколів
1️⃣ Внутрішні (IGP)

👉 працюють всередині однієї мережі

2️⃣ Зовнішні (EGP)

👉 працюють між різними мережами (Інтернет)

## 🏢 Що таке автономна система

`Autonomous System (AS)` — це:
> група мереж під контролем одного оператора

📌 Приклади:
- велика компанія
- інтернет-провайдер

## Протоколи внутрішнього шлюзу (IGP)
**📌 Визначення**
> використовуються для маршрутизації в межах однієї AS

**🧠 Основні типи:**
1. Distance Vector
2. Link State

### 1️⃣ Distance Vector (вектор відстані)
**📌 Ідея**

Маршрутизатор каже сусідам:
> “ось список мереж і скільки до них стрибків”

**🧾 Що передається:**
- список мереж
- відстань (hop count)

**🔁 Як працює**
1. роутер має таблицю
2. відправляє її сусідам
3. отримує їх таблиці
4. оновлює свою

**🧠 Приклад**

Було:
- Router A → 4 hops до мережі X
- Router B → 2 hops

👉 A отримує дані від B і думає:
> “через B швидше!”

👉 оновлює маршрут

**❗ Мінуси**
- мало інформації
- повільна реакція
- не бачить “повну картину”

### 2️⃣ Link State (стан зв’язку)
**📌 Ідея**

Маршрутизатор каже:
> “ось всі мої підключення і їх стан”

**🧾 Що передається:**
- інформація про всі інтерфейси
- стан кожного з’єднання

**🌍 Результат**

👉 кожен роутер знає:
> всю топологію мережі

**⚙️ Як працює**
1. всі маршрутизатори діляться інформацією
2. кожен будує “карту мережі”
3. запускає алгоритм
4. обирає найкращий шлях

**🧠 Плюси**
- швидше реагує
- точніші маршрути
- бачить всю мережу

**❗ Мінуси**
- більше пам’яті
- більше CPU

### ⚖️ Порівняння
| Характеристика | Distance Vector  | Link State  |
| -------------- | ---------------- | ----------- |
| Дані           | список маршрутів | повна карта |
| Швидкість      | повільна         | швидка      |
| Складність     | проста           | складна     |
| Точність       | нижча            | вища        |

## 🧠 Головна ідея

- Distance Vector:
  > “я знаю відстань до мереж”

- Link State:
  > “я знаю всю карту мережі”

## 🔥 Висновок
- IGP = маршрутизація всередині мережі
- існує 2 підходи:
  - простий (distance vector)
  - розумний (link state)

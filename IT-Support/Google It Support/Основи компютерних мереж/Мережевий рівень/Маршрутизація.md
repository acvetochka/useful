# Маршрутизація (Routing)
## Вступ

Інтернет = це:
> мережа з мільйонів мереж

👉 і все це працює завдяки:
> маршрутизації

## 📌 Визначення

`Маршрутизація` — це:
> процес вибору шляху для передачі даних між мережами

## 📡 Маршрутизатор (Router)
📌 Що це:
> пристрій, який пересилає пакети на основі IP-адреси

**🔧 Особливість:**
- має мінімум 2 мережеві інтерфейси
- підключений до різних мереж

**🧠 Простими словами:**
> router = “перехрестя для пакетів”

## 🔄 Як працює маршрутизація
**📌 Основні кроки:**
1. 📥 отримує пакет
2. 🔍 дивиться IP призначення
3. 📊 шукає в таблиці маршрутизації
4. 📤 відправляє через потрібний інтерфейс

🔁 І це повторюється:
- багато разів
- поки пакет не дійде

## 📋 Таблиця маршрутизації
📌 Що це:
> список маршрутів:
- куди йти
- через який інтерфейс

🧠 Простими словами:
> “карта доріг” для маршрутизатора

## 📦 Приклад (2 мережі)
**🌐 Мережі:**
- Network A → 192.168.1.0/24
- Network B → 10.0.0.0/24

**📡 Router:**
- A: 192.168.1.1
- B: 10.0.0.254

**💻 Сценарій:**
- 192.168.1.100 → 10.0.0.10

**🔄 Що відбувається:**  
1️⃣ ПК бачить:
- IP не локальний

👉 відправляє на gateway

2️⃣ Router:
- отримує пакет
- “знімає” Ethernet

3️⃣ Дивиться IP:
- 10.0.0.10

4️⃣ Шукає в таблиці

👉 знаходить:
- 10.0.0.0/24 → локально

5️⃣ Відправляє
- створює новий Ethernet-кадр
- зменшує TTL
- відправляє

> [!NOTE]
> 🔥 Важливий момент  
> 👉 кожен router:
> - змінює MAC-адреси
> - але IP не змінює

## 🌐 Приклад (3 мережі)
**📡 Додаємо:**
- Network C → 172.16.1.0/23

**🔗 Топологія:**
```
A → Router1 → B → Router2 → C
```

**💻 Сценарій:**
```
192.168.1.100 → 172.16.1.100
```

**🔄 Що відбувається:**  
**Router1:**
- не знає напряму
- бачить маршрут через:
  ```
  10.0.0.1
  ```

👉 відправляє туди

**Router2:**
- бачить, що мережа C локальна  
👉 відправляє кінцевому хосту

**🔁 Ключова ідея**
> пакет “стрибає” від router до router

## 🌍 Реальний Інтернет
**📌 Насправді:**
- десятки маршрутизаторів
- багато можливих шляхів
- резервні маршрути

**🧠 Простими словами:**
> це як GPS:
- є багато доріг
- вибирається найкраща

## ⚙️ Що змінюється на кожному кроці
📌 Router:
- ↓ TTL
- перераховує checksum
- змінює MAC  

❗ Але:
- IP призначення = той самий

## 🧾 Висновок
- маршрутизація:
  - доставляє пакети між мережами
- router:
  - дивиться IP
  - використовує таблицю
- пакет:
  - проходить через багато вузлів

## 📌 Головна ідея
> Інтернет працює, бо кожен маршрутизатор знає   
“куди передати пакет далі”

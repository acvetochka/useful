# Маска підмережі (Subnet Mask)

Підмережі: Network / Subnet / Host
## Вступ

Раніше IP-адреса ділилась на:
- Network ID (мережа)
- Host ID (хост)

👉 Але цього недостатньо для складних мереж

**📌 Нове поняття:**
> додається ідентифікатор підмережі (Subnet ID)

**💡Тепер IP =**
- мережа
- підмережа
- хост

**🧠 Простими словами:**
> IP = країна → місто → будинок

## 🔢 Як це реалізується
**📌 Ключова ідея:**

IP — це 32 біти  
👉 частина бітів “позичається” у хоста  
→ стає підмережею

**⚙️ Було:**
```
[Network][Host]
```

**⚙️ Стало:**
```
[Network][Subnet][Host]
```

## 🔗 Як працює маршрутизація
**📡 Рівні маршрутизації:**
- 🌍 Інтернет → дивиться тільки на мережу
- 🚪 Gateway → дивиться на підмережу
- 🖥️ Останній маршрутизатор → доставляє до хоста

**🧠 Простими словами:**
> великі маршрутизатори → приблизно  
локальні → точно

## 🔢 Маска підмережі (Subnet Mask)
**📌 Визначення:**

`маска підмережі` — це:
> число, яке визначає, де мережа, підмережа і хост

**📡 Формат:**
- теж 32 біти
- запис:
  ```
  255.255.255.0
  ```

**🔍 Як працює**

маска має:
- 1 (одиниці) → мережа + підмережа
- 0 (нули) → хости

**🧠 Простими словами:**
> 1 = “це мережа”  
0 = “це хост”

## 🔢 Приклад  
**📦 IP:**
```
9.100.100.100
```
**📦 Маска:**
```
255.255.255.0
```

**📡 Що означає:**
- перші 24 біти → мережа/підмережа
- останні 8 біт → хости

## 📊 Кількість хостів
**📌 Формула:**
```
2^(кількість бітів хоста)
```
**📌 Приклад:**

8 біт:
```
2⁸ = 256 адрес
```

⚠️ Але:
- 1 адреса → мережа
- 1 адреса → broadcast

👉 доступно:
- 254 хости
  
**🧠 Простими словами:**
> завжди “мінус 2”

## 🔢 CIDR-нотація
**📌 Сучасний запис:**

→ CIDR

**📡 Формат:**
```
IP/кількість_бітів_мережі
```

**📦 Приклад:**
```
9.100.100.100/24
```
👉 означає:
- 24 біти → мережа
- 8 біт → хости

## 🔢 Приклад складнішої маски
**📦 Маска:**
```
255.255.255.224
```
**📡 У двійковому вигляді:**
- 27 одиниць
- 5 нулів

**📊 Хости:**
```
2⁵ = 32 адреси
```
👉 доступно:
- 30 хостів

**📌 CIDR:**
```
/27
```

**🧠 Простими словами**
> чим більше “1” у масці →   
тим менше хостів, але більше підмереж

## ⚠️ Практичні моменти
***📌 Часті помилки:***
- неправильна маска
- різні підмережі → немає зв’язку
- неправильний gateway

**💡 Важливо для IT:**
> subnetting — одна з найчастіших причин мережевих проблем

## 🧾 Висновок
- IP тепер має:
  - мережу
  - підмережу
  - хост
- маска:
  - визначає межі
- CIDR:
  - сучасний стандарт

## 📌 Головна ідея
> Підмережа — це спосіб “розрізати” IP-простір,  
щоб мережа була керованою і масштабованою

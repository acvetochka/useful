Поширені рішення для роботи зі скриптами 

З цього матеріалу ви дізнаєтеся про різні скриптові мови, їх використання й ризики. Як фахівцю з IT-підтримки, вам може бути потрібно автоматизувати рутинні завдання, наприклад резервне копіювання даних компанії, яке виконується щоночі. Крім того, вам може знадобитись автоматизувати масштабні завдання, наприклад змінення параметрів захищеного доступу для тисяч файлів. Для автоматизації часто використовуються скрипти. Вони допомагають фахівцям з IT-підтримки заощадити час і ресурси в навантаженому робочому середовищі компанії.  

## Скриптові мови
Для виконання різних завдань у різних середовищах операційних систем доступно багато скриптових мов. Більшість скриптів пишуться в середовищах командного рядка.

### Скриптові мови для середовищ Windows

- `PowerShell (.ps1)`. Windows PowerShell – один із найпоширеніших інструментів для створення скриптів у командному рядку в середовищах Windows. PowerShell побудований на платформі .NET і використовує багато тих самих елементів, що й мови програмування. Крім автоматичного керування системою, скрипти PowerShell застосовуються для розробки, тестування й розгортання рішень.

- `Пакетні скрипти (.bat)`, які також називають пакетними файлами, існують із перших днів використання MS DOS і OS/2. Пакетні файли можуть виконувати прості завдання, наприклад запускати набір програм під час увімкнення комп’ютера. Цей тип скриптів може бути корисним для налаштування робочих областей працівників під час запуску комп’ютерів.

- `Visual Basic Script (.vbs)` – старіша скриптова мова. Термін дії її підтримки Microsoft закінчився. Її замінено на скрипти PowerShell. Однак у деяких застарілих системах вам ще можуть траплятися скрипти .vbs.

### Скриптові мови для середовищ Linux і Unix

- `Скрипти оболонки (.sh)`. Скриптові мови оболонки, як-от Bash, використовуються в середовищах Unix або Linux. Ці скрипти часто застосовуються для керування файлами, зокрема для зміни налаштувань безпеки, створення, копіювання, редагування, перейменування й видалення файлів. За їх допомогою також можна запускати програми, друкувати документи, переходити між елементами операційної системи тощо. Ці скрипти запускаються в оболонках інтерпретатора командного рядка (CLI), наприклад Bourne, Bourne Again SHell (Bash), C й Korn (KSH).

### Мови програмування, які можна використовувати для створення скриптів

- `JavaScript (.js)`. Це найпопулярніша у світі мова програмування. Вона достатньо проста й використовується для створення скриптів у сфері веб-розробки, мобільних і веб-додатків, ігор тощо. На її основі також можна розробляти програмне забезпечення й автоматизувати функції веб-сервера.

- `Python (.py)`. Це зручна мова програмування, за допомогою якої можна виконувати складні завдання й імпортувати модулі з бібліотек, розроблених спеціально для скриптів автоматизації.

## Поради щодо створення скриптів: пошук оптимального інструмента
- **Базові завдання автоматизації.** Python – чудовий скрипт для автоматизації. Це один із найпопулярніших інструментів із численними бібліотеками автоматизації.

- **Перезапуск комп’ютерів.** Багато досвідчених користувачів виконують перезапуск комп’ютерів (Windows) за допомогою скриптів PowerShell (.ps1). На пристроях з ОС Linux можна використовувати скрипти оболонки (.sh).

- П**ідключення мережевих дисків.** Раніше ця операція виконувалась за допомогою скриптів .bat або .vbs. Однак зараз для підключення дисків у середовищах Windows найчастіше використовуються скрипти PowerShell. Користувачі Linux можуть застосовувати для цього скрипти оболонки.

- В**становлення додатків.** Для автоматичного встановлення програмного забезпечення часто використовуються пакетні файли й скрипти оболонки.

- **Автоматичне резервне копіювання.** Скрипти Windows PowerShell і Linux/Unix дають змогу автоматизувати резервне копіювання.

- З**бирання інформації і даних.** Python – популярний вибір для цієї цілі. У Python доступно багато бібліотек.

- **Ініціювання оновлень.** Для ініціювання оновлень у Windows і Linux можна використовувати скрипти PowerShell і оболонки відповідно.

## Загрози безпеці під час використання скриптів
Фахівці з IT-підтримки повинні бути дуже обережними, використовуючи скрипти, особливо якщо вони скопійовані або завантажені з Інтернету. Нижче наведено деякі ризики для безпеки під час використання скриптів.

- **Випадкове встановлення зловмисного програмного забезпечення.** Як фахівець з IT-підтримки, який раніше не працював зі скриптами, ви можете спробувати знайти інформацію про написання скриптів в Інтернеті. У результатах пошуку може з’явитися скрипт для завдання, яке ви хочете автоматизувати. У вас може виникнути бажання заощадити час і зусилля, завантаживши скрипт і розгорнувши його у своєму мережевому середовищі. Однак це небезпечно, оскільки скрипти з неперевіреного джерела можуть містити зловмисне програмне забезпечення. Шкідливі скрипти можуть видаляти файли, пошкоджувати дані й програмне забезпечення, викрадати конфіденційну інформацію, вимикати системи й навіть зупиняти роботу всієї мережі. Вони можуть створювати слабкі місця в системі безпеки для проникнення кіберзлочинців у мережу. Скрипти також можуть запускати програми-вимагачі, які часто шифрують файлові системи, а потім вимагають заплатити за ключі розшифрування.

- **Ненавмисне змінення налаштувань системи.** Скрипти є потужними інструментами для змінення налаштувань системи. Використовуючи неправильний скрипт, можна випадково задати шкідливі налаштування. Наприклад, через одну незначну помилку в скрипті оболонки, яка встановлює рівень захисту файлів в ОС Linux, можна надати загальний доступ до конфіденційних файлів.  

- **Збої в роботі веб-переглядача чи системи через неналежне використання ресурсів**. Неправильне використання ресурсів може призвести до помилок веб-переглядача або збою в роботі всього комп’ютера. Наприклад, якщо спрямовувати у веб-переглядач забагато пам’яті, це може перевантажувати систему комп’ютера.

## Основні висновки
Базові навички створення скриптів – це важливий інструмент для ІТ-фахівців. Вам може знадобитися підвищити ефективність робочих процесів, автоматизувавши основні функції за допомогою скриптової мови. Нижче перелічено поширені скриптові мови.

- Середовища Windows: пакетні скрипти (.bat), Powershell (.ps1), Visual Basic Script (.vbs).

- Середовища Linux/Unix: скрипти оболонки (.sh).

- Більшість середовищ ОС: JavaScript (.js), Python (.py).

Завдання, які можна виконувати за допомогою скриптів:

- базова автоматизація;
- перезапуск комп’ютерів;
- перепідключення мережевих дисків;
- встановлення додатків;
- автоматичне резервне копіювання;
- збирання інформації і даних;
- ініціювання оновлень.

Ризики використання скриптів:

- випадкове встановлення зловмисного програмного забезпечення;

- ненавмисне змінення налаштувань системи;

- збої веб-переглядача або системи через неналежне використання ресурсів.

# Markdown

Повний гайд з Markdown, який охоплює все: від базового форматування до складних функцій розширених парсерів (GitHub Flavored Markdown).

## 1. Заголовки (Headers)
Використовуйте символ #. Кількість решіток відповідає рівню заголовка (від 1 до 6).
```markdown
# Заголовок 1 (H1)
## Заголовок 2 (H2)
### Заголовок 3 (H3)
###### Заголовок 6 (H6)
```

## 2. Форматування тексту
- __Жирний__: `**текст**` або `__текст__`
- _Курсив_: `*текст*` або `_текст_`
- ***Жирний курсив***: `***текст***`
- ~~Закреслений~~: `~~текст~~`
- Моноширинний (код): 
  ```md
  `текст` 
  ```

## 3. Списки
- Марковані:
  ```markdown
  * Пункт 1
  - Пункт 2
    * Вкладений пункт (2 пробіли відступу)
  ```
  Вигляд
  * Пункт 1
  - Пункт 2
    * Вкладений пункт (2 пробіли відступу)

- Нумеровані:
  ```markdown
  1. Перший
  2. Другий (цифри можуть бути будь-якими, браузер сам пронумерує по порядку)
  ```

  Вигляд
  1. Перший
  2. Другий 

- Списки завдань (Task Lists):
  ```markdown
  - [x] Завершене завдання
  - [ ] Незавершене завдання
  ```

  Вигляд 
  - [x] Завершене завдання
  - [ ] Незавершене завдання

## 4. Посилання та Зображення
- Посилання: 
  ```md
  [Назва](URL "Підказка при наведенні")

  # Приклад: [Google](https://google.com)
  ```
  

- Зображення: 
  ```md
  ![Опис](URL_картинки)
  # Приклад: ![Logo](https://example.com)
  ```
- Автопосилання: 
  ```md
  <https://google.com>
  ```

## 5. Цитати та Лінії
- Цитати:
  ```markdown
  > Це багаторівнева
  >> цитата
  ```

  Вигляд 
  > Це багаторівнева
  >> цитата

- Горизонтальна лінія (Роздільник):
  
  Використовуйте три або більше зірочок, дефісів або підкреслень.
  ```markdown
  ***
  ---
  ```

  Вигляд 

  ---

## 6. Блоки коду

- Інлайновий код: 
  ```markdown
  `print()` всередині речення.
  ```
- Блок коду: три бектіки (```) на початку та в кінці. Можна вказувати мову для підсвітки (python, sql, cpp тощо).
  
  [Докладніше про підтримувані мови]()

## 7. Таблиці
Використовуйте `|` для розділення колонок та `-` для заголовка. Двокрапки `:` визначають вирівнювання.
```markdown

| Заголовок 1 | Заголовок 2 | Заголовок 3 |
| :--- | :---: | ---: |
| Зліва | По центру | Справа |
| Текст | Текст | Текст |
```
  Вигляд 

  | Заголовок 1 | Заголовок 2 | Заголовок 3 |
| :--- | :---: | ---: |
| Зліва | По центру | Справа |
| Текст | Текст | Текст |

## 8. Спеціальні можливості (GFM & Extensions)
Це функції, які підтримуються не всюди, але дуже поширені (GitHub, Obsidian, VS Code).
- Footnotes (Виноски):

```markdown
Ось текст з виноскою[^1].
[^1]: А ось і сама виноска внизу сторінки.
```

Ось текст з виноскою[^1].

[^1]: А ось і сама виноска внизу сторінки.

- Emoji:
Використовуйте коди: `:smile:`, `:rocket:`, `:warning:`.

- Математичні формули (LaTeX):
  ```markdown
  $E = mc^2$ (в рядку)
  $$
  \frac{n!}{k!(n-k)!}
  $$ (окремий блок)
  ```

  Вигляд 

  $E = mc^2$ (в рядку)

$$
  \frac{n!}{k!(n-k)!}
$$

  > ПРИМІТКА
  >
  > GitHub підтримує математику, але він дуже прискіпливий до пробілів:
  > 
  > Блочна формула ($$): Має бути на окремих рядках, і перед/після блоку обов'язково мають бути порожні рядки.
  > 
  > Інлайнова ($): Не повинно бути пробілу між символом $ та текстом формули.

- HTML: Більшість парсерів дозволяють вставляти чистий HTML для того, що Markdown не вміє (наприклад, зміна кольору тексту або розміру картинки).
  ```html
  <ins>Підкреслений текст</ins>
  <img src="url" width="200">
  ```

## 9. Екранування символів
Якщо вам потрібно вивести символ, який є частиною розмітки (наприклад, саму зірочку `*`), додайте зворотний слеш `\`:
  ```markdown
  \*Цей текст не буде курсивом\*
  ```

  Вигляд

  \*Цей текст не буде курсивом\*

Це блоки коду з тегом `mermaid`, які перетворюються на графіки (Gantt, Flowchart, Pie Chart, ER-діаграми).

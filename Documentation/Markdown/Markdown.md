# Markdown

Повний гайд з Markdown, який охоплює все: від базового форматування до складних функцій розширених парсерів (GitHub Flavored Markdown).

## 1. Заголовки (Headers)
Використовуйте символ #. Кількість решіток відповідає рівню заголовка (від 1 до 6).
```markdown
# Заголовок 1 (H1)
## Заголовок 2 (H2)
### Заголовок 3 (H3)
###### Заголовок 6 (H6)
```

## 2. Форматування тексту
- __Жирний__: `**текст**` або `__текст__`
- _Курсив_: `*текст*` або `_текст_`
- ***Жирний курсив***: `***текст***`
- ~~Закреслений~~: `~~текст~~`
- Моноширинний (код): 
  ```md
  `текст` 
  ```

## 3. Списки
- Марковані:
  ```markdown
  * Пункт 1
  - Пункт 2
    * Вкладений пункт (2 пробіли відступу)
  ```
  Вигляд
  * Пункт 1
  - Пункт 2
    * Вкладений пункт (2 пробіли відступу)

- Нумеровані:
  ```markdown
  1. Перший
  2. Другий (цифри можуть бути будь-якими, браузер сам пронумерує по порядку)
  ```

  Вигляд
  1. Перший
  2. Другий 

- Списки завдань (Task Lists):
  ```markdown
  - [x] Завершене завдання
  - [ ] Незавершене завдання
  ```

  Вигляд 
  - [x] Завершене завдання
  - [ ] Незавершене завдання

## 4. Посилання та Зображення
- Посилання: 
  ```md
  [Назва](URL "Підказка при наведенні")

  # Приклад: [Google](https://google.com)
  ```
  

- Зображення: 
  ```md
  ![Опис](URL_картинки)
  # Приклад: ![Logo](https://example.com)
  ```
- Автопосилання: 
  ```md
  <https://google.com>
  ```

## 5. Цитати та Лінії
- Цитати:
  ```markdown
  > Це багаторівнева
  >> цитата
  ```

  Вигляд 
  > Це багаторівнева
  >> цитата

- Горизонтальна лінія (Роздільник):
  
  Використовуйте три або більше зірочок, дефісів або підкреслень.
  ```markdown
  ***
  ---
  ```

  Вигляд 

  ---

## 6. Блоки коду

- Інлайновий код: 
  ```markdown
  `print()` всередині речення.
  ```
- Блок коду: три бектіки (```) на початку та в кінці. Можна вказувати мову для підсвітки (python, sql, cpp тощо).
  
  [Докладніше про підтримувані мови]()

## 7. Таблиці
Використовуйте `|` для розділення колонок та `-` для заголовка. Двокрапки `:` визначають вирівнювання.
```markdown

| Заголовок 1 | Заголовок 2 | Заголовок 3 |
| :--- | :---: | ---: |
| Зліва | По центру | Справа |
| Текст | Текст | Текст |
```
  Вигляд 

  | Заголовок 1 | Заголовок 2 | Заголовок 3 |
| :--- | :---: | ---: |
| Зліва | По центру | Справа |
| Текст | Текст | Текст |


## Пробіли

Пробіли в Markdown — це не просто "порожнє місце", а активні керуючі символи. Новачки часто ігнорують їх, через що розмітка "ламається".

Ось розбір ситуацій, де пробіл вирішує все:
1. **Пробіл після символів розмітки (Обов'язковий)**
   
   Це найчастіша помилка. Більшість елементів вимагають пробілу після службового символу.
    
    - Заголовки: `#Заголовок` не спрацює. Треба `# Заголовок`.
    - Списки: `*Пункт` залишиться текстом. Треба `* Пункт`.
    - Цитати: `>Цитата` працює в деяких парсерах, але за стандартом треба `> Цитата`.

2. **Перенесення рядка (Правило "Двох пробілів")**
   
   У Markdown звичайне натискання Enter не завжди створює новий рядок у фінальному документі.
   - М'яке перенесення (Line Break): Щоб текст почався з нового рядка без створення нового абзацу, потрібно поставити два пробіли в кінці попереднього рядка.
   - Новий абзац: Потрібен один повністю порожній рядок між блоками тексту.

3. **Відступи для вкладеності (Lists & Quotes)**

   Відступи визначають ієрархію:
   - Для вкладеного списку потрібно 2 або 4 пробіли (залежить від налаштувань редактора, але 4 — це стандарт).
   - Якщо ви хочете вставити код або другий абзац всередину пункту списку, вони мають бути вирівняні по тексту пункту (зазвичай 4 пробіли).
      ```markdown
      1. Перший пункт
          * Вкладений пункт (4 пробіли)
          Це текст, що належить до першого пункту.
      ```

4. **Блоки коду та відступи**
   
   Якщо ви не використовуєте "бек-тіки" (```), то 4 пробіли або 1 Tab на початку рядка автоматично перетворять цей рядок на блок коду. Це часто стає причиною випадкового "ламання" верстки, коли текст раптом стає моноширинним у сірій рамці.
5. **Таблиці та вирівнювання**
   
   Пробіли всередині комірок таблиць `| текст |` зазвичай ігноруються і служать лише для зручності читання коду.   
   Однак, у розділювачі заголовка `:---:` пробіли навколо двокрапок ставити не можна, інакше вирівнювання (по центру/краю) може не спрацювати.

6. **Математика (LaTeX) та пробіли**
   
   Як ми з'ясували раніше, для GitHub це критично:
   - `$ E=mc^2 $` (з пробілами всередині) — не спрацює.
   - `$E=mc^2$` (впритул до знаків $) — спрацює. => $E=mc^2$
  
7. **Спеціальний нерозривний пробіл**
   
   Якщо вам потрібно поставити кілька пробілів поспіль (Markdown зазвичай стискає їх в один), використовуйте HTML-сутність: `&nbsp;`.   
   Кожен такий код додасть один чіткий пробіл, який не ігноруватиметься браузером.

## Як використовувати
```markdown
    ```<мова>
    код
    ```
```


Наприклад:

  ```python
  def hello():
      print("Hello, world!")
  ```

Якщо вказати невідому мову або не вказати її зовсім, Markdown просто відобразить текст у моноширинному шрифті.

## Приклади

  1) diff (Відображення змін)
    
      Використовується для порівняння версій коду. Рядки, що починаються з -, зазвичай підсвічуються червоним, а з + — зеленим.
  
      ```diff
        def hello_world():
      -    print("Привіт, Світ")
      +    print("Вітаю, Громадо!")
      ```

2)  mermaid (Діаграми та схеми)
  
    GitHub, GitLab та деякі плагіни VS Code рендерять цей код у графічні схеми.

      ```mermaid

          A[Початок] --> B{Користувач авторизований?};
          B -- Так --> C[Головна сторінка];
          B -- Ні --> D[Сторінка логіну];
      ```

    - Діаграма послідовності (Sequence Diagram):
      ```mermaid
      sequenceDiagram
          Користувач->>Сервер: Запит даних

          База Даних-->>Сервер: Результат
          Сервер-->>Користувач: JSON Відповідь
      ```

## У чому різниця між схожими мовами (в контексті Markdown)?
Головна відмінність не в тому, як ви пишете код, а в тому, як бібліотека підсвічування розфарбує ключові слова.

- Bash vs Sh
  - `sh` (Shell): Базова підсвітка стандартних команд `POSIX`.
  - `bash`: Розуміє специфічні "башизми" (наприклад, подвійні дужки [[ ]], масиви).

  Порада: Якщо пишете скрипт для `Linux` — використовуйте bash, це найнадійніший варіант для підсвічування.
  - `powershell` (або ps1): Має зовсім інший синтаксис (змінні через $, командлети Verb-Noun). 
  
    ```postgresql
    -- PostgreSQL (розуміє специфічні оператори типу :: для кастингу)
    SELECT '100'::integer, info->>'name' FROM users;
    ```
  Якщо позначити код PowerShell як bash, він майже не підсвітиться або підсвітиться некоректно.
- SQL vs PostgreSQL vs MySQL
  - `sql`: Універсальне підсвічування стандартних команд (SELECT, INSERT, JOIN).
  - `postgresql` / `mysql`: Додають підтримку специфічних типів даних (наприклад, JSONB у Postgres) та системних функцій, притаманних лише цій БД.

## Мови, що пишуться "несподівано"
Іноді назва мови відрізняється від того, що ми звикли бачити:

- React (JSX/TSX): Краще писати `jsx` або `tsx`, щоб правильно підсвічувались HTML-теги всередині коду.
- LaTeX (Математика): Часто використовується `latex` або `math` (якщо платформа підтримує рендеринг формул).
- Docker: Використовується `dockerfile`.
- Vim Script: Використовується `vim`.
Makefiles: Використовується `make`.

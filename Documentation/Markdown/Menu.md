## 1. Що таке зміст (TOC) у Markdown

Зміст — це список посилань на заголовки (#, ##, ###), який дозволяє швидко переходити по документу.

👉 Приклад:

- [Вступ](#вступ)
- [Встановлення](#встановлення)
- [Налаштування](#налаштування)

## 2. Встановлення Markdown All in One
Кроки:
1. Відкрий VS Code
2. Перейди в Extensions (Ctrl + Shift + X)
3. Знайди: Markdown All in One
4. Натисни Install

## 3. Підготовка документа

Щоб TOC працював, потрібні заголовки
```markdown
# Проект

## Вступ
## Встановлення
## Використання

### Команди
### Приклади
```

👉 Важливо:
- Використовуй тільки #, ##, ###
- Не роби пропусків у структурі (краще ієрархія)

## 4. Автоматичне створення TOC
🔹 Варіант 1 (найпростіший)
1. Постав курсор там, де хочеш зміст
2. Натисни:
    ```bash
    Ctrl + Shift + P
    ```
    Введи:
    ```
    Markdown All in One: Create Table of Contents
    ```
🔹 Варіант 2 (швидка команда)
  ```
  Ctrl + Shift + P → Create Table of Contents
  ```

## 5. Що ти отримаєш

Плагін автоматично створить:
```md
<!-- TOC -->

- [Проект](#проект)
  - [Вступ](#вступ)
  - [Встановлення](#встановлення)
  - [Використання](#використання)
    - [Команди](#команди)
    - [Приклади](#приклади)

<!-- /TOC -->
```
##  6. Автооновлення змісту

💡 Фішка плагіна — TOC оновлюється автоматично

Що потрібно:
Залишити коментарі:
```md
<!-- TOC -->
<!-- /TOC -->
```
👉 Після змін заголовків:
```
Ctrl + Shift + P → Update Table of Contents
```

## 7. Як формуються якорі (anchors)

Markdown перетворює заголовки в ID:

Приклад:
```md
## My Section Title
```
👉 стане:
```
#my-section-title
```

## Правила:
| Заголовок     | Якір           |
| ------------- | -------------- |
| `Hello World` | `#hello-world` |
| `Привіт світ` | `#привіт-світ` |
| `Section 1.2` | `#section-12`  |

👉 символи:

- пробіли → -
- великі літери → маленькі
- спецсимволи → видаляються

## 8. Налаштування Markdown All in One

Відкрий settings.json:
```
Ctrl + Shift + P → Preferences: Open Settings (JSON)
```
Корисні опції:
```json
{
  "markdown.extension.toc.levels": "2..4",
  "markdown.extension.toc.orderedList": false,
  "markdown.extension.toc.updateOnSave": true
}
```

🔍 Пояснення:
- levels → які заголовки включати
- orderedList → нумерований список
- updateOnSave → автооновлення при збереженні

## 9. Ручний TOC (без плагіна)

Можна писати вручну:
```md
- [Вступ](#вступ)
- [Встановлення](#встановлення)
```
👉 Але:
- ❌ довго
- ❌ легко помилитися
- ❌ не оновлюється

## 10. Best Practices
✅ Роби:
- Чітку ієрархію (# → ## → ###)
- Англійські заголовки (менше проблем з anchor)
- Не змінюй текст заголовків після TOC (або оновлюй)

❌ Уникай:
- Дублікатів заголовків
- Спецсимволів (!@#$)
- Занадто довгих назв

## 11. Просунутий варіант (частковий TOC)

Можна обмежити рівні:
```json
"markdown.extension.toc.levels": "2..3"
```
👉 Буде тільки:
```md
##
###
```

## 12. Типові проблеми
❌ TOC не працює

✔ Перевір:
- є заголовки #
- є <!-- TOC -->
- перезапусти команду

❌ Посилання не клікаються

✔ У preview (Ctrl + Shift + V) — працює  
✔ У GitHub — працює

❌ Українські заголовки ламаються

👉 іноді краще:
```md
## instalacia
📦 13. Ідеальний шаблон
# Project Name

<!-- TOC -->
<!-- /TOC -->

## Introduction
## Installation
## Usage
### Commands
### Examples
```

## 13. Виключення заголосока із TOC

### 13.1. Найпростіший спосіб — коментар `<!-- omit in toc -->`

👉 Просто додай цей коментар в кінець заголовка
```md
## Внутрішній розділ <!-- omit in toc -->
```
📌 Результат:
- заголовок залишиться в документі ✅
- але не з’явиться в змісті ❌

🔍 Приклад
```md
# Проект

<!-- TOC -->
<!-- /TOC -->

## Вступ
## Встановлення
## Секретний розділ <!-- omit in toc -->
```

👉 TOC буде таким:
```md
- [Проект](#проєкт)
  - [Вступ](#вступ)
  - [Встановлення](#встановлення)
```
(без "Секретний розділ")

### 13.2. Як це працює

Плагін просто ігнорує заголовки, де є:
```md
<!-- omit in toc -->
```
💡 Це офіційно підтримується самим розширенням.

### 13.3. Коли це використовувати
✅ Добре підходить для:
- технічних блоків
- внутрішніх нотаток
- допоміжних секцій (debug, draft, notes)
- повторюваних заголовків

### 13.4. Альтернатива — обмеження рівнів

Якщо потрібно виключити цілий клас заголовків, краще через налаштування:
```json
"markdown.extension.toc.levels": "2..3"
```
👉 Тоді:
- `#` — ❌ не включається
- `##` — ✅
- `###` — ✅
- `####` — ❌

### 13.5. Частий лайфхак

Іноді роблять так:
```md
## 🔧 Debug info <!-- omit in toc -->
```
👉 Виглядає красиво в документі, але:
- не засмічує TOC
- зручно для внутрішніх блоків

###  13.6. Типові помилки
❌ Не працює
```md
## Title <!--omit in toc-->
```
👉 Потрібен пробіл:
```md
## Title <!-- omit in toc -->
```
❌ Коментар не в тому місці
```md
<!-- omit in toc -->
## Title
```
👉 ❌ НЕ працює  
✔ Має бути в одному рядку з заголовком

### 13.7. Комбінування

Можеш комбінувати:
```md
## Main section
### Subsection
### Hidden subsection <!-- omit in toc -->
```
👉 У TOC:

- буде Main section
- буде Subsection
- НЕ буде Hidden subsection

### 13.8. Рекомендований підхід

👉 Найкраща практика:

- глобально → керуй рівнями (levels)
- локально → використовуй <!-- omit in toc -->

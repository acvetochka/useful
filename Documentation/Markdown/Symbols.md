# Символи в Markdown

Для оформлення Markdown-документів (особливо `README` на GitHub чи особистих нотаток в `Obsidian`) часто використовують `Unicode`-символи та `Emoji`. Вони не ламають розмітку, але роблять структуру візуально зрозумілою.

Ось підбірка, розбита за категоріями:

## 1. Маркери списків (замість стандартних * та -)
Використовуйте їх на початку рядка, щоб виділити важливі пункти:  
📌 Важливо: — для головних акцентів.  
✅ Done: — для завершених завдань.  
🚀 Реліз: — для запусків чи фіч.  
💡 Ідея: — для пропозицій.  
⚠️ Увага! — для попереджень.  
🔍 Деталі: — для аналізу.  

## 2. Спеціальні розділювачі та стрілки
Якщо стандартна лінія `---` здається нудною, можна використовувати текстові стрілки для логічних зв'язків:  
➔ ➜ ➤ ➥ — для вказування напрямку або переходів.  
∟ — для позначення вкладеності (наприклад, структура папок).  
— (довге тире) — Alt + 0151 на Windows.  
• ◦ ▪ ▫ — для багаторівневих списків вручну. 

Ці символи часто використовують для візуального оформлення логіки:  
`=>` перетворюється на ⇒ (у деяких парсерах).  
`->` перетворюється на →.  
`~>` — хвиляста стрілка.  
±, ×, ÷, ≈ — корисні для документації розрахунків.   

## 3. Оформлення блоків (Картки та секції)
Можна створювати візуальні "рамки" за допомогою емодзі на початку заголовків:  
📘 Документація (H2 заголовок)  
🛠️ Налаштування  
📦 Встановлення  
🧪 Тестування  
🤝 Внесок у проект (Contributing)  

### 3.1. Технічна документація та Код
⚙️ Налаштування (Setup / Config)  
💎 Особливості (Features)  
📦 Залежності / Пакети (Dependencies)  
🏗️ Архітектура (Architecture)  
🔌 Інтеграції / Плагіни (API / Plugins)  
🧹 Очищення / Рефакторинг (Refactoring)  
📜 Логи (Logs)  

### 3.2. Управління проектом та Робота
📅 Плани / Roadmap  
🎯 Цілі (Objectives)  
📈 Прогрес / Аналітика  
👥 Команда / Контакти  
⏳ В процесі (In Progress)  
🏁 Завершено (Milestones)  
💰 Бюджет / Витрати  

### 3.3. Навчання та Нотатки (для Obsidian)
📚 Теорія / Матеріали  
🧠 Концепти / Ідеї  
✍️ Вправи / Практика  
📑 Джерела / Референси  
🌍 Контекст / Огляд  
🌓 Порівняння (За/Проти)  

### 3.4. Пошук проблем та Підтримка  
🐛 Баги / Помилки (Known Issues)   
🚑 Термінові виправлення (Hotfix)  
🛡️ Безпека (Security)  
🆘 Допомога / FAQ  
💬 Обговорення (Feedback)  

### 3.5. Оформлення "підвалу" (Footer)
📄 Ліцензія (License)  
💜 Подяка (Acknowledgments)  
☕ Підтримати автора (Donate / Buy me a coffee)  

### 🔥 Лайфхак для заголовків:
Якщо ви хочете, щоб заголовок був чистим текстом, але мав іконку лише у візуальному режимі, деякі розробники ставлять емодзі в кінці. Це не відволікає при читанні коду:
```md
## Налаштування бази даних 🗄️
```

Також можна комбінувати символи для створення "візуальних папок":
```
📂 src/
&nbsp;&nbsp; 📄 main.py
&nbsp;&nbsp; 📄 utils.py
```
📂 src/
&nbsp;&nbsp; 📄 main.py
&nbsp;&nbsp; 📄 utils.py

## 4. Статуси та бейджі (Текстові)
Замість довгих слів можна використовувати кольорові квадрати для візуалізації пріоритетів:  
🔴 High (Критично)  
🟡 Medium (Середньо)  
🟢 Low (Низько)  
🔵 Info (Інформація)

## 5. Структура файлової системи (Дерево)
Щоб показати структуру проекту без блоку коду, використовуйте ці символи:
```text
my-project/
├── src/
│   ├── main.py
│   └── utils.py
├── tests/
└── README.md
```

(Ці вертикальні та горизонтальні лінії — це символи `│`, `├──`, `└──`)

## 6. HTML-бейджі (Просунутий рівень)
На GitHub часто використовують сервіс Shields.io, щоб створювати гарні динамічні плашки:  
  [Shields Badges](https://shields.io/badges)

  Підбірки готових плашок
 - [Ileriayo/markdown-badges](https://github.com/Ileriayo/markdown-badges)
 - [envoy1084/awesome-badges](https://github.com/envoy1084/awesome-badges)
 - [badges/awesome-badges](https://github.com/badges/awesome-badges)

**Статичні бейджі (Свої власні)**

 - Якщо вам потрібен просто гарний текст (наприклад, "Made with: C#"), на сайті є розділ "Your Badge".
 - Шаблон посилання виглядає так:
   ```md
     https://img.shields.io<LABEL>-<MESSAGE>-<COLOR>
   ```

## 💡 Порада для VS Code:
Натисніть `Win + .` (крапка), щоб відкрити панель емодзі прямо під час написання Markdown — це працює набагато швидше, ніж копіювання з сайтів.

## 7. Cистема Callouts (приміток)

Кожне ключове слово має свій колір та іконку.
На GitHub та в Obsidian підтримуються різні набори, але ось основні, які працюють майже всюди:

### 7.1. Стандартні типи (GitHub + Obsidian)
Використовуйте ці слова після > `[!...]`:
- `[!NOTE]` — Синій. Для загальної інформації.  
- `[!TIP]` — Зелений. Поради або лайфхаки.
- `[!IMPORTANT]` — Блакитний/Фіолетовий. Те, що не варто пропускати.
- `[!WARNING]` — Жовтий/Помаранчевий. Попередження про можливі помилки.
- `[!CAUTION]` — Червоний. Критична інформація, яка застерігає від небезпечних дій.

### Приклади

> [!NOTE] 
> Загальна інформація

> [!TIP] 
> Поради або лайфхаки.

> [!IMPORTANT]
> Те, що не варто пропускати.

> [!WARNING]
> Попередження про можливі помилки.

> [!CAUTION]
> Критична інформація, яка застерігає від небезпечних дій.



### 7.2. Розширені типи (Тільки Obsidian)
Obsidian підтримує набагато більше варіантів «з коробки»:
- `[!INFO]` — Аналог Note.
- `[!TODO]` — Синій з чекбоксом. Для списку справ.
- `[!EXAMPLE]` — Фіолетовий. Для прикладів коду чи тексту.
- `[!QUOTE]` — Сірий. Оформлення цитат.
- `[!SUCCESS]` — Зелений. Для успішних результатів.
- `[!QUESTION]` — Помаранчевий зі знаком питання.
- `[!FAILURE]` / `[!DANGER]` / `[!ERROR]` — Різні відтінки червоного для помилок.
- `[!ABSTRACT]` / `[!SUMMARY]` — Смарагдовий. Для короткого змісту.

Як це виглядає в коді:
```markdown
> [!TIP]
> **Порада:** Використовуйте `Ctrl` + `S` частіше!
```

> [!TIP]
> **Порада:** Використовуйте `Ctrl` + `S` частіше!

### Фішки Obsidian для Callouts:
- Згортання: Якщо додати мінус після дужок > `[!INFO]-`, блок буде закритим (згорнутим) за замовчуванням, поки ви на нього не натиснете.
- Свої заголовки: Ви можете змінити стандартне слово "Warning" на свій текст:
```markdown
> [!WARNING] Увага, важливий момент!
(Тоді замість англійського напису буде ваш).
```

> [!WARNING] Увага, важливий момент!
(Тоді замість англійського напису буде ваш).

## 8. Клавіші клавіатури `(<kbd>)`
Щоб показати, яку кнопку треба натиснути, використовуйте цей HTML-тег.

```md
<kbd>Ctrl</kbd> + <kbd>C</kbd>
```
Приклад: 

<kbd>Ctrl</kbd> + <kbd>C</kbd>

Результат: Ctrl + C (виглядає як фізична клавіша).

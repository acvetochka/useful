# РОЗДІЛ 1. МАСИВИ (ARRAYS)
## 1.1. Що таке структура даних

`Структура даних` — це спосіб організації та зберігання даних у пам’яті комп’ютера.

Інакше кажучи:

> структура даних визначає, як дані лежать у памʼяті і як до них звертатися.

Структура даних впливає на:

- швидкість доступу до елементів
- швидкість пошуку
- швидкість вставки
- використання памʼяті

**Приклад**

Можна зберігати числа різними способами:
```
5 8 2 9
```

- у масиві
- у списку
- у дереві
- у хеш-таблиці

І кожен варіант буде мати різну швидкість операцій.

## 1.2. Що таке масив

`Масив (Array)` — це структура даних, у якій елементи зберігаються послідовно в пам’яті.

Наприклад:
```
[10, 20, 30, 40]
```
У памʼяті це виглядає приблизно так:
```
адреса 1000 → 10
адреса 1001 → 20
адреса 1002 → 30
адреса 1003 → 40
```
Тобто елементи лежать підряд.

## 1.3. Індексація

Кожен елемент має індекс.

`Індекс` — це позиція елемента.

```
Індекс:   0   1   2   3
Масив:  [10, 20, 30, 40]
```
У більшості мов програмування індексація починається з 0.

Доступ до елемента
```python
arr[2]
```
Результат:

> 30

## 1.4. Чому доступ до масиву дуже швидкий

Це одна з головних причин популярності масивів.

Адресу елемента можна обчислити за формулою:
```
address = base_address + index × size
```
Наприклад:
> 1000 + 2 × 1 = 1002

Процесор одразу знає, де лежить елемент.

Тому доступ має складність:
```
O(1)
```
Це називається random access (довільний доступ).

## 1.5. Основні операції з масивом
- Доступ до елемента
    ```python
    arr[i]
    ```
    Складність:
    ```
    O(1)
    ```

- Зміна елемента
    ```python
    arr[i] = value
    ```
    Складність:
    ```
    O(1)
    ```

- Перебір масиву
    ```python
    for x in arr:
    ```
    Складність:
    ```
    O(n)
    ```

- Пошук елемента

    Якщо масив не відсортований:
    ```
    O(n)
    ```
    Бо потрібно перевіряти кожен елемент.

## 1.6. Вставка елемента

Це слабке місце масивів.

Приклад:
```python
[10, 20, 30, 40]
```
Вставимо 15 у позицію 1.

Було:
```
10 20 30 40
```
Потрібно пересунути елементи:
```
10 15 20 30 40
```
Всі елементи справа зсуваються.

Складність:
```
O(n)
```

## 1.7. Видалення елемента

Те саме.

Було:
```
10 20 30 40
```
Видалимо 20:
```
10 30 40
```
Елементи потрібно зсунути.

Складність:
```
O(n)
```

## 1.8. Фіксований розмір масиву

У класичних масивах (наприклад, у C):

Розмір задається під час створення.
```py
int arr[10]
```

І не може змінюватися.

Якщо потрібно більше місця — створюється новий масив.

## 1.9. Динамічні масиви

Більшість сучасних мов використовує динамічні масиви.

Наприклад:

- Python list
- Java ArrayList
- C++ vector
- JavaScript Array

Коли масив заповнюється:

1. створюється більший масив
2. дані копіюються
3. старий видаляється

**Чому це все одно швидко**

Розширення відбувається рідко.

Тому середня складність вставки в кінець:
```
O(1) amortized
```
(амортизована складність).

## 1.10. Переваги масивів
1. Дуже швидкий доступ до елементів — O(1)
2. Простота структури
3. Хороша локальність памʼяті
4. Ефективна робота CPU кешу

## 1.11. Недоліки масивів
1. Повільні вставки
2. Повільні видалення
3. Може витрачатися зайва памʼять
4. Фіксований розмір у класичній реалізації


## 1.12. Коли використовують масиви

Масиви використовують, коли:

- потрібен швидкий доступ за індексом
- дані читаються частіше, ніж змінюються
- порядок елементів важливий

## 1.13. Реальні приклади використання

Масиви використовуються практично всюди:

- зображення (матриця пікселів)
- відео
- таблиці даних
- обробка сигналів
- алгоритми сортування
- матриці у машинному навчанні

## Підсумок розділу

Масив — це:

- послідовна структура даних у памʼяті
- з дуже швидким доступом до елементів
- але повільними вставками та видаленням.

Основні складності:

| Операція  | Складність |
| --------- | ---------- |
| доступ    | O(1)       |
| зміна     | O(1)       |
| пошук     | O(n)       |
| вставка   | O(n)       |
| видалення | O(n)       |

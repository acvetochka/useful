# РОЗДІЛ 6. SET ТА MAP

## 6.1. Що таке Abstract Data Type (ADT)

Перед початком потрібно зрозуміти одну важливу річ.

До цього ми вивчали:

- Array
- Linked List
- Stack
- Queue
- Hash Table

Це були переважно структури реалізації.

Тепер ми будемо говорити про:

- Set
- Map

Це часто називають:
 `Abstract Data Types (ADT)` 
або
`Абстрактні типи даних`

Ідея проста:
> ADT описує, ЩО можна робити з даними.

А реалізація описує:
> ЯК це зробити всередині.

Наприклад:
```
Stack
```
можна реалізувати через:

- Array
- Linked List

Аналогічно:
```
Set
```
можна реалізувати через:

- Hash Table
- Tree

## 6.2. Що таке Set

`Set` (множина) — це структура даних, яка зберігає лише унікальні значення.

Математична множина:
```
{1, 2, 3}
```
Не може містити:
```
{1, 2, 2, 3}
```
Дублікати автоматично прибираються.

## 6.3. Основна властивість Set

У множині:
```
1
```
або є,

або немає.

Кількість повторень не важлива.

Приклад:

Було:
```py
{1, 2, 3}
```
Додаємо:
```py
2
```
Результат:
```py
{1, 2, 3}
```
Нічого не змінилося.

## 6.4. Операції Set
- **Add**
    ```py
    s.add(10)
    ```
    Середня складність:
    ```
    O(1)
    ```

- **Remove**
    ```py
    s.remove(10)
    ```
    Середня складність:
    ```
    O(1)
    ```

- **Contains**

    Перевірка наявності.
    ```py
    10 in s
    ```
    Середня складність:
    ```
    O(1)
    ```

## 6.5. Чому Set такий швидкий

Тому що більшість реалізацій використовують:
```
Hash Table
```
всередині.

Фактично Set можна уявити як:
```py
{
    10: True,
    20: True,
    30: True
}
```
Значення не важливе.

Важливий лише ключ.

## 6.6. Найпоширеніше застосування Set

Перевірка унікальності.

Задача:
> Чи є дублікати?

Погане рішення:
```py
for i:
    for j:
```
Складність:
```
O(n²)
```
Хороше рішення:
```py
seen = set()

for x in arr:
    if x in seen:
        return True

    seen.add(x)
```
Складність:
```
O(n)
```
Це одна з найважливіших оптимізацій в алгоритмах.

## 6.7. Математичні операції над Set

Set підтримує операції з теорії множин.

- **Union (об'єднання)**
    ```py
    A = {1,2,3}
    B = {3,4,5}
    ```
    Результат:
    ```
    {1,2,3,4,5}
    ```

- **Intersection (перетин)**

    Спільні елементи.
    ```py
    {3}
    ```

- **Difference (різниця)**
    ```py
    A - B
    ```
    Результат:
    ```
    {1,2}
    ```

## 6.8. Що таке Map

`Map` — це структура виду:
> ключ → значення

Фактично:
> Hash Table

і є найпоширенішою реалізацією Map.

Приклад:
```py
{
    "Ivan": 31,
    "Anna": 25
}
```
Тут:

- ключ     → Ivan
- значення → 31

## 6.9. Навіщо потрібен Map

Уявімо список:
```py
[
 ("Ivan", 31),
 ("Anna", 25)
]
```
Щоб знайти:
```
Ivan
```
доведеться перебирати.

Через Map:
```py
ages["Ivan"]
```
Отримуємо:
```
O(1)
```

## 6.10. Set vs Map

Це питання дуже люблять на співбесідах.

**Set**

Зберігає:
- значення

Приклад:
```py
{1,2,3}
```

Map

Зберігає:
- ключ → значення

Приклад:
```py
{
 "Ivan":31
}
```

## 6.11. Python Set
```py
s = {1,2,3}
```
або
```py
s = set()
```
Операції:
```py
s.add(4)

4 in s

s.remove(4)
```

Усі середньо:

```
O(1)
```

## 6.12. Python Dict
```py
user = {
    "name": "Ivan"
}
```
Доступ:
```py
user["name"]
```
Середньо:
```
O(1)
```

## 6.13. JavaScript Object
```js
const user = {
  name: "Ivan"
}
```
Доступ:
```js
user.name
```
Історично Object використовувався як Map.

Але є нюанси.

## 6.14. JavaScript Map

Сучасна структура.
```js
const users = new Map();
```
Запис:
```js
users.set("Ivan", 31);
```

Отримання:
```js
users.get("Ivan");
```
Переваги над Object:

- будь-який тип ключів;
- простіше працювати;
- передбачуваніша поведінка.
 
 ## 6.15. JavaScript Set
 ```js
const s = new Set();
```
Додавання:
```js
s.add(10);
```
Перевірка:
```js
s.has(10);
```

## 6.16. Частотний словник (Frequency Map)

Одна з найважливіших технік.

Задача:
> Порахувати слова.

Рядок:
```
cat dog cat cat dog
```
Результат:
```js
{
  "cat": 3,
  "dog": 2
}
```


Алгоритм:
```js
counts[word] += 1
```
Це використовується буквально всюди:

- пошук
- NLP
- аналітика
- статистика

## 6.17. Найважливіший патерн співбесід

Дуже багато задач мають схему:

Погано:
```
подвійний цикл

O(n²)
```

Добре:
```
HashSet
або
HashMap

O(n)
```
Тому при аналізі задачі завжди варто питати себе:
> Чи можна запам'ятати вже побачені елементи?

## 6.18. Найважливіший інсайт

Set і Map не існують самі по собі.

Вони майже завжди побудовані поверх:

- Hash Table
- або Tree

Саме тому всі їхні швидкі операції походять від хешування або дерев.

## Підсумок розділу
**Set**

Зберігає:
- унікальні значення

Основні операції:

| Операція | Складність |
| -------- | ---------- |
| add      | O(1)       |
| contains | O(1)       |
| remove   | O(1)       |


**Map**

Зберігає:
- ключ → значення

Основні операції:

| Операція | Складність |
| -------- | ---------- |
| get      | O(1)       |
| put/set  | O(1)       |
| delete   | O(1)       |


`Set` і `Map` — одні з найважливіших інструментів для оптимізації алгоритмів.

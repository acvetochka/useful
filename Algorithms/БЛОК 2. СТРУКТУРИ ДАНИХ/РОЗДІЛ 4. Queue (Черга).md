# РОЗДІЛ 4. QUEUE (ЧЕРГА)

## 4.1. Що таке Queue

`Queue` (черга) — це структура даних, яка працює за принципом:

```
FIFO
First In, First Out
```

тобто:
> Перший елемент, який потрапив у чергу, буде витягнутий першим.

Аналогія з магазином

Уявімо чергу:
```
Анна
Богдан
Віктор
Ганна
```
Анна прийшла першою.

Тому першою і буде обслугована.

Порядок виходу:
```
Анна
Богдан
Віктор
Ганна
```
На відміну від стеку:
```
Stack  → LIFO
Queue  → FIFO
```

## 4.2. Основні операції Queue

У черги є дві основні точки:
```
Front ← [Queue] ← Rear
```
або
```
Початок ← [Queue] ← Кінець
```

- **Enqueue**

    Додати елемент у кінець черги.

    Було:
    ```
    10 20 30
    ```
    Виконуємо:
    ```py
    enqueue(40)
    ```
    Стало:
    ```
    10 20 30 40
    ```
    Складність:
    ```
    O(1)
    ```

- **Dequeue**

    Видалити елемент з початку.

    Було:
    ```
    10 20 30 40
    ```
    Виконуємо:
    ```py
    dequeue()
    ```
    Отримуємо:
    ```
    10
    ```
    Черга стає:
    ```
    20 30 40
    ```
    Складність:
    ```
    O(1)
    ```

- **Front (Peek)**

    Подивитися перший елемент.

    Було:
    ```
    10 20 30 40
    ```
    Результат:
    ```
    10
    ```
    Черга не змінюється.

    Складність:
    ```
    O(1)
    ```

- **Is Empty**

    Перевірка чи черга порожня.

    Складність:
    ```
    O(1)
    ```

## 4.3. Візуалізація Queue
```
Front                Rear
 ↓                    ↓

[10] [20] [30] [40]
```

Додавання
```py
enqueue(50)
```
```
Front                     Rear
 ↓                         ↓

[10] [20] [30] [40] [50]
```

Видалення
```py
dequeue()
```
```

Front                Rear
 ↓                    ↓

[20] [30] [40] [50]
```

## 4.4. Чому звичайний масив — погана реалізація Queue

На перший погляд можна зробити так:
```py
queue = [10, 20, 30]
```
Додавання:
```py
queue.append(40)
```
Швидко:
```
O(1)
```
Але видалення:
```py
queue.pop(0)
```
Проблема:

Було:
```
10 20 30 40
```
Після видалення:
```
20 30 40
```
Потрібно пересунути всі елементи.

Складність:
```
O(n)
```
Тому звичайний масив — не найкращий варіант.

## 4.5. Реалізація через Linked List

Чергу дуже зручно будувати через зв'язаний список.

Зберігаємо два покажчики:
```
head
tail
```
Приклад:
```
head
 ↓
10 → 20 → 30 → 40
                  ↑
                 tail
```

Додавання
```
tail.next = newNode
tail = newNode
```

Складність:
```
O(1)
```

Видалення
```
head = head.next
```

Складність:
```
O(1)
```

## 4.6. Circular Queue

Є ще одна популярна реалізація.

Називається:
> Circular Queue

або
 > Кільцева черга

Уявімо масив:
```
[ ][ ][ ][ ][ ]
```
Коли доходимо до кінця:
```
0 1 2 3 4
```
починаємо знову з початку.

Схематично:
```
0 → 1 → 2 → 3 → 4
↑               ↓
└───────────────┘
```
Так працюють:

- мережеві буфери
- драйвери
- операційні системи

## 4.7. Deque (Double Ended Queue)

Дуже важлива структура.

Назва:
> Deque
(Double Ended Queue)



Дозволяє:

- додавати з початку;
- додавати з кінця;
- видаляти з початку;
- видаляти з кінця.

```
← [Deque] →
```
Операції:
```py
push_front()
push_back()

pop_front()
pop_back()
```
У Python:
```py
from collections import deque
```
Deque використовується дуже часто в алгоритмах.

Набагато частіше, ніж класична Queue.

## 4.8. Queue в операційних системах

Практично всі ОС використовують черги.

**Планувальник процесів**

Процеси чекають виконання.
```
Процес 1
Процес 2
Процес 3
```
CPU бере їх по черзі.

**Черга друку**
```
Документ A
Документ B
Документ C
```
Принтер друкує в порядку надходження.

## 4.9. Queue у веб-розробці

Дуже популярний сценарій.

Наприклад:

Користувачі завантажують відео.

Якщо одразу обробляти всі відео:

- сервер перевантажиться.

Тому створюється черга задач.
```
Video 1
Video 2
Video 3
```
Працівники (workers) обробляють їх по черзі.

Саме так працюють:

- Celery
- RabbitMQ
- Kafka
- AWS SQS

## 4.10. Queue і BFS

Коли будемо вивчати графи, побачимо:
```
BFS
Breadth First Search
```

повністю побудований на Queue.

Без черги BFS не існує.

## 4.11. Пріоритетна черга (Priority Queue)

Не всі елементи однаково важливі.

Наприклад:
```
Пацієнт А (критичний)
Пацієнт Б (легкий випадок)
```
Лікар прийме:
```
Пацієнт А
```

хоча він прийшов пізніше.

Тобто порядок визначає не час надходження, а пріоритет.

Ця структура дуже важлива і пізніше приведе нас до Heap.

## 4.12. Stack vs Queue
| Властивість  | Stack    | Queue      |
| ------------ | -------- | ---------- |
| Принцип      | LIFO     | FIFO       |
| Додавання    | зверху   | в кінець   |
| Видалення    | зверху   | з початку  |
| Використання | рекурсія | планування |


Приклад

Було додано:
```
1 2 3 4
```
Stack поверне:
```
4 3 2 1
```
Queue поверне:
```
1 2 3 4
```

## 4.13. Найважливіший інсайт

`Stack` і `Queue` — це не стільки способи зберігання даних, скільки способи керування порядком обробки даних.

Самі елементи можуть лежати:

- у масиві;
- у зв'язаному списку;
- у спеціальній структурі.

Головне — правила доступу до них.

## Підсумок розділу

Queue працює за принципом:
- FIFO
- First In, First Out

Основні операції:

| Операція | Складність |
| -------- | ---------- |
| enqueue  | O(1)       |
| dequeue  | O(1)       |
| front    | O(1)       |
| isEmpty  | O(1)       |


Queue використовується для:

- планування задач;
- мережевих систем;
- обробки повідомлень;
- BFS;
- операційних систем;
- черг друку.

# РОЗДІЛ 2. LINKED LIST (ЗВ'ЯЗАНИЙ СПИСОК)

## 2.1. Чому з'явилися зв'язані списки

У попередньому розділі ми побачили проблему масивів.

Приклад:
```
[10, 20, 30, 40]
```
Потрібно вставити число 15 між 10 і 20.

Отримаємо:
```
[10, 15, 20, 30, 40]
```
Для цього потрібно:

1. Знайти місце вставки.
2. Пересунути всі елементи праворуч.

Якщо масив містить мільйон елементів, може знадобитися пересунути сотні тисяч значень.

Саме для вирішення цієї проблеми була створена структура Linked List.

## 2.2. Основна ідея Linked List

У масиві елементи лежать у пам'яті підряд:
```
10 20 30 40
```
У Linked List елементи можуть лежати будь-де.

Кожен елемент знає:

- своє значення;
- адресу наступного елемента.

Такий елемент називається вузол (Node).

Візуально
```
[10|•] → [20|•] → [30|•] → [40|NULL]
```
де:

- ліва частина — дані;
- права частина — посилання на наступний вузол.

Реальна картина в пам'яті

Уявімо:
```
Адреса 1000 → Node(10)
Адреса 5020 → Node(20)
Адреса 8800 → Node(30)
Адреса 2100 → Node(40)
```
Вузли можуть бути розкидані по всій пам'яті.

Головне — що вони з'єднані посиланнями.

## 2.3. Що таке Node

Вузол зазвичай містить два поля.
```py
class Node:
    value
    next
```
Наприклад:
```py
value = 20
next = адреса вузла з 30
```

## 2.4. Що таке Head

Для роботи зі списком потрібно знати, де він починається.

Для цього зберігається спеціальний покажчик:
```
head
```
Приклад:
```
head
 ↓
10 → 20 → 30 → 40
```
Саме через head ми отримуємо доступ до всіх елементів.

## 2.5. Доступ до елементів

Тут з'являється головна відмінність від масиву.

**Масив**

Щоб отримати третій елемент:
```py
arr[2]
```
Складність:
```
O(1)
```

**Linked List**

Щоб отримати третій елемент:
```
head
 ↓
10 → 20 → 30
```
Потрібно:

1. перейти до 10;
2. перейти до 20;
3. перейти до 30.

Складність:
```
O(n)
```
Тобто Linked List НЕ підтримує швидкий доступ за індексом.

## 2.6. Пошук елемента

Пошук виконується послідовно.
```
10 → 20 → 30 → 40
```
Шукаємо 40.

Потрібно пройти:
```
10
20
30
40
```
Складність:
```
O(n)
```

## 2.7. Вставка елемента

Ось де Linked List стає сильним.

Було:
```
10 → 20 → 40
```
Потрібно вставити:
```
30
```
У масиві:
- пересуваємо елементи.

У Linked List:
```
20.next = 30
30.next = 40
```
Все.

Вставка після відомого вузла:
```
O(1)
```
Це одна з головних переваг структури.

## 2.8. Видалення елемента

Було:
```
10 → 20 → 30 → 40
```
Видаляємо:
```
30
```
Достатньо змінити посилання:

Було:
```
20 → 30 → 40
```
Стало:
```
20 → 40
```
Складність:
```
O(1)
```
якщо вузол уже знайдений.

## 2.9. Важливий нюанс

Часто пишуть:
```
Видалення O(1)
```
Але це не завжди правда.

Потрібно розрізняти:

Пошук вузла
```
O(n)
```

Саме видалення
```
O(1)
```
Тому повна операція часто:
```
O(n)
```
бо спочатку потрібно знайти вузол.

## 2.10. Singly Linked List

Найпростіший варіант.

Кожен вузол знає лише наступний.
```
10 → 20 → 30 → 40
```
Рух можливий лише вперед.

## 2.11. Doubly Linked List

Кожен вузол знає:

- попередній;
- наступний.

Вигляд:
```
NULL ← 10 ⇄ 20 ⇄ 30 ⇄ 40 → NULL
```
Node:
```py
class Node:
    value
    prev
    next
```
Переваги:

- можна рухатися назад;
- простіше видаляти елементи.

Недоліки:

- більше пам'яті;
- складніша реалізація.

## 2.12. Circular Linked List

Останній вузол вказує на перший.

Замість:
```
40 → NULL
```
буде:
```
40 → 10
```
Отримаємо цикл:
```
10 → 20 → 30 → 40
↑              ↓
└──────────────┘
```
Використовується рідко, але корисний для:

- планувальників задач;
- циклічних буферів;
- ігор.

## 2.13. Порівняння Array і Linked List
| Операція           | Array | Linked List |
| ------------------ | ----- | ----------- |
| Доступ за індексом | O(1)  | O(n)        |
| Пошук              | O(n)  | O(n)        |
| Вставка в середину | O(n)  | O(1)*       |
| Видалення          | O(n)  | O(1)*       |
| Пам'ять            | менше | більше      |

* якщо вузол уже знайдений.

## 2.14. Чому Linked List використовують рідше, ніж здається

Після вивчення списків багато новачків думають:

> "О, вставка O(1), значить Linked List кращий."

На практиці часто ні.

Причина:

> Масиви дуже дружать із CPU-кешем

**Масив:**
```
10 20 30 40 50
```
лежить поруч у пам'яті.

Процесор читає його дуже швидко.

**Linked List:**
```
10 → 20 → 30 → 40
```
може бути розкиданий по різних ділянках пам'яті.

Через це реальна швидкість часто гірша, ніж підказує Big O.

## 2.15. Де реально використовують Linked List

Сьогодні напряму їх використовують не так часто, але вони лежать в основі багатьох структур:

- HashMap (у деяких реалізаціях)
- LRU Cache
- черги
- деякі файлові системи
- менеджери пам'яті

## Підсумок розділу

`Linked List` — це структура, де кожен елемент зберігає посилання на наступний.

Основна ідея:
> швидка зміна зв'язків замість переміщення даних.

Основні складності:

| Операція            | Складність |
| ------------------- | ---------- |
| доступ за індексом  | O(n)       |
| пошук               | O(n)       |
| вставка після вузла | O(1)       |
| видалення вузла     | O(1)       |
| пам'ять             | O(n)       |

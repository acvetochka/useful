# РОЗДІЛ 3. STACK (СТЕК)

## 3.1. Що таке стек

`Stack` (стек) — це структура даних, яка працює за принципом:
```
LIFO
Last In, First Out
```
тобто:
> Останній доданий елемент буде витягнутий першим.

**Аналогія зі стопкою тарілок**

Уяви стопку тарілок.

Якщо покласти тарілки так:
```
4
3
2
1
```
то взяти спочатку можна тільки верхню:
```
4
```
Після цього:
```
3
2
1
```
Тобто порядок буде:

Поклали:
```
1 2 3 4
```
Витягнули:
```
4 3 2 1
```
Саме так працює стек.

## 3.2. Основні операції стеку

У стеку дуже мало операцій.

- **Push**

    Додати елемент на вершину стеку.

    Було:
    ```
    3
    2
    1
    ```
    Виконали:
    ```
    push(4)
    ```
    Стало:
    ```
    4
    3
    2
    1
    ```
    Складність:
    ```
    O(1)
    ```

- **Pop**

    Видалити верхній елемент.

    Було:
    ```
    4
    3
    2
    1
    ```
    Виконали:
    ```
    pop()
    ```
    Отримали:
    ```
    4
    ```
    Залишилось:
    ```
    3
    2
    1
    ```
    Складність:
    ```
    O(1)
    ```

- **Peek (Top)**

    Подивитися верхній елемент без видалення.

    Було:
    ```
    4
    3
    2
    1
    ```
    Результат:
    ```
    4
    ```

    Стек не змінюється.

    Складність:
    ```
    O(1)
    ```

- **Is Empty**

    Перевірити чи стек порожній.
    ```
    stack.is_empty()
    ```
    Складність:
    ```
    O(1)
    ```

## 3.3. Візуалізація стеку

Стек часто малюють так:
```
TOP
 ↓
[40]
[30]
[20]
[10]
```

Тільки верхній елемент доступний напряму.

## 3.4. Реалізація через масив

Найчастіше стек реалізують через звичайний масив.

Було:
```
[10, 20, 30]
```
Push:
```
[10, 20, 30, 40]
```
Pop:
```
[10, 20, 30]
```
Чому це швидко?

Тому що додаємо і видаляємо тільки кінець масиву.

Складність:
```
push = O(1)
pop = O(1)
peek = O(1)
```

## 3.5. Реалізація через Linked List

Можна реалізувати стек через список.

Вершина стеку буде головою списку.

Було:
```
30 → 20 → 10
```
Push:
```
40 → 30 → 20 → 10
```
Pop:
```
30 → 20 → 10
```
Також:
```
O(1)
```

## 3.6. Стек викликів функцій (Call Stack)

Тепер повертаємося до рекурсії.

Було:
```
foo()
```
У стек додається:
```
foo
```
Якщо всередині:
```py
foo():
    bar()
```

Стек:
```
bar
foo
```
Якщо:
```py
bar():
    baz()
```

Стек:
```
baz
bar
foo
```

Коли baz завершується:
```
bar
foo
```
Коли bar завершується:
```
foo
```
Це чистий LIFO.

## 3.7. Чому рекурсія використовує багато пам'яті

Кожен рекурсивний виклик додає новий запис у стек.

Наприклад:
```py
factorial(5)
```
Стек:
```
factorial(5)
factorial(4)
factorial(3)
factorial(2)
factorial(1)
factorial(0)
```

Глибина рекурсії:
```
n
```
Пам'ять:
```
O(n)
```

## 3.8. Stack Overflow

Якщо стек стає занадто великим:
```py
def f():
    f()
```
отримаємо:
```
Stack Overflow
```
Тобто місце у стеку закінчилося.

## 3.9. Де використовуються стеки
### 1. Рекурсія

Найвідоміший приклад.

### 2. Кнопка "Назад" у браузері

Відкрили:
```
Google
YouTube
Wikipedia
```
Стек:
```
Wikipedia
YouTube
Google
```
Натискаємо Back:
```
Wikipedia
```
видаляється.

### 3. Undo / Redo

У редакторах:

- Photoshop
- Word
- VS Code

Кожна дія додається у стек.

### 4. Парсинг дужок

Задача:
```
((()))
```
Потрібно перевірити правильність.

Для цього використовують стек.

### 5. Інтерпретатори мов

JavaScript Engine

Python Interpreter

Компілятори

Активно використовують стек.

## 3.10. Класична задача зі співбесід

Перевірити баланс дужок.

Правильно:
```
()
(())
{[]}
```
Неправильно:
```
((
))
([)]
```
Рішення:

- відкриваючу дужку кладемо в стек;
- закриваюча повинна відповідати вершині стеку.

Тут стек ідеально підходить через принцип LIFO.

## 3.11. Переваги стеку
1. Дуже проста структура.
2. Усі основні операції O(1).
3. Ідеально підходить для вкладених структур.

## 3.12. Недоліки стеку
1. Доступ тільки до вершини.
2. Не можна швидко отримати довільний елемент.
3. Не підходить для пошуку.

## 3.13. Stack vs Array

Стек часто реалізується через масив.

Але:

- Масив дозволяє доступ за індексом.
- Стек навмисно обмежує доступ.

Тобто Stack — це не окрема форма зберігання пам'яті.

Це:
> спосіб використання структури даних.

## 3.14. Найважливіший інсайт

`Стек` — це не просто структура даних.

Це одна з фундаментальних моделей роботи комп'ютера.

На стеку побудовані:

- виклики функцій;
- рекурсія;
- обробка винятків;
- багато алгоритмів обходу графів і дерев.

## Підсумок розділу

Stack працює за принципом:

- LIFO
- Last In, First Out

Основні операції:

| Операція | Складність |
| -------- | ---------- |
| push     | O(1)       |
| pop      | O(1)       |
| peek     | O(1)       |
| isEmpty  | O(1)       |


Стек використовується для:

- рекурсії;
- історії дій;
- навігації;
- парсингу;
- компіляторів.

# РОЗДІЛ 5. РЕКУРСІЯ

## 5.1. Що таке рекурсія

**Рекурсія** — це ситуація, коли функція викликає саму себе.

Формально:

**Рекурсивна функція** — це функція, визначена через саму себе.

## 5.2. Найпростіший приклад
```python
def count_down(n):
    if n == 0:
        return
    print(n)
    count_down(n - 1)
```
Виклик:
```python
count_down(3)
```
Виконання:
```bash
3
2
1
```

## 5.3. Дві обов’язкові частини рекурсії

Будь-яка правильна рекурсія має:

1️⃣ Базовий випадок (base case)

Умова зупинки.
```py
if n == 0:
    return
```
Без нього функція викликатиме себе нескінченно.

2️⃣ Рекурсивний крок

Виклик функції з «меншою» задачею.
```py
count_down(n - 1)
```

## 5.4. Як комп’ютер виконує рекурсію

Це ключове.

Кожен виклик функції:
- створює новий запис у стеку викликів (call stack)
- зберігає локальні змінні
- запам’ятовує, куди повернутися

Приклад:
```py
count_down(3)
```
Стек виглядає так:
```py
count_down(3)
  count_down(2)
    count_down(1)
      count_down(0)
```
Потім функції завершуються у зворотному порядку.

👉 Це принцип `LIFO` (Last In — First Out).

## 5.5. Візуальна модель

Уяви стопку тарілок.

Кожен виклик — нова тарілка.

Щоб дістати першу, треба зняти всі зверху.

Саме так працює стек.

## 5.6. Приклад — факторіал

Математично:
```py
n! = n × (n-1)!
```
Базовий випадок:
```
0! = 1
```
Реалізація:
```py
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
Виклик factorial(4)
4 * factorial(3)
    3 * factorial(2)
        2 * factorial(1)
            1 * factorial(0)
                1
```
Потім стек починає згортатися:
```py
1
1 * 1 = 1
2 * 1 = 2
3 * 2 = 6
4 * 6 = 24
```

## 5.7. Просторова складність рекурсії

Кожен виклик займає місце в стеку.

Для factorial(n):
- глибина = n
- просторова складність = O(n)

Навіть якщо ми не створюємо масивів.

## 5.8. Чому рекурсія може бути небезпечною

Якщо рекурсія дуже глибока:
- стек переповнюється
- виникає Stack Overflow

Наприклад:
```py
def f():
    f()
```

Це аварійне завершення.

## 5.9. Рекурсія vs цикл

Будь-яку рекурсію можна переписати через цикл.

Факторіал через цикл:
```py
def factorial(n):
    result = 1
    for i in range(1, n+1):
        result *= i
    return result
```

**Порівняння**
| Рекурсія | 	Цикл |
| -------- | ------ |
| Часто простіша логіка |	Часто ефективніша по памʼяті |
| Використовує стек |	O(1) памʼяті |
| Природна для дерев і графів	| Зручна для лінійних задач |

## 5.10. Коли рекурсія природна

Рекурсія ідеально підходить для:
- дерев
- графів
- задач «розділяй і володарюй»
- задач, що мають самоподібну структуру

Наприклад:
- обхід папок
- обхід DOM
- DFS
- QuickSort
- MergeSort

5.11. Мислення рекурсією

Головна помилка новачків:

> намагатися відстежити всі виклики одразу

Правильний підхід:

1. Довіритися, що менша задача працює.
2. Сконцентруватися на:
   - базовому випадку
   - правильному рекурсивному кроці

## 5.12. Ментальна модель

Рекурсія — це:

> Розв’яжи задачу через меншу версію тієї ж задачі.

Якщо задача не зменшується — рекурсія неправильна.

## 5.13. Типові помилки
1. Немає базового випадку.
2. Задача не зменшується.
3. Неправильне повернення значення.
4. Забули return перед рекурсивним викликом.
5. Неправильна умова зупинки.

## 5.14. Глибший приклад — сума масиву
```py
def sum_array(arr):
    if len(arr) == 0:
        return 0
    return arr[0] + sum_array(arr[1:])
```
Тут кожен виклик:
- створює новий підмасив
- використовує додаткову памʼять

Час: O(n)  
Памʼять: O(n)

Через:
- стек
- створення нових списків

## 5.15. Хвостова рекурсія (Tail Recursion)

Рекурсія називається хвостовою, якщо:
- рекурсивний виклик — остання операція

Наприклад:
```py
def f(n, acc=1):
    if n == 0:
        return acc
    return f(n-1, acc*n)
```
У деяких мовах це оптимізується до циклу.

У Python — ні.

## 5.16. Найглибший інсайт

Рекурсія — це не про самовиклик.

Це про структуру задачі.

Якщо задача природно розбивається на:
- менші копії самої себе
- рекурсія буде чистим і красивим рішенням.

## Підсумок розділу

Рекурсія:
- вимагає базового випадку
- використовує стек викликів
- має просторову складність через стек
- природна для дерев і самоподібних структур
- може бути переписана в цикл

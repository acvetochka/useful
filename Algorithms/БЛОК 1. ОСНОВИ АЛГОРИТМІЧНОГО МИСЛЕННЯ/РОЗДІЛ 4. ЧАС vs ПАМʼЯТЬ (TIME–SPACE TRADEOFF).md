# РОЗДІЛ 4. ЧАС vs ПАМʼЯТЬ (TIME–SPACE TRADEOFF)

## 4.1. Дві головні характеристики алгоритму

Будь-який алгоритм оцінюють за двома параметрами:

1. Часова складність (Time Complexity)
   
    Скільки операцій виконується (Big O).

2. Просторова складність (Space Complexity)
   
    Скільки пам’яті використовується.

Раніше ми розглядали тільки час.
Тепер додаємо другий вимір — памʼять.

## 4.2. Що таке просторова складність

**Просторова складність** — це залежність кількості додаткової памʼяті від розміру вхідних даних.

Позначення таке ж:
```
O(1), O(n), O(n²)
```
але тепер це означає обсяг пам’яті, а не кількість операцій.

## 4.3. Приклад 1 — мінімальне використання памʼяті

Задача: знайти суму елементів масиву.
```python
def sum_array(arr):
    total = 0
    for x in arr:
        total += x
    return total
```
Що ми використовуємо?
- одну змінну total
- змінну циклу x

Незалежно від розміру масиву — памʼять не зростає.

Просторова складність:
```
O(1)
```

## 4.4. Приклад 2 — створення нового масиву
```python
def double_array(arr):
    result = []
    for x in arr:
        result.append(x * 2)
    return result
```
Тут:
- створюється новий масив розміром n

Просторова складність:
```
O(n)
```
Бо памʼять зростає разом із кількістю елементів.

## 4.5. Головна ідея tradeoff

У програмуванні часто існує вибір:

- зробити швидше, але використати більше памʼяті

    або
- використати менше памʼяті, але працювати повільніше

Це називається:
```
Time–Space Tradeoff
```

## 4.6. Класичний приклад — пошук елемента
- Варіант 1: Лінійний пошук
    ```python
    for x in arr:
        if x == target:
            return True
    ```
    Час:
    ```
    O(n)
    ```
    Памʼять:
    ```
    O(1)
    ```
- Варіант 2: Використати множину (Set)
  ```python
  s = set(arr)
  return target in s
  ```

    Тепер:
    - створюємо структуру даних
    - використовуємо додаткову памʼять

    Час пошуку:
    ```
    O(1)
    ```
    Але памʼять:
    ```
    O(n)
    ```
    Ми пожертвували памʼяттю заради швидкості.

## 4.7. Що таке додаткова памʼять

Важливо розуміти:
- вхідні дані не рахуються
- рахується лише памʼять, яку алгоритм створює додатково

Наприклад:
```python
def f(arr):
    return arr[0]
```
Навіть якщо масив величезний —
просторова складність:
```
O(1)
```
Бо ми нічого нового не створюємо.

## 4.8. Рекурсія і памʼять

Кожен виклик функції додає запис у стек.
```python
def f(n):
    if n == 0:
        return
    f(n-1)
```
Глибина викликів = n

Просторова складність:
```
O(n)
```
Навіть якщо ми не створюємо масивів.

## 4.9. In-place алгоритми

Алгоритм називається in-place, якщо:
- він змінює дані без створення додаткових структур
- використовує мінімум памʼяті

Наприклад:
- розворот масиву шляхом перестановки елементів
- сортування вставками

Просторова складність:
```
O(1)
```

## 4.10. Коли памʼять стає проблемою

У звичайних програмах це майже непомітно.

Але в:
- Big Data
- обробці відео
- машинному навчанні
- серверах із мільйонами запитів

памʼять — критичний ресурс.

## 4.11. Порівняння підходів
| Підхід	| Час	| Памʼять |
| Лінійний пошук	| O(n)	| O(1)|
| Set / HashMap	| O(1)	| O(n)| 
| Сортування + бінарний пошук	| O(n log n)	| O(1) або O(n)| 

## 4.12. Як мислить досвідчений розробник

Перед вибором рішення ставляться питання:
1. Який розмір даних?
2. Чи обмежена памʼять?
3. Чи критична швидкість?
4. Чи буде операція виконуватися багато разів?

## 4.13. Головний інсайт

Швидкість майже завжди коштує памʼяті.

Більшість "магічно швидких" структур даних:
- використовують хешування
- створюють додаткові таблиці
- кешують результати

Алгоритми оцінюють у двох вимірах:
- час
- памʼять

Ідеального рішення не існує.

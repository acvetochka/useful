# РОЗДІЛ 2. ЯК КОМП’ЮТЕР РЕАЛЬНО ВИКОНУЄ ПРОГРАМУ
## 2.1. Чому це важливо

Більшість проблем з алгоритмами виникає тому, що програміст не розуміє, що відбувається всередині комп’ютера.

Алгоритми оцінюють не абстрактно —
вони оцінюють роботу реального процесора і пам’яті.

Щоб зрозуміти:

- Big O
- швидкість програм
- структури даних

потрібно знати базову модель виконання програми.

## 2.2. Спрощена модель комп’ютера

Будь-який комп’ютер складається з трьох основних частин:

**1. CPU (процесор)**

- Виконує інструкції.
- Він уміє лише дуже прості операції:
  - додати числа
  - порівняти
  - скопіювати значення
  - перейти до іншої інструкції

- 👉 процесор не думає — він виконує команди одну за одною.

**2. RAM (оперативна пам’ять)**

- Місце, де зберігаються дані під час роботи програми.
- Уявляй RAM як довгу стрічку комірок:
  ```
  [0][1][2][3][4][5]...
  ```
- Кожна комірка має адресу.

**3. Storage (диск)**

- Довготривале зберігання:
  - файли
  - база даних
  - операційна система

Алгоритми переважно працюють з RAM, а не з диском.

## 2.3. Як запускається програма

Коли ти запускаєш програму:
- Код читається з диска.
- Завантажується в RAM.
- CPU починає виконувати інструкції по черзі.

## 2.4. Програма = послідовність інструкцій

Розглянемо код:
  ```python
  a = 5
  b = 3
  c = a + b
  ```
Для процесора це:
- записати 5 у памʼять
- записати 3 у памʼять
- взяти значення a
- взяти значення b
- скласти
- записати результат

👉 навіть простий рядок коду = багато операцій.

## 2.5. Найважливіша ідея алгоритмів

Алгоритми вимірюють:
> скільки елементарних операцій виконає комп’ютер.

Саме тому ми рахуємо:
  - кількість кроків
  - кількість проходів циклу
  - кількість порівнянь

## 2.6. Змінні та памʼять

Коли створюється змінна:
```python
x = 10
```
відбувається:

- виділяється місце в RAM;
- значення записується за певною адресою.

Схематично:
```
Адреса     Значення
1001  →    10
```
Змінна — це ім’я для адреси памʼяті.

## 2.7. Що таке доступ до памʼяті

Є дві принципово різні операції:

✅ Швидкий доступ

Отримати значення за відомою адресою.
  ```
  memory[1001]
  ```
Це дуже швидко — майже миттєво.

❌ Повільний пошук

Якщо адреса невідома — потрібно перебирати.

Саме через це:
- масиви швидкі
- списки можуть бути повільніші

(детально буде в структурах даних).

## 2.8. Послідовне виконання

CPU виконує інструкції:
```
1 → 2 → 3 → 4 → 5
```
одну за одною.

Навіть якщо код виглядає складним — всередині це просто послідовність кроків.

## 2.9. Як працює цикл
```python
for i in range(5):
    print(i)
```
Процесор фактично робить:
- i = 0
- перевірка умови
- виконати тіло
- збільшити i
- перейти назад

Це означає:

👉 цикл = повторення тих самих інструкцій.

## 2.10. Чому великі дані роблять програму повільною

Приклад:
```
for item in list:
    ...
```
Якщо список має:

- 10 елементів → 10 виконань
- 1 000 → 1 000 виконань
- 1 000 000 → 1 000 000 виконань

CPU реально виконує більше роботи.

## 2.11. Стек викликів (Call Stack)

Коли викликається функція:
```python
foo()
```
комп’ютер:
- запам’ятовує, звідки викликали;
- створює нову область пам’яті;
- виконує функцію;
- повертається назад.

Це називається стек викликів.
```
main
 └── foo
      └── bar
```
Остання викликана функція завершується першою.

👉 принцип `LIFO` (Last In — First Out).

Це буде критично важливо для рекурсії.

## 2.12. Heap vs Stack (дуже базово)
Stack
- локальні змінні
- виклики функцій
- швидко
- автоматично очищається

Heap
- великі об’єкти
- cкладні структури
- керується програмою / garbage collector

Поки достатньо інтуїції.

## 2.13. Головний висновок розділу

Комп’ютер:
- не розуміє задач;
- не оптимізує алгоритм;
- не думає.

Він лише:

👉 виконує величезну кількість простих операцій.

`Алгоритми` — це спосіб зменшити кількість цих операцій.

## Підсумок

Щоб оцінити алгоритм, потрібно розуміти:
- кожен рядок коду → набір операцій;
- цикли множать кількість операцій;
- доступ до пам’яті має значення;
- функції використовують стек.

# РОЗДІЛ 1. ЩО ТАКЕ АЛГОРИТМ

## 1.1. Визначення алгоритму

`Алгоритм` — це скінченна послідовність чітко визначених дій, яка перетворює вхідні дані на результат.

Інакше кажучи:

- алгоритм — це інструкція розв’язання задачі.

Формально алгоритм має:

- Вхідні дані — що ми отримуємо.

- Кроки виконання — що потрібно зробити.

- Вихід — результат.

- Завершення — алгоритм обов’язково закінчується.

**Приклад**

Задача: знайти найбільше число у списку.

Вхід:
```
[3, 7, 2, 9, 5]
```
Алгоритм:

- Взяти перше число як поточний максимум.

- Порівняти його з наступним.

- Якщо нове число більше — замінити максимум.

- Повторити для всіх елементів.

- Повернути максимум.

Вихід:
```
9
```

## 1.2. Алгоритм ≠ програма

| Поняття	| Що це |
| Алгоритм	| Ідея рішення |
| Програма	| Реалізація алгоритму мовою програмування |

Один алгоритм можна реалізувати:

- Python
- JavaScript
- C++
- навіть на папері.

## 1.3. Основні властивості алгоритму
### 1. Детермінованість

Кожен крок визначений однозначно.
```
❌ "обробити дані якось"
✅ "відсортувати масив за зростанням"
```
### 2. Скінченність

- Алгоритм має завершитися
- Нескінченний цикл без умови виходу — не алгоритм.

### 3. Масовість

Алгоритм працює для класу задач, а не одного випадку.

Не:
```
знайти максимум у [3,7,2]
```
А:
```
знайти максимум у будь-якому списку
```
### 4. Результативність

Після виконання завжди отримується результат.

## 1.4. Подання алгоритмів

Алгоритми можна записувати різними способами:

### 1. Текстовий опис

Послідовність кроків.

### 2. Псевдокод
```python
max = first element
for each number:
    if number > max:
        max = number
return max
```
### 3. Блок-схеми

(використовуються рідше, але важливі концептуально).

## 1.5. Типи алгоритмів (загальне уявлення)

Алгоритми відрізняються способом мислення:

- перебір (brute force)
- пошук
- сортування
- рекурсивні алгоритми
- жадібні алгоритми
- динамічне програмування

Поки це лише орієнтація — детально буде пізніше.

## 1.6. Чому алгоритми важливі

Одна й та сама задача може бути вирішена різними алгоритмами.

Наприклад, пошук елемента:
- перевіряти всі елементи → повільно
- використовувати відсортованість → швидко

Різниця може складати тисячі разів.

Саме тому програмування — це не написання коду, а вибір алгоритму.

## 1.7. Типова помилка початківців

Новачок думає:
```
"Який код написати?"
```
Правильне питання:
```
"Який алгоритм вирішує задачу?"
```

## 1.8. Алгоритмічне мислення

Алгоритмічне мислення — це вміння:
- розбити задачу на кроки;
- описати ці кроки формально;
- оцінити ефективність рішення.

## Підсумок розділу

Алгоритм — це:
- не код;
- не мова програмування;
- а чітка стратегія розв’язання задачі.

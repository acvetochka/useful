# Команда install

## Призначення

`install` — це утиліта Linux для копіювання файлів, створення файлів і каталогів з одночасним встановленням прав доступу, власника та групи.

Команда часто використовується:
- у Makefile
- у скриптах інсталяції програм
- системними адміністраторами для безпечного створення конфігураційних файлів

Основна перевага install — можливість однією командою виконати дії, які зазвичай роблять кількома командами:
```bash
cp
chmod
chown
mkdir
```

## Синтаксис
Копіювання файлу
```bash
install [OPTIONS] SOURCE DESTINATION
```
Копіювання кількох файлів
```bash
install [OPTIONS] SOURCE... DIRECTORY
```
Створення каталогу
```bash
install -d [OPTIONS] DIRECTORY
```

## Основні параметри
| параметр     | значення                       |
| ------------ | ------------------------------ |
| -m MODE      | встановити права доступу       |
| -o OWNER     | встановити власника            |
| -g GROUP     | встановити групу               |
| -d           | створити каталог               |
| -t DIRECTORY | вказати директорію призначення |
| -v           | verbose режим                  |
| -p           | зберегти timestamps            |

### Параметр -m (mode)

  Дозволяє встановити права доступу для створюваного файлу або каталогу.

  Формат:
  ```bash
  install -m MODE SOURCE DEST
  ```
  Приклад
  ```bash
  install -m 600 config.txt /tmp/config.txt
  ```
  Результат:
  > `/tmp/config.txt` отримає права `rw-------`

  **Чим це відрізняється від cp**

  Звичайне копіювання:
  ```bash
  cp config.txt /tmp/config.txt
  chmod 600 /tmp/config.txt
  ```
  install робить це однією командою.

### Параметри -o і -g

Дозволяють одразу встановити власника і групу.

Приклад
```bash
install -m 644 -o root -g root config.conf /etc/config.conf
```
Результат:
```
owner = root
group = root
permissions = 644
```

### Створення файлів (використання `/dev/null`)

`install` може створювати нові файли, якщо використовувати `/dev/null` як джерело.

Приклад
```bash
install -m 600 /dev/null secret.txt
```
Що відбувається
- копіюється `/dev/null`
- створюється новий файл
- встановлюються права

Результат:
```
-rw------- secret.txt
```
***

**Чому використовують /dev/null**

`/dev/null` — спеціальний файл, який не містить даних.

Копіювання /dev/null фактично створює порожній файл.

***
**Чому не використовують touch**

Команда
```bash
touch file.txt
```
створює файл з правами:
> default permissions - umask

Наприклад:
> 644

Тоді потрібно додатково виконувати:
```bash
chmod 600 file.txt
```
install дозволяє створити файл одразу з правильними правами.

### Створення каталогів (`-d`)

Для створення каталогів використовується параметр:
`-d`

Приклад
```bash
install -d -m 700 ~/.ssh
```
Це еквівалент:
```bash
mkdir ~/.ssh
chmod 700 ~/.ssh
```

Приклад з власником
```bash
install -d -m 750 -o www-data -g www-data /var/www/site
```
Еквівалент:
```bash
mkdir /var/www/site
chmod 750 /var/www/site
chown www-data:www-data /var/www/site
```

### Параметр `-t`

Дозволяє вказати директорію призначення.

Приклад
```bash
install -m 755 script.sh -t /usr/local/bin
```
Файл буде скопійований у:
> /usr/local/bin/script.sh

### Параметр `-v`

Verbose режим — показує що робить команда.
```bash
install -v -m 755 script.sh /usr/local/bin/script
```

### Параметр -p

Зберігає часи файлу:
- atime
- mtime
  
## Практичні сценарії використання
### 1. Створення `authorized_keys`

SSH вимагає права:
> 600

Команда:
```bash
install -m 600 /dev/null ~/.ssh/authorized_keys
```

### 2. Створення каталогу `.ssh`

SSH вимагає права:
> 700

```bash
install -d -m 700 ~/.ssh
```

### 3. Встановлення скрипта у систему
```bash
install -m 755 backup.sh /usr/local/bin/backup
```
Це:
- копіює файл
- встановлює execute права

### 4. Встановлення конфігураційного файлу
```bash
install -m 644 nginx.conf /etc/nginx/nginx.conf
```

### 5. Створення каталогу веб-сайту
```bash
install -d -m 750 -o www-data -g www-data /var/www/site
```

### 6. Створення файлу секретів
```bash
install -m 600 /dev/null secrets.env
```

## Використання у Makefile

Команда `install` широко використовується у процесі встановлення програм.

Приклад:
```bash
install -m 755 program /usr/local/bin/program
```

Різниця між install і cp
| команда       | що робить                     |
| ------------- | ----------------------------- |
| cp            | копіює файл                   |
| cp --preserve | копіює з атрибутами           |
| install       | копіює і задає права/власника |

Порівняння з іншими командами
| команда | призначення                               |
| ------- | ----------------------------------------- |
| cp      | копіювання                                |
| chmod   | зміна прав                                |
| chown   | зміна власника                            |
| mkdir   | створення каталогу                        |
| install | копіювання + встановлення прав і власника |

## Особливість команди

install часто вважається безпечнішим способом встановлення системних файлів, тому що:
- права доступу задаються одразу
- власник і група встановлюються одразу
- не потрібно виконувати кілька команд

Це зменшує ризик помилок у скриптах.

Типові приклади
| команда                                        | результат                   |
| ---------------------------------------------- | --------------------------- |
| install -m 600 /dev/null file                  | створити файл з правами 600 |
| install -m 755 script.sh /usr/local/bin/script | встановити програму         |
| install -d -m 700 ~/.ssh                       | створити каталог            |
| install -m 644 config.conf /etc/app.conf       | встановити конфігурацію     |

## Підсумок

Команда install дозволяє:
- копіювати файли
- створювати файли
- створювати каталоги
- встановлювати права доступу
- встановлювати власника і групу

Все це можна зробити однією командою, що робить її зручною для автоматизації та інсталяційних скриптів.

# ls

системна довідка по `ls`: що саме ця команда вміє показувати, які ключі найкорисніші, як комбінувати їх між собою, і що означає кожна колонка в довгому форматі.

ls здається простою командою, але насправді в неї багато режимів. Важливо одразу розділити:

- ls показує записи директорії
- частину інформації він бере із самої директорії
- частину — через читання метаданих inode

Тому `ls` — це по суті інструмент для перегляду вмісту директорії та метаданих файлів.

## 1. Базова ідея ls

Найпростіший виклик:
```bash
ls
```

Покаже імена файлів і директорій у поточній директорії.

Наприклад:
```bash
file.txt  photo.jpg  scripts  notes.md
```
За замовчуванням:
- приховані файли не показуються
- інформація виводиться коротко
- сортування йде за іменем

## 2. Найуживаніші ключі

```bash
ls -ialh
```
Це дуже хороший набір, але не весь. Ось логіка основних опцій.

- `-l` — long format

Показує детальну інформацію:
```bash
ls -l
```

- `-a` — all

Показує всі файли, включно з прихованими:
```bash
ls -a
```

Сюди входять:  
 `.bashrc`  
 `.gitignore`  
 `.`  
 `..`  
***


- `-A` — almost all

Схоже на `-a`, але без `.` і `..`
```bash
ls -A
```
Це часто зручніше за `-a`.
***

-`-i` — inode

Показує номер inode:
```bash
ls -i
```
***

- `-h` — human readable

Працює разом із `-l` і показує розміри у зручному вигляді:
```bash
ls -lh
```
Наприклад:
```
4.0K

12M

1.1G
```

## 3. Що ще дуже корисно знати
- `-R` — recursive

Рекурсивно проходить по всіх піддиректоріях:
```bash
ls -R
```

Це корисно, якщо треба побачити дерево вмісту.
***

- `-d` — show directory itself

Без цього ключа ls dir показує вміст директорії.
З цим ключем — саму директорію як об’єкт.
```bash
ls -ld mydir
```

Це дуже важливо для перегляду прав доступу саме на директорію.
***

- `-F` — classify entries

Додає символ типу файлу в кінці імені:

  ```bash
  ls -F
  ```
  Наприклад:  
   `/` — директорія  
   `*` — виконуваний файл  
   `@` — symlink  
   `|` — pipe
***

- `-p`

Додає / лише для директорій:
```bash
ls -p
```
***

- `-Q`

Бере імена файлів у лапки:
```bash
ls -Q
```
Зручно для імен із пробілами.
***

- `-1`

Один файл на рядок:
```bash
ls -1
```
***

- `-m`

Виводить список через кому:
```bash
ls -m
```
***

- `-C`

Вивід у кілька колонок:
```bash
ls -C
```
Часто це поведінка за замовчуванням у терміналі.
***

- `-x`

Сортування по рядках, а не по колонках:
```bash
ls -x
```
***

- `-S` — sort by size

Сортування за розміром, від більшого до меншого:
```bash
ls -lS
```
***

- `-t` — sort by time

Сортування за часом зміни:
```bash
ls -lt
```
Найновіші зверху.
***

- `-r` — reverse

Обертає порядок сортування:
```bash
ls -ltr
```
Тоді найстаріші будуть зверху.
***

- `-U`

Не сортувати взагалі, показати як є в директорії:
```bash
ls -U
```

Цікаво для розуміння, що ls зазвичай не показує фізичний порядок записів, а сортує їх.
***

- `-X`

Сортування за розширенням:
```bash
ls -X
```
***

- `-v`

Natural sort, тобто file2 піде перед file10:
```bash
ls -v
```
***

- `-c`

Показувати час зміни inode (ctime) замість mtime:
```bash
ls -lc
```
***

- `-u`

Показувати час останнього доступу (atime):
```bash
ls -lu
```
***

- `--time=...`

Дозволяє явно вибрати час:
```bash

ls -l --time=atime
ls -l --time=ctime
ls -l --time=mtime
ls -l --time=birth
```
`birth` працює не всюди, залежить від файлової системи та реалізації.
***

- `--full-time`


Показує повну дату і час, аж до секунд і часто наносекунд:
```bash
ls -l --full-time
```
***

- `-n`

Замість імен користувача і групи показує UID і GID:
```bash
ls -ln
```
***

- `-g`

Не показувати власника:
```bash
ls -lg
```
***

- `-o`

Не показувати групу:
```bash
ls -lo
```
***

- `-s`

Показує кількість зайнятих блоків:
```bash
ls -ls
```
Не плутати з реальним розміром файлу.

***

- `-k`

Показувати розмір блоків у KB:
```bash
ls -lk
```
***

- `--block-size=SIZE`

Можна задати одиницю:
```
ls -l --block-size=M
ls -l --block-size=K
```
***

- `--color=auto`

Кольоровий вивід:
```bash
ls --color=auto
```
Часто вже увімкнено через alias.
***

- `-L`

Якщо файл — symbolic link, показати інформацію про ціль, а не про сам symlink:
```bash
ls -lL link.txt
```
***

- `-H`

Схоже на -L, але тільки для symlink, переданих у командному рядку.
***

- `-P`

Не переходити за symbolic links. Це стандартна поведінка, але іноді корисно знати.
***

- `-Z`
Показати security context, наприклад SELinux:
```bash
ls -lZ
```
Працює не на всіх системах.
***

## 4. Практичні комбінації

Ось реально корисні варіанти.

Показати все, включно з inode і зрозумілими розмірами
```bash
ls -ialh
```
Показати права саме на директорію, а не її вміст
```bash
ls -ld /var/log
```
Відсортувати за часом
```bash
ls -lt
```
Від найстаріших до найновіших
```bash
ls -ltr
```
Від найбільших до найменших
```bash
ls -lhS
```
Показати всі файли, крім . і ..
```bash
ls -Al
```
Показати повний час
```bash
ls -l --full-time
```
Показати UID/GID замість імен
```bash
ls -ln
```
Подивитися symlink і його ціль
```bash
ls -l link.txt
```
## 5. Розбір колонок ls -l

Візьмемо приклад:
```bash
-rw-r--r-- 1 user staff 1200 Mar 10 14:35 file.txt
```
Або з inode:
```bash
12345 -rw-r--r-- 1 user staff 1200 Mar 10 14:35 file.txt
```
Розберемо все по частинах.

**Колонка 1: inode, якщо є -i**
```
12345
```
Це номер inode.

Якщо два файли мають один і той самий inode — це hard links на той самий файл.
***

**Колонка 2: тип файлу + права**
```
-rw-r--r--
```
Перший символ — тип файлу:
- `-` — звичайний файл
- `d` — директорія
- `l` — symbolic link
- `c` — character device
- `b` — block device
- `p` — named pipe
- `s` — socket

Далі йдуть 9 символів прав:
```
rw- r-- r--
```
Це три трійки:
- owner
- group
- others

Розшифрування:
- `r` — read
- `w` — write
- `x` — execute
- `-` — немає права

Приклад:
```bash
-rwxr-x---
```
означає:
- власник: читати, писати, виконувати
- група: читати, виконувати
- інші: нічого

**Спеціальні біти**

Іноді в правах можна побачити:
- s
- S
- t
- T

Наприклад:
```bash
-rwsr-xr-x
drwxrwxrwt
```
`s` / `S`
- setuid
- setgid

`t` / `T`
- sticky bit

Наприклад `/tmp` часто має:
```bash
drwxrwxrwt
```
Це означає, що навіть якщо директорія доступна на запис усім, видаляти файли може тільки власник файлу або root.
***

**Колонка 3: link count**
```
1
```
Це кількість жорстких посилань на inode.

Для звичайного файлу:
- 1 — одне ім’я
- 2, 3, ... — є hard links

Для директорій число зазвичай більше через:
- `.`
- `..`
- піддиректорії
***

**Колонка 4: owner**
user

Власник файлу.
***

**Колонка 5: group**
```
staff
```
Група файлу.
***

**Колонка 6: size**
```
1200
```
Розмір файлу в байтах.

Для директорії це не “розмір усіх файлів усередині”, а розмір самого файлу директорії, тобто простору, зайнятого її записами.

Для symlink це часто довжина шляху, який він містить.
***

**Колонка 7–9: дата і час**
```
Mar 10 14:35
```
За замовчуванням це `mtime` — час останньої зміни вмісту файлу.

Якщо файл старий, ls може показати рік замість часу:
```
Mar 10 2024
```
***

**Остання колонка: ім’я**
```
file.txt
```
Для symbolic link буде так:
```bash
link.txt -> target.txt
```
Тобто ls показує і symlink, і куди він вказує.

## 6. Що ще може показувати ls
Блоки через `-s`

Приклад:
```bash
ls -ls
```
Може дати щось таке:
```bash
4 -rw-r--r-- 1 user staff 1200 Mar 10 14:35 file.txt
```
Перший `4` тут — не inode, а кількість дискових блоків, зайнятих файлом.

Це відрізняється від розміру в байтах, бо файлова система працює блоками.
***

**Security context через -Z**

На системах із `SELinux`:
```bash
ls -lZ
```
Додається ще одна колонка з контекстом безпеки.

## 7. Як побачити “майже все”, що ls взагалі вміє дати

Найбільш інформативні варіанти:
```bash
ls -laih
```
або
```bash
ls -lAih --full-time
```
або ще глибше:
```bash
ls -lAihs --full-time
```
Це покаже:
- inode
- права
- кількість hard links
- власника
- групу
- розмір
- блоки
- повний час
- приховані файли

Але `ls` все одно не покаже абсолютно всі метадані файла. Для цього вже потрібен `stat`.

## 8. Межа можливостей ls

ls дуже корисний, але не всесильний. Він не показує зручно:
- повний набір часових полів
- деталі inode
- номер пристрою
- IO block size
- birth time у всіх випадках
- ACL детально
- xattrs детально

Для цього краще використовувати:
```
stat file
getfacl file
lsattr file
file file
```

Тобто:
- `ls` — для огляду
- `stat` — для точних метаданих

## 12. Готовий розбір колонок

Колонки `ls -l`

Приклад:

```bash
-rw-r--r-- 1 user staff 1200 Mar 10 14:35 file.txt
```

Пояснення:
1. `-rw-r--r--` — тип файлу і права доступу
2. `1` — кількість жорстких посилань
3. `user` — власник
4. `staff` — група
5. `1200` — розмір у байтах
6. `Mar 10 14:35` — час останньої модифікації
7. `file.txt` — ім’я файлу

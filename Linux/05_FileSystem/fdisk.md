# fdisk

`fdisk` — це потужна утиліта командного рядка в Linux для керування таблицями розділів жорстких дисків (створення, видалення, зміна розміру). Вона працює з дисками як GPT, так і MBR. 

> [!WARNING] 
> Робота з fdisk вимагає прав суперкористувача (`sudo`) і може призвести до втрати даних при неправильному використанні.

## 1. Перегляд існуючих розділів
Перед внесенням змін перегляньте поточну структуру диска:
  ```bash
  sudo fdisk -l
  ```
  Ця команда виведе список усіх дисків (наприклад, /dev/sda, /dev/nvme0n1) та їхніх розділів. 

## 2. Запуск fdisk для редагування
Щоб почати роботу з конкретним диском, запустіть fdisk із вказанням цього диска:
```bash
sudo fdisk /dev/sdb
```
(Замініть `/dev/sdb` на ваш диск). Ви потрапите в інтерактивний режим.

## 3. Інтерактивний режим fdisk (основні команди)
Усередині програми (після запуску) використовуйте односимвольні команди. Щоб побачити список усіх команд, натисніть m: 

- `p` — надрукувати таблицю розділів (показати поточний стан).
- `n` — створити новий розділ.
- `d` — видалити розділ.
- `t` — змінити тип файлової системи розділу (наприклад, Linux, swap, NTFS).
- `l` — список відомих типів розділів.
- `L` — переглянути список кодів доступних типів розділів.
- `w` — записати зміни на диск і вийти (важливо!).
- `q` — вийти БЕЗ збереження змін. 

## 4. Покрокова інструкція: Створення нового розділу

1. Запустіть: 
   ```bash
   sudo fdisk /dev/sdb
   ```
2. Створіть розділ: 
   > Натисніть n.
3. Виберіть тип: 
   - `p` (primary) або 
   - `e` (extended). 
  Зазвичай вибирають `p`.
4. Номер розділу: 
   > Натисніть Enter (за замовчуванням).
5. Перший сектор: 
   > Натисніть Enter (за замовчуванням).
6. Останній сектор (розмір):   
   Вкажіть розмір. Приклади:
   - +5G — для створення розділу на 5 ГБ.
   - Enter — щоб використати весь вільний простір.
6. Запишіть зміни: 
   > Натисніть w. 

## 5. Покрокова інструкція: Видалення розділу
1. Запустіть: 
   ```bash
   sudo fdisk /dev/sdb
   ```
2. Видаліть: 
  > Натисніть d.
3. Номер розділу: 
   > Введіть номер розділу, який треба видалити (подивіться через p, якщо не знаєте).
4. Запишіть зміни: 
   > Натисніть w. 

## 6. Після роботи з fdisk
Після виходу з fdisk (w), система може не відразу побачити зміни.
Оновіть таблицю розділів:
```bash
sudo partprobe /dev/sdb
```
Відформатуйте новий розділ (наприклад, в `ext4`):
```bash
sudo mkfs.ext4 /dev/sdb1
```
(Замініть /dev/sdb1 на ваш новий розділ).

## 7. Створення файлової системи (Форматування)
fdisk лише створює розділи, але не форматує їх. Використовуйте mkfs після створення:
- EXT4: `sudo mkfs.ext4 /dev/sdb1`
- FAT32: `sudo mkfs.vfat /dev/sdb1`
- NTFS: `sudo mkfs.ntfs /dev/sdb1 `

## Поради
`sudo fdisk -l` — найкращий спосіб побачити правильні назви ваших дисків, щоб не видалити системний розділ.
Якщо ви помилилися в інтерактивному режимі, натисніть q, щоб вийти без збереження, і почніть заново

## Зміна типу таблиці розділів

У fdisk команди g і o створюють нову таблицю розділів, але різного типу.

g vs o у fdisk
| команда | 	що створює |
| ------ | ------ |
| g	| GPT (GUID Partition Table)|
| o	| MBR (DOS partition table)|

Тобто:
```
g → GPT
o → MBR
```

**Що реально відбувається**

Коли ти запускаєш:
```
sudo fdisk /dev/sda
```
і вводиш:
```
g
```
fdisk створює нову GPT таблицю розділів.

Якщо вводиш:
```
o
```
створюється класична DOS/MBR таблиця.

Важливий момент

Ці команди:
- видаляють існуючу таблицю розділів
- створюють нову

Але зміни застосовуються тільки після `w`.

Тобто поки ти не введеш:
```
w
```
нічого на диску не зміниться.

Якщо передумаєш:
```
q
```
вийде без змін.

**Як виглядає процес створення GPT**
```bash
sudo fdisk /dev/sda
```
потім:
```
g
```
виведе щось типу:
```
Created a new GPT disklabel
```
далі створюєш розділ:
```
n
```
після цього:
```
w
```
***

**Як виглядає процес створення MBR**
```bash
sudo fdisk /dev/sda
```
потім:
```
o
```
Created a new DOS disklabel

далі:
```
n
```
і:
```
w
```
***

**Як перевірити тип таблиці**
```bash
sudo fdisk -l /dev/sda
```
буде показано:
```
Disklabel type: gpt
```
або
```
Disklabel type: dos
```
***

**Чому dos**

У Linux MBR часто називається dos, бо це історична сумісність з MS-DOS.

Тобто:

> dos = MBR

***
**Коли використовувати GPT**

GPT використовується у більшості сучасних систем, бо:
- підтримує диски >2TB
- може мати багато розділів
- має backup таблицю
- має CRC перевірку

Коли використовується MBR

MBR іноді використовують:
- для дуже старих BIOS систем
- для сумісності
- для маленьких дисків

**Маленький лайфхак**

Якщо хочеш побачити тип таблиці дуже швидко:
```bash
lsblk -o NAME,SIZE,TYPE,PTTYPE
```
приклад:
```
NAME   SIZE TYPE PTTYPE
sda    500G disk gpt
```
або
```
sda    500G disk dos
```
💡 До речі, у fdisk є ще одна команда, яка часто викликає питання:

- `t`

Вона змінює тип розділу (partition type).

Наприклад:
```
EFI System
Linux filesystem
Linux swap
```

> [!WARNING]
> Застереження: Будьте вкрай обережні, оскільки видалення розділу або запис нової таблиці (w) призведе до втрати даних на відповідних ділянках диска. Для дисків з розміткою GPT часто рекомендують використовувати сучаснішу альтернативу — gdisk

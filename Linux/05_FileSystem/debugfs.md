# Debugfs 

`Debugfs` — це віртуальна файлова система в ядрі Linux, розроблена спеціально для спрощення налагодження (debugging) та відлагодження. Вона базується в оперативній пам'яті (RAM) і дозволяє розробникам ядра експортувати інформацію в користувацький простір. 

> [!IMPORTANT]
> `debugfs` працює тільки з файловими системами сімейства `ext` (ext2 / ext3 / ext4), тому що це утиліта з пакету `e2fsprogs`, який реалізує інструменти саме для цієї файлової системи.  
> 
> **Причина проста:** структура inode, директорій і метаданих різна у різних файлових системах, тому універсальної утиліти, яка працює з усіма FS, фактично не існує.

## Основні характеристики debugfs
**Призначення:** Надання інформації про роботу драйверів та підсистем ядра.

**Розташування:** Зазвичай монтується в `/sys/kernel/debug`.  

**Відсутність правил:** На відміну від `/proc` або `/sys`, у debugfs немає жорстких правил щодо формату даних, що дозволяє розробникам розміщувати будь-яку інформацію.

**Інструмент:** Часто використовується замість printk для легшого ввімкнення/вимкнення налагодження. 

## Як працювати з debugfs (команди)
Щоб використовувати debugfs, ядро має бути зібране з підтримкою `CONFIG_DEBUG_FS`. 

Монтування (якщо не змонтовано автоматично):
```bash
mount -t debugfs none /sys/kernel/debug
```
Перегляд вмісту:
```bash
ls /sys/kernel/debug
```
Перегляд інформації про файлову систему (e2fsprogs):
```bash
debugfs -R 'stat <inode>' /dev/sdXY  # Показує атрибути файлу
```
Відновлення видалених файлів:
```bash
debugfs -w /dev/sdXY
debugfs: undelete <inode> <new_name> #
```

## Різниця між debugfs та debugfs(8)
Важливо розрізняти два поняття:
- Файлова система /sys/kernel/debug — механізм ядра для відлагодження.
- Утиліта debugfs(8) — інтерактивний налагоджувач для файлових систем ext2, ext3, ext4, який використовується для відновлення даних або перегляду структури диска. 

## Доступ до debugfs
За замовчуванням доступ до `/sys/kernel/debug` має лише користувач `root`. Щоб дозволити доступ звичайному користувачеві, потрібно перемонтувати його з параметром `uid`.

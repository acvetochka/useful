# Встановлення програмного забезпечення

Встановлення програмного забезпечення в ОС Linux суттєво відрізняється від Windows і базується на використанні централізованих репозиторіїв (сховищ) та менеджерів пакетів. 

## 1. Встановлення пакетів (RPM)
`RPM (Red Hat Package Manager)` — це формат пакетів для дистрибутивів сімейства Red Hat (Fedora, CentOS, RHEL, openSUSE). Пакет .rpm — це архів, що містить скомпільовані файли програми та інструкції щодо їх розміщення.

- Встановлення: `sudo rpm -ivh package.rpm` (-i — install, -v — verbose, -h — hash/progress).
- Оновлення: `sudo rpm -Uvh package.rpm`
- Видалення: `sudo rpm -e package_name`.
- Перевірка списку файлів у пакеті: `rpm -ql package_name`.
- Інформація про пакет: `rpm -qi package_name`.
- Головний мінус: RPM сам по собі не вміє автоматично завантажувати залежності з інтернету.

## 2. Встановлення з вихідного коду (Source Code)
Цей метод є універсальним, але складним, оскільки потребує ручного керування залежностями та компіляції.

**Алгоритм дій:**
- Розпакування: `tar -xvf archive.tar.gz`.
- Конфігурація: Перехід у папку та запуск `./configure`. Скрипт перевіряє наявність необхідних бібліотек.
- Компіляція: Команда `make`. Перетворює код на виконувані файли (використовує файл `Makefile`).
- Інсталяція: `sudo make install`. Копіює файли у системні директорії (зазвичай `/usr/local/bin`).
- Видалення: Якщо розробник передбачив це, використовується sudo make uninstall (потрібно зберігати папку з кодом).

## 3. Робота з менеджерами пакетів
Менеджери пакетів — це надбудови над низькорівневими інструментами (як RPM або DPKG), які автоматично вирішують залежності та працюють з репозиторіями.

- APT (Debian, Ubuntu, Linux Mint)
  Використовує формат .deb. Офіційна документація доступна на Debian Wiki.
  
  | Дія	| Команда |
  | -----| ------|
  | Встановлення	| sudo apt install name |
  | Видалення	| sudo apt remove name (залишає конфіги) / purge (повне) |
  | Оновлення списку репозиторіїв	| sudo apt update |
  | Оновлення всіх програм	| sudo apt upgrade |
  | Пошук пакетів	| apt search keyword |
  | Інформація про пакет	| apt show name |
  | Перевірка файлів у пакеті	| dpkg -L name (використовує інструмент dpkg) |
  | Список репозиторіїв	| cat /etc/apt/sources.list та папка sources.list.d/ |
  | Очищення кешу (.deb файлів)	| sudo apt clean |

- DNF / YUM (Fedora, RHEL, CentOS)

  Сучасний стандарт для RPM-систем. Деталі можна знайти в документації Fedora.
  
  | Дія	| Команда|
  | ----| ------| 
  | Встановлення	| sudo dnf install name |
  | Видалення	| sudo dnf remove name |
  | Оновлення системи	| sudo dnf upgrade |
  | Пошук	| dnf search keyword |
  | Інформація про пакет	|  dnf info name |
  | Перевірка файлів у пакеті |	dnf repoquery -l name |
  | Перевірка репозиторіїв	| dnf repolist |
  | Очищення кешу |	sudo dnf clean all |

- Pacman (Arch Linux, Manjaro)

  Відомий своєю швидкістю та лаконічністю. Детальна інструкція на Arch Wiki.
  | Дія	| Команда|
  | ----| ------| 
  | Встановлення	|sudo pacman -S name|
  |Видалення	|sudo pacman -Rs name (-s видаляє непотрібні залежності)|
  |Оновлення (синхронізація)	|sudo pacman -Syu (оновлення бази + системи)|
  |Пошук	|pacman -Ss keyword|
  |Інформація про встановлене	|pacman -Qi name|
  |Список файлів у пакеті	|pacman -Ql name|
  |Перевірка репозиторіїв	|Перегляд файлу /etc/pacman.conf|
  |Очищення кешу	|sudo pacman -Sc|

- Zypper (openSUSE)
  
  Потужний інструмент з інтелектуальним вирішенням конфліктів. Документація на openSUSE Wiki.
  | Дія	| Команда|
  | ----| ------| 
  | Встановлення	| sudo zypper install name|
  | Видалення|	sudo zypper remove name|
  | Оновлення	sudo| zypper update|
  | Пошук|	zypper search keyword|
  | Список репозиторіїв|	zypper repos (або lr)|
  | Інфо про пакет|	zypper info name|
  | Очищення кешу	| sudo zypper clean|

- Apk (Alpine Linux)
  
  Команда apk (Alpine Package Keeper) — це менеджер пакетів, що використовується в дистрибутиві Alpine Linux. 

  Ця ОС надзвичайно популярна у сфері Docker-контейнерів завдяки своєму мінімальному розміру (базовий образ важить лише ~5 МБ), тому ви найчастіше зустрінете apk саме в Docker-файлах. 
  
  **Основні команди apk**
  
  Менеджер apk вирізняється лаконічністю синтаксису (замість install використовується add, замість remove — del). 
  
  | Дія |	Команда |
  | ----| -------|
  | Встановлення	| apk add назва_пакета |
  | Видалення	| apk del назва_пакета |
  | Оновлення списку репозиторіїв	| apk update|
  | Оновлення всіх пакетів	| apk upgrade|
  | Пошук пакетів	| apk search назва (або -v для опису)|
  | Інформація про пакет |	apk info назва (або -a для повної інфо) |
  | Перевірка файлів у пакеті	| apk info -L назва|
  | Список репозиторіїв	| cat /etc/apk/repositories |
  | Очищення кешу |	rm -rf /var/cache/apk/* (у apk немає окремої команди clean)|
  
  **Важливі особливості:**
  
  - Формат файлів: Пакети мають розширення `.apk`, але це не додатки для Android (хоча розширення однакове).
  - Швидкість: apk вважається одним із найшвидших менеджерів пакетів у світі Linux.
  - Світ (world): В Alpine є файл `/etc/apk/world`, де перелічені всі явно встановлені вами пакети. Команда `apk fix` допомагає переінсталювати або виправити пакети згідно з цим списком.
  - Локальне встановлення: Щоб встановити завантажений файл без перевірки підпису, використовуйте `apk add --allow-untrusted /шлях/до/файла.apk`.

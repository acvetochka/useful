# date

Команда date у Linux використовується для відображення або встановлення системного дати та часу. Вона є частиною пакета coreutils і підтримує безліч ключів для маніпуляцій з часовими проміжками.

Ось повна довідка по основних ключах та сценаріях використання.

## 1. Основні ключі (Options)
| Ключ                   | Опис                                                                     |
| ---------------------- | ------------------------------------------------------------------------ |
| -d, --date=STRING      | Відображає час, описаний рядком (наприклад, "yesterday"), а не поточний. |
| -s, --set=STRING       | Встановлює системний час (потребує прав root).                           |
| -u, --utc, --universal | Виводить або встановлює час у форматі UTC (Гринвіч).                     |
| -R, --rfc-email        | Виводить дату у форматі RFC 5322 (зручно для заголовків листів).         |
| -I[SPEC], --iso-8601   | Виводить дату у форматі ISO 8601.                                        |
| -r, --reference=FILE   | Виводить час останньої зміни вказаного файлу.                            |
| -f, --file=DATEFILE    | Читає рядки з файлу та обробляє кожен як окрему дату.                    |

## 2. Приклади використання -d (Магія відносної дати)
Це найкорисніша функція для скриптів та автоматизації.

Минуле:
- `date -d "yesterday"` — вчорашній день.
- `date -d "10 days ago"` — 10 днів тому.
- `date -d "2 weeks ago"` — 2 тижні тому.
- `date -d "last friday"` — дата минулої п'ятниці.
Майбутнє:
- `date -d "tomorrow"` — завтра.
- `date -d "next month"` — через місяць.
- `date -d "+90 days"` — через 90 днів.
Конкретні моменти:
- `date -d "2023-12-31 + 1 day"` — покаже 1 січня 2024 року.

## 3. Комбінування ключів та форматів
Ви можете використовувати ключі разом зі специфікаторами (які починаються на +), щоб отримати результат у потрібному вигляді.

Дата для імені бекапу:
```bash
date +%F_%T  # Результат: 2026-04-04_22:37:05
```

Тільки назва дня тижня для вчора:
```bash
date -d "yesterday" +%A
```

Перетворення конкретної дати в Unix Timestamp:
```bash
date -d "2025-01-01 12:00:00" +%s
```

## 4. Встановлення часу (Тільки для root)
Якщо ви не використовуєте NTP (автосинхронізацію), час можна поставити вручну:

Простий варіант: 
```bash
sudo date -s "2026-04-04 22:45:00"
```
Тільки час (дата лишається поточною): 
```bash
sudo date -s "10:00:00"
```

## 5. Робота з файлами
Дуже зручно, якщо треба дізнатися, коли востаннє редагували конфіг:
```bash
date -r /etc/nsswitch.conf +%Y-%m-%d
```

## 6. Формат ISO 8601
Цей ключ дозволяє швидко міняти точність виводу:
- `date -I` — 2026-04-04
- `date -Ihours` — 2026-04-04T22+02:00
- `date -Iseconds` — 2026-04-04T22:37:05+02:00

## 7. Специфікатори для команди date
Специфікаторів для команди date (формат POSIX/GNU) дуже багато. Їх можна розділити на кілька логічних груп: `Час`, `Дата`, `Тиждень/Рік` та `Комбіновані формати`.

Ось повний перелік найкоштовніших із них:
### 1. Форматування дати (день, місяць, рік)
- `%d` — день місяця з нулем попереду (01..31).
- `%e` — день місяця з пробілом попереду ( 1..31). Ідеально для логів.
- `%m` — номер місяця (01..12).
- `%b` або `%h` — скорочена назва місяця (Jan, Feb...).
- `%B` — повна назва місяця (January, February...).
- `%y` — останні дві цифри року (00..99).
- `%Y` — повний рік (приклад: 2024).

### 2. Форматування часу (години, хвилини, секунди)
- `%H` — година у 24-годинному форматі (00..23).
- `%I` — година у 12-годинному форматі (01..12).
- `%M` — хвилини (00..59).
- `%S` — секунди (00..60).
- `%p` — позначка AM або PM (у верхньому регістрі).
- `%P` — позначка am або pm (у нижньому регістрі).
- `%N` — наносекунди (000000000..999999999).

### 3. Дні тижня та календарні дані
- `%a` — скорочений день тижня (Mon, Tue...).
- `%A` — повний день тижня (Monday, Tuesday...).
- `%u` — номер дня тижня (1..7, де 1 — понеділок).
- `%w` — номер дня тижня (0..6, де 0 — неділя).
- `%j` — день року (001..366).
- `%V` — номер тижня у році за стандартом ISO (01..53).

### 4. Готові комбінації (Shortcuts)
Замість того, щоб писати довгі ланцюжки, можна використовувати ці літери:
- `%D` — те саме, що %m/%d/%y (американський формат).
- `%F` — повна дата ISO 8601: %Y-%m-%d. Найзручніший для імен файлів.
- `%T` — час у форматі 24 години: %H:%M:%S.
- `%R` — час у форматі 24 години без секунд: %H:%M.
- `%x` — дата відповідно до локалі (наприклад, 04.04.2024 в Україні).
- `%X` — час відповідно до локалі.

### 5. Робота з Unix Timestamp та Зоною
- `%s` — кількість секунд з 1 січня 1970 року (Unix Epoch). Незамінно для розрахунків в awk.
- `%z` — числова часова зона (наприклад, +0300).
- `%Z` — абревіатура часової зони (наприклад, EEST).

### Спеціальні прапорці (Модифікатори)
Ви можете додавати символи між % та літерою, щоб змінити поведінку:
- `%-` (дефіс) — не додавати заповнювач (замість 01 буде 1).
- `%_` (підкреслення) — заповнювати пробілом (замість 01 буде 1).
- `%0` (нуль) — заповнювати нулями (замість 1 буде 01).

### Приклад використання:
```bash
date +"Сьогодні %A, %-d %B %Y року. Час: %T"
```

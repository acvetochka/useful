# POSIX

## 1. Що таке POSIX

👉 `POSIX` = стандарт (набір правил), який визначає:

- як працюють утиліти (grep, sed, awk)
- як виглядають регулярні вирази
- як поводиться shell

📌 Повна назва:
> Portable Operating System Interface

## 2. Навіщо потрібен POSIX

👉 щоб одна і та сама команда:
```bash
grep 'pattern' file
```
працювала:

- в Linux
- в macOS
- в BSD
- в різних shell

🧠 Простими словами

👉 POSIX =
> "як це має працювати скрізь однаково"

## 3. POSIX regex — головна суть

POSIX визначає два стандарти regex:

| Тип | Назва    |
| --- | -------- |
| BRE | Basic    |
| ERE | Extended |

👉 те, що ми вже розбирали — це і є POSIX regex

## 4. Головні особливості POSIX regex
🔹 1. Немає "сучасних фіч"

❌ немає:

- \d
- \w
- lookahead/lookbehind
- lazy (*?)

🔹 2. POSIX класи (дуже важливо)

Замість `\d`, `\w` використовуються класи:

- Основні класи
  | Клас          | Значення      | Еквівалент    | Приклад  |
  | ------------- | ------------- | ------------- | -------- |
  | `[[:alnum:]]` | букви + цифри | `[A-Za-z0-9]` | `abc123` |
  | `[[:alpha:]]` | тільки букви  | `[A-Za-z]`    | `abc`    |
  | `[[:digit:]]` | цифри         | `[0-9]`       | `123`    |
  | `[[:lower:]]` | малі букви    | `[a-z]`       | `abc`    |
  | `[[:upper:]]` | великі букви  | `[A-Z]`       | `ABC`    |

- Пробіли та керуючі
  | Клас          | Значення              | Що включає          |
  | ------------- | --------------------- | ------------------- |
  | `[[:space:]]` | пробільні символи     | space, tab, newline |
  | `[[:blank:]]` | горизонтальні пробіли | space, tab          |
  | `[[:cntrl:]]` | керуючі символи       | ASCII 0–31, 127     |

- Символи та пунктуація
  | Клас          | Значення          | Що включає              |
  | ------------- | ----------------- | ----------------------- |
  | `[[:punct:]]` | пунктуація        | `! " # $ % & ' ( ) ...` |
  | `[[:graph:]]` | видимі символи    | усе, крім пробілів      |
  | `[[:print:]]` | друковані символи | включає пробіли         |

- Спеціальні
  | Клас           | Значення  | Коментар      |
  | -------------- | --------- | ------------- |
  | `[[:xdigit:]]` | hex цифри | `0-9 A-F a-f` |

📌 Приклад
```bash
grep '[[:digit:]]\+' file
```
👉 PCRE-аналог:
```bash
grep -P '\d+'
```
🔹 3. Жадібність без альтернатив

👉 POSIX regex:
- завжди greedy
- немає lazy

🔹 4. Leftmost-longest правило

👉 це ключова фішка POSIX ❗

📌 Що це означає

Regex знаходить:
- найлівіший match
- і найдовший можливий

🔍 Приклад
```bash
echo "abbbc" | grep -E 'ab*'
```
👉 результат:
> abbb

👉 чому:
- b* → максимум b
- POSIX завжди бере найдовший варіант

⚠️ ВАЖЛИВА різниця з PCRE
| Параметр                   | POSIX                                   | PCRE                                 |
| -------------------------- | --------------------------------------- | ------------------------------------ |
| Як обирається збіг (Match) | найдовший можливий (leftmost-longest)   | перший, що підійшов (leftmost-first) |
| Алгоритм                   | детермінований (без перебору варіантів) | бек-трекінг (перебір варіантів)      |


## 5. Де використовується POSIX
🔧 1. Утиліти Linux
| Утиліта | Тип               |
| ------- | ----------------- |
| `grep`  | BRE/ERE (ключ -E) |
| `sed`   | BRE/ERE (ключ -E) |
| `awk`   | ERE               |

🐚 2. Shell (bash)
```bash
[[ $var =~ regex ]]
```
👉 використовує ERE (POSIX)

📁 3. Системні інструменти
- find
- cut
- sort
- tr

## 6. Чому POSIX важливий
✅ Плюси
- переносимість
- стабільність
- працює всюди

❌ Мінуси
- менш потужний
- незручний синтаксис
- немає сучасних можливостей

## 7. Приклади "POSIX vs PCRE"
🔹 Числа
- POSIX:
  ```bash
  grep -E '[0-9]+'
  ```
- PCRE:
  ```bash
  grep -P '\d+'
  ```
🔹 Слово
- POSIX:
  ```bash
  grep -E '[[:alnum:]_]+'
  ```
- PCRE:
  ```bash
  grep -P '\w+'
  ```
🔹 Пробіли
- POSIX:
  ```bash
  grep '[[:space:]]'
  ```
- PCRE:
  ```bash
  grep -P '\s'
  ```
## 8. Найважливіше запам’ятати

👉 POSIX regex:
- це BRE + ERE
- використовується у Linux утилітах
- не має "сучасних" фіч
- але гарантує переносимість

✅ Висновок

✔️ POSIX = стандарт поведінки  
✔️ BRE/ERE = його regex  
✔️ використовується в grep, sed, awk  
✔️ простіший, але стабільний

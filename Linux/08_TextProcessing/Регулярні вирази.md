# Регулярні вирази в Linux — повна довідка

## 1. Що таке регулярні вирази

`Регулярні вирази` (Regular Expressions, regex) — це шаблони для пошуку та обробки тексту.

У Linux вони використовуються в:
- grep
- sed
- awk
- less, vi, nano
- shell-скриптах

## 2. Типи regex у Linux

У Linux існує два основних стандарти:

🔹 BRE — Basic Regular Expressions  
- використовується в:
  - grep
  - sed
- деякі символи треба екранувати \

🔹 ERE — Extended Regular Expressions
- використовується в:
  - grep -E
  - egrep
  - awk
- має більше можливостей (без \)

## 3. Базові символи

| Символ | Значення              |
| ------ | --------------------- |
| `a`    | конкретний символ     |
| `.`    | будь-який один символ |
| `\`    | escape (екранування)  |

## 4. Метасимволи (ядро regex)
🔹 4.1 Крапка
```
.
````

👉 будь-який символ (крім newline)

🔹 4.2 Якір початку і кінця

| Символ | Значення      |
| ------ | ------------- |
| `^`    | початок рядка |
| `$`    | кінець рядка  |


📌 Приклад:
```bash
^hello
```
👉 рядок починається з hello
```bash
world$
```
👉 рядок закінчується на world

## 5. Квантифікатори (повтори)
🔹 Основні
| Символ | Значення           |
| ------ | ------------------ |
| `*`    | 0 або більше       |
| `+`    | 1 або більше (ERE) |
| `?`    | 0 або 1 (ERE)      |


📌 Приклади:
```
a*
```
👉 "", "a", "aa", "aaa"
```
a+
```
👉 "a", "aa", "aaa"

🔹 Фігурні дужки
| Запис   | Значення   |
| ------- | ---------- |
| `{n}`   | рівно n    |
| `{n,}`  | ≥ n        |
| `{n,m}` | від n до m |


📌 Приклад:
```bash
a{2,4}
```
👉 "aa", "aaa", "aaaa"

## 6. Класи символів
🔹 Простий клас
```bash
[abc]
```
👉 a або b або c

🔹 Діапазон
```bash
[a-z]
[A-Z]
[0-9]
```
🔹 Заперечення
```bash
[^abc]
```
👉 будь-що, крім a, b, c

## 7. POSIX класи символів (дуже важливо)

| Клас          | Значення      |
| ------------- | ------------- |
| `[[:alpha:]]` | букви         |
| `[[:digit:]]` | цифри         |
| `[[:alnum:]]` | букви + цифри |
| `[[:space:]]` | пробіли       |
| `[[:lower:]]` | малі          |
| `[[:upper:]]` | великі        |
| `[[:punct:]]` | пунктуація    |

📌 Приклад:
```bash
[[:digit:]]+
```

## 13. Жадібність (Greedy)
💡 Ідея:

Regex намагається захопити МАКСИМУМ тексту

📌 Приклад:
```bash
echo "abc123xyz" | grep -o 'a.*z'
```
👉 результат:
> abc123xyz

❗ Чому?
- .* = будь-що багато разів
- regex "захоплює" максимально можливий шматок

🔹 Lazy (нежадібний)

Працює тільки в:
```bash
grep -P
```
```bash
echo "a123z456z" | grep -oP 'a.*?z'
```
👉 результат:
> a123z

👉 тут вже бере мінімум

## 14. Backreferences (зворотні посилання)
💡 Ідея:

Можеш використати те, що вже знайшов

📌 Приклад:
```bash
echo "abab" | grep -E '(ab)\1'
```
👉 результат:
> abab

❗ Як це працює:
- (ab) → група №1
- \1 → повтор того ж

👉 тобто:
```
ab + ab
```
📌 Ще приклад:
```bash
echo "hello hello" | grep -E '([a-z]+) \1'
```
👉 знайде:
> hello hello

## 15. Lookahead / Lookbehind

👉 працює тільки в:
```bash
grep -P
```
🔹 Lookahead (дивитись вперед)
```bash
echo "abc123" | grep -oP '\w+(?=\d)'
```
👉 результат:
> abc12

👉 пояснення:
- шукаємо слово
- але тільки якщо після нього цифра

🔹 Negative lookahead
```bash
echo "abcX" | grep -oP 'abc(?!\d)'
```
👉 знайде `abc`, бо після не цифра

🔹 Lookbehind
```bash
echo "123abc" | grep -oP '(?<=\d)abc'
```
👉 знайде abc, бо перед ним цифра

| Тип                 | Синтаксис | Що означає                | Що знайде                        |
| ------------------- | --------- | ------------------------- | -------------------------------- |
| Lookahead           | `a(?=b)`  | після `a` має бути `b`    | знайде `a` у `ab`                |
| Negative Lookahead  | `a(?!b)`  | після `a` НЕ має бути `b` | знайде `a` у `ac`, але не в `ab` |
| Lookbehind          | `(?<=b)a` | перед `a` має бути `b`    | знайде `a` у `ba`                |
| Negative Lookbehind | `(?<!b)a` | перед `a` НЕ має бути `b` | знайде `a` у `ca`, але не в `ba` |


## 16. Regex у Linux командах
🔹 grep
```bash
grep "pattern" file
grep -E "pattern"
grep -P "pattern"
```
🔹 sed
```bash
sed 's/regex/replacement/'
```
🔹 awk
```bash
awk '/regex/ { print }'
```

## 17. Приклади
🔹 Знайти email
```bash
[[:alnum:]._%+-]+@[[:alnum:].-]+\.[a-z]{2,}
```
🔹 Тільки числа
```bash
^[0-9]+$
```
🔹 IP-адреса
```bash
([0-9]{1,3}\.){3}[0-9]{1,3}
```

## 18. Важливі нюанси
❗ BRE vs ERE
| Символ | BRE  | ERE |     |
| ------ | ---- | --- | --- |
| `+`    | `\+` | `+` |     |
| `?`    | `\?` | `?` |     |
| `      | `    | ❌   | ✅   |


❗ Екранування в shell
```bash
grep "\." file
```
або:
```bash
grep '\.'
```

## 19. Debug та тестування

Інструменти:
- grep -E
- grep -P
- онлайн regex testers

## 20. Best practices
- використовуй `' '` замість `" "`
- тестуй regex окремо
- не зловживай .*
- використовуй POSIX класи

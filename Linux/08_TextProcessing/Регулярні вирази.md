# Регулярні вирази в Linux — повна довідка

## 1. Що таке регулярні вирази

`Регулярні вирази` (Regular Expressions, regex) — це шаблони для пошуку та обробки тексту.

У Linux вони використовуються в:
- grep
- sed
- awk
- less, vi, nano
- shell-скриптах

## 2. Типи regex у Linux

У Linux існує два основних стандарти:

🔹 BRE — Basic Regular Expressions  
- використовується в:
  - grep
  - sed
- деякі символи треба екранувати \

🔹 ERE — Extended Regular Expressions
- використовується в:
  - grep -E
  - egrep
  - awk
- має більше можливостей (без \)

## 3. Базові символи

| Символ | Значення              |
| ------ | --------------------- |
| `a`    | конкретний символ     |
| `.`    | будь-який один символ |
| `\`    | escape (екранування)  |

## 4. Метасимволи (ядро regex)
🔹 4.1 Крапка
```
.
````

👉 будь-який символ (крім newline)

🔹 4.2 Якір початку і кінця

| Символ | Значення      |
| ------ | ------------- |
| `^`    | початок рядка |
| `$`    | кінець рядка  |


📌 Приклад:
```bash
^hello
```
👉 рядок починається з hello
```bash
world$
```
👉 рядок закінчується на world

## 5. Квантифікатори (повтори)
🔹 Основні
| Символ | Значення           |
| ------ | ------------------ |
| `*`    | 0 або більше       |
| `+`    | 1 або більше (ERE) |
| `?`    | 0 або 1 (ERE)      |


📌 Приклади:
```
a*
```
👉 "", "a", "aa", "aaa"
```
a+
```
👉 "a", "aa", "aaa"

🔹 Фігурні дужки
| Запис   | Значення   |
| ------- | ---------- |
| `{n}`   | рівно n    |
| `{n,}`  | ≥ n        |
| `{n,m}` | від n до m |


📌 Приклад:
```bash
a{2,4}
```
👉 "aa", "aaa", "aaaa"

## 6. Класи символів
🔹 Простий клас
```bash
[abc]
```
👉 a або b або c

🔹 Діапазон
```bash
[a-z]
[A-Z]
[0-9]
```
🔹 Заперечення
```bash
[^abc]
```
👉 будь-що, крім a, b, c

## 7. POSIX класи символів (дуже важливо)

Використовуються в Linux regex:

| Клас          | Значення      |
| ------------- | ------------- |
| `[[:alpha:]]` | букви         |
| `[[:digit:]]` | цифри         |
| `[[:alnum:]]` | букви + цифри |
| `[[:space:]]` | пробіли       |
| `[[:lower:]]` | малі          |
| `[[:upper:]]` | великі        |
| `[[:punct:]]` | пунктуація    |

📌 Приклад:
```bash
[[:digit:]]+
```

## 8. Групування
🔹 Круглі дужки
```bash
(abc)
```
👉 група

📌 Важливо:

- в BRE: \( \)
- в ERE: ( )

## 9. Альтернатива (OR)
```bash
a|b
```
👉 або a, або b

📌 Тільки в ERE:
```bash
grep -E "cat|dog"
```

## 10. Escape-послідовності

| Запис | Значення          |
| ----- | ----------------- |
| `\.`  | літеральна крапка |
| `\*`  | символ `*`        |
| `\\`  | зворотний слеш    |


## 11. Спеціальні скорочення (залежить від утиліти)

У GNU `grep -P` (Perl regex):

| Символ | Значення   |
| ------ | ---------- |
| `\d`   | цифра      |
| `\w`   | слово      |
| `\s`   | пробіл     |
| `\b`   | межа слова |

## 12. Межі слова
```bash
\bword\b
```
👉 тільки повне слово

## 13. Жадібність (greedy)

За замовчуванням:
```bash
.*
```
👉 захоплює максимум

Lazy (тільки в PCRE):
```bash
.*?
```

## 14. Зворотні посилання (backreferences)
```bash
(abc)\1
```
👉 abcabc

📌 В BRE:
```bash
\(abc\)\1
```

## 15. Lookahead / Lookbehind (PCRE)

Тільки з:
```bash
grep -P
```

| Тип       | Приклад  |
| --------- | -------- |
| Lookahead | `a(?=b)` |
| Negative  | `a(?!b)` |

## 16. Regex у Linux командах
🔹 grep
```bash
grep "pattern" file
grep -E "pattern"
grep -P "pattern"
```
🔹 sed
```bash
sed 's/regex/replacement/'
```
🔹 awk
```bash
awk '/regex/ { print }'
```

## 17. Приклади
🔹 Знайти email
```bash
[[:alnum:]._%+-]+@[[:alnum:].-]+\.[a-z]{2,}
```
🔹 Тільки числа
```bash
^[0-9]+$
```
🔹 IP-адреса
```bash
([0-9]{1,3}\.){3}[0-9]{1,3}
```

## 18. Важливі нюанси
❗ BRE vs ERE
| Символ | BRE  | ERE |     |
| ------ | ---- | --- | --- |
| `+`    | `\+` | `+` |     |
| `?`    | `\?` | `?` |     |
| `      | `    | ❌   | ✅   |


❗ Екранування в shell
```bash
grep "\." file
```
або:
```bash
grep '\.'
```

## 19. Debug та тестування

Інструменти:
- grep -E
- grep -P
- онлайн regex testers

## 20. Best practices
- використовуй `' '` замість `" "`
- тестуй regex окремо
- не зловживай .*
- використовуй POSIX класи

## Висновок

Regex у Linux — це:
- потужний інструмент для тексту
- має кілька стандартів (BRE, ERE, PCRE)
- використовується всюди (grep, sed, awk)

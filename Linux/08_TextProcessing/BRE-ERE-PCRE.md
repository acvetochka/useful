# BRE / ERE / PCRE

## 1. Що це взагалі таке?

У Linux немає “одного regex” — є різні стандарти (движки):

| Тип  | Повна назва                         |
| ---- | ----------------------------------- |
| BRE  | Basic Regular Expressions           |
| ERE  | Extended Regular Expressions        |
| PCRE | Perl Compatible Regular Expressions |

👉 Вони відрізняються:
- синтаксисом
- можливостями
- командами, які їх підтримують

## 2. BRE — Basic Regular Expressions
**📌 Що це**

👉 Найстаріший і базовий стандарт (POSIX)

🔧 Де використовується
- grep
- sed
- vi

❗ Особливість

👉 багато символів треба екранувати

🔹 Приклади
```bash
grep 'a\+' file
```
👉 + працює тільки з \
```bash
grep '\(abc\)\1' file
```
👉 групи:
- \( \)
- backreference: \1

🔥 Основні обмеження
- немає нормального |
- складніший синтаксис
- менш читабельний

**Висновок**

👉 BRE = *мінімальний функціонал + багато *

## 3. ERE — Extended Regular Expressions
**📌 Що це**

👉 Розширена версія BRE

🔧 Де використовується
- grep -E
- egrep
- awk

🔹 Головна перевага

👉 не потрібно екранувати

🔹 Приклади
```bash
grep -E 'a+'
grep -E '(cat|dog)'
```

🔥 Що додає ERE
| Можливість | Приклад      |
| ---------- | ------------ |
| `+`        | `a+`         |
| `?`        | `a?`         |
| `\|`       | `cat \| dog` |
| `( )`      | `(abc)`      |


**Висновок**

👉 ERE = те, що реально використовують у Linux щодня

## 4. PCRE — Perl Compatible Regular Expressions
**📌 Що це**

👉 Найпотужніший regex (як у Perl / Python / JS)

🔧 Де використовується
- grep -P
- perl
- багато мов програмування

**🔥 Додає дуже важливі речі**  
 1. Скорочення
    ```
    \d   # цифра
    \w   # слово
    \s   # пробіл
    ```

  2. Lazy квантифікатори
      ```
      .*?
      ```

  3. Lookaround
      ```
      (?=...)
      (?!...)
      (?<=...)
      (?<!...)
      ```

4. Named groups
    ```bash
    (?<name>abc)
    ```

5. Складні патерни
   - умовні вирази
   - рекурсія
   - atomic groups
  
🔹 Приклади
  ```bash
  grep -P '\d+' file
  grep -P '(?<=\$)\d+' file
  ```

⚠️ Мінуси
- не всюди доступний (grep -P може бути відключений)
- повільніший

**Висновок**

👉 PCRE = максимальна потужність

## 5. Порівняльна таблиця
| Функція    | BRE     | ERE | PCRE |
| ---------- | ------- | --- | ---- |
| `.`        | ✅       | ✅   | ✅    |
| `*`        | ✅       | ✅   | ✅    |
| `+`        | `\+`    | ✅   | ✅    |
| `?`        | `\?`    | ✅   | ✅    |
| `{}`       | `\{ \}` | ✅   | ✅    |
| `()`       | `\(\)`  | ✅   | ✅    |
| `\|`       | ❌       | ✅   | ✅    |
| `\d`       | ❌       | ❌   | ✅    |
| Lazy `*?`  | ❌       | ❌   | ✅    |
| Lookaround | ❌       | ❌   | ✅    |

## 6. Один і той самий приклад у 3 стилях
🎯 Завдання: знайти числа

🔹 BRE
```bash
grep '[0-9]\+' file
```
🔹 ERE
```bash
grep -E '[0-9]+'
```
🔹 PCRE
```bash
grep -P '\d+'
```

## 7. Коли що використовувати
✅ Використовуй BRE коли:
- працюєш з sed
- пишеш POSIX-сумісні скрипти
  
✅ Використовуй ERE коли:
- працюєш з grep
- потрібен чистий синтаксис

👉 це твій default

✅ Використовуй PCRE коли:
- потрібен lookahead/lookbehind
- складна логіка
- обробка тексту як у мовах програмування

## 8. Типові помилки
❌ Забула -E
```bash
grep '(a|b)' file
```
👉 не працює як OR

❌ Зайві `\`
```bash
grep -E '\(abc\)'
```
👉 не потрібно екранувати

❌ Очікування PCRE в ERE
```bash
grep -E '\d+'
```
👉 не працює

## 9. Найпростіше запам’ятати так

👉 BRE → старий і незручний  
👉 ERE → стандартний Linux regex  
👉 PCRE → як у програмуванні

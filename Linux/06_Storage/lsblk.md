# lsblk

Команда `lsblk` (list block devices) виводить інформацію про всі доступні блокові пристрої (жорсткі диски, SSD, флешки, розділи). На відміну від df, вона показує структуру пристроїв (дерево), навіть якщо вони не змонтовані.

## Основні ключі та параметри
| Ключ	| Повна назва |	Опис|
| -----| -----| -----|
| -a	| --all |	Показує всі пристрої, включаючи порожні та RAM-диски.|
| -f	| --fs	| Виводить дані про файлові системи: тип (UUID, Label, Mountpoint). |
| -m	| --perms	| Додає інформацію про права доступу, власника та групу пристрою.|
| -l |	--list	Виводить дані плоским списком, а не деревом.|
| -n	| --noheadings |	Прибирає рядок заголовків (корисно для скриптів).|
| -o [cols]|	--output|	Виводить лише обрані колонки (напр. NAME,SIZE,TYPE).|
| -p	| --paths|	Виводить повні шляхи до пристроїв (напр. /dev/sda1).|
| -d	| --nodeps |	Показує лише фізичні диски, ігноруючи їхні розділи.|
| -J	| --json|	Виводить дані у форматі JSON для обробки програмами.|

## Приклади використання
Класичне дерево пристроїв:
```bash
lsblk
```

Показує назву, розмір, тип (disk/part) та точку монтування.

***
Детальна інформація про файлові системи та UUID:
```bash
lsblk -f
```

Найкращий спосіб дізнатися UUID розділу для запису в /etc/fstab.
***

Вивід лише потрібних колонок:
```bash
lsblk -o NAME,SIZE,FSTYPE,MOUNTPOINT
```
Допомагає позбутися зайвої інформації та сфокусуватися на важливому.
***

Перевірка топології (фізичний сектор):
```bash
lsblk -t
```
Показує розмір логічного та фізичного секторів (корисно для оптимізації SSD).
*** 

Повний шлях до пристроїв:
```bash
lsblk -p
```

Замість `sda1` ви побачите `/dev/sda1`.

## Чим lsblk краща за fdisk -l?
`lsblk` не потребує прав `root` (sudo) для простого перегляду списку пристроїв і візуально набагато зрозуміліша завдяки деревовидній структурі (відразу видно, який розділ до якого диска належить).

## Важлива порада:
Якщо ви додали новий диск або флешку і не бачите її в df, скористайтеся `lsblk` — вона покаже пристрій, навіть якщо він ще не має файлової системи або не підключений до жодної папки.

## можна вивести абсолютно всі колонки?

Так, для цього в команді lsblk передбачено спеціальний прапорець -O (велика літера O, скорочення від --output-all).
Команда:
```bash
lsblk -O
```

**Що це дасть:**  
Ви побачите величезну таблицю (зазвичай понад 40 колонок), яка включає:
- Стандартні дані: назву, розмір, тип, точку монтування.
- Ідентифікатори: UUID, PARTUUID, Label, WWN (унікальний серійний номер заліза).
- Фізичні параметри: розмір сектора, крок видалення (TRIM/DISCARD), серійний номер диска (SERIAL).
- Стан: чи є диск лише для читання (RO), чи він змінний (RM — наприклад, флешка), чи є він частиною RAID.

**Порада щодо перегляду:**  
Оскільки колонок дуже багато, вони не помістяться на екрані й «розсиплються». Щоб зручно їх прогортати вліво-вправо, використовуйте утиліту less з вимкненим переносом рядків:
```bash
lsblk -O | less -S
```

## як вивести в стовпчик всі назви колонок?

Для цього ідеально підходить прапор --help. Він виводить повний список доступних колонок з коротким описом кожної.
Команда для виводу лише назв:
```bash
lsblk --help | sed -n '/Available output columns:/,$p'
```
Будьте обачні, використовуючи код.

Або простіший варіант (виведе назви у стовпчик без описів):
```bash
lsblk -O  | head -n 1 | tr -s ' ' '\n'
```

**Навіщо це знати?**  
Ось кілька крутих колонок, які зазвичай не показуються:
- `SERIAL` — серійний номер диска (зручно, коли треба знайти фізичний диск у сервері).
- `MODEL` — назва моделі (наприклад, Samsung 970 EVO).
- `TRAN` — тип підключення (sata, nvme, usb).
- `ROTA` — чи диск обертається (1 — HDD, 0 — SSD).
- `DISC-MAX` — чи підтримує диск TRIM (важливо для SSD).

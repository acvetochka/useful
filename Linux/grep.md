# Команда grep

Команда grep (Global Regular Expression Print) — це один із найпотужніших інструментів у Linux. Вона призначена для пошуку рядків, що відповідають заданому шаблону, у файлах або вхідному потоці даних.

Ось детальна інструкція, адаптована до твого стилю.

## 1. Основний синтаксис
```bash
grep [опції] "шаблон" [файл ...]
```

- `"шаблон"` — текст або регулярний вираз, який ми шукаємо.
- `[файл ...]` — файл (або список файлів), у яких шукаємо. Якщо файл не вказано, grep чекає дані на вхід (наприклад, через pipe |).

## 2. Найкорисніші опції
Опції можна комбінувати (наприклад, -rin).
- `-i `(ignore case): Ігнорувати регістр. Знайде і "Apple", і "apple".
- `-r` або `-R` (recursive): Рекурсивний пошук у всіх підпапках. Незамінно, якщо треба знайти текст у всьому проєкті.
- `-v` (invert match): Пошук "від зворотного". Покаже всі рядки, де немає вказаного шаблону.
- `-n` (line number): Виводить номер рядка у файлі, де було знайдено збіг.
- `-l` (files with matches): Замість самих рядків виведе лише імена файлів, у яких знайдено текст.
- `-w` (word regexp): Шукає ціле слово. Якщо шукаєш "cat", то "category" вже не підійде.
- `-c `(count): Покаже лише кількість знайдених рядків.

## 3. Робота з контекстом (Опції "Сусіди")
Іноді важливо бачити не тільки рядок зі збігом, а й те, що стоїть поруч:
- `-A n` (After): Показати рядок зі збігом + n рядків після нього.
- `-B n` (Before): Показати рядок зі збігом + n рядків перед ним.
- `-C n` (Context): Показати по n рядків з обох боків.

## 4. Варіанти використання (Приклади)

А. Пошук у файлі
  ```bash
  grep "error" server.log
  ```

  Знайде всі згадки слова "error" у файлі server.log.

Б. Пошук у всіх файлах поточної папки (рекурсивно)
  ```bash
  grep -ri "password" .
  ```
  
  Шукає слово "password" у всіх файлах (.) у поточній директорії та всіх підпапках, ігноруючи регістр.

В. Використання з Pipe (конвеєром)

Це найчастіший сценарій. Наприклад, знайти конкретний процес у списку:
  ```bash
  ps aux | grep "python"
  ```
  
  Команда ps виводить список процесів, а grep фільтрує лише ті, де є "python".

Г. Фільтрація конфігураційних файлів

Часто в конфігах багато коментарів (#). Щоб побачити лише налаштування:
  ```bash
  grep -v "^#" config.conf
  ```
  
  (Тут ^# — це регулярний вираз: рядок, що починається з решітки. Опція -v прибере їх з екрана).

## 5. Регулярні вирази (коротко)

grep розуміє спеціальні символи:

- `^` — початок рядка (наприклад, ^Start).
- `$` — кінець рядка (наприклад, End$).
- `.` — будь-який один символ.
- `*` — будь-яка кількість попередніх символів.

Важлива примітка щодо дужок `[]` у документації:

у `grep [опції] "шаблон" [файл ...]` квадратні дужки кажуть, що:
- `[опції]` — можна не вказувати.
- `"шаблон`" — обов'язковий.
- `[файл ...]` — можна вказати один файл, кілька або жодного (якщо дані йдуть через |).

# Способи відображення файлів команд Linux (на прикладі команди ls)

Файл `/bin/ls` у Linux є бінарним виконуваним файлом (machine code), а не текстовим, тому його не можна просто відкрити в блокноті. Щоб прочитати його "зрозумілою мовою" (людською або зрозумілою для розробників), використовують спеціальні утиліти для аналізу бінарних даних.

Ось основні способи, як це зробити:

## 1. Перегляд текстових рядків у файлі (найпростіший спосіб)

Команда strings витягує всі ASCII-рядки (повідомлення про помилки, назви функцій, довідкові дані), які закладені в бінарний файл.
```bash
strings /bin/ls
```
Що ви побачите: Назви бібліотек, текст помилок (invalid option), прапорці (наприклад, --color, -l, -a).

## 2. Дезасемблювання (переклад у мову Асемблера)
Щоб побачити, які саме інструкції виконує процесор, використовують objdump. Це перекладає двійковий код у мову Асемблера.
```bash
objdump -d /bin/ls | less
```
Що ви побачите: Низькорівневий код, зрозумілий програмістам (mov, push, call).

`-d` — дезасемблювати (disassemble).

`| less` — дозволяє гортати текст (вихід буде дуже довгим).

## 3. Hex-viewer (шістнадцятковий перегляд)
Якщо ви хочете побачити файл у форматі шістнадцяткових кодів (Hex) і ASCII-представленні поруч, використовуйте `hexdump` або `xxd`.
```bash
hexdump -C /bin/ls | less
```
Що ви побачите: Структуру файлу: 4f 75 74 70 75 74 (hex) та їх текстовий відповідник Output (ASCII). 

## 4. Аналіз залежностей та функцій
Щоб зрозуміти, які функції програма використовує з системних бібліотек, використовуйте `readelf` або `nm`.

Функції: `nm -D /bin/ls` (показує динамічні символи/функції).

Інформація про файл: `file /bin/ls` (показує, що це 64-бітний ELF виконуваний файл, скомпілований для Linux).

## 5. Перегляд викликів системи (що команда робить під час роботи)
Замість аналізу самого файлу, можна побачити, що він робить під час запуску, за допомогою strace.
```bash
strace ls /tmp
````

Що ви побачите: Усі системні запити до ядра, які робить ls (відкриття директорій, читання файлів, виведення на екран). 

## Як подивитися початковий код?
`/bin/ls` — це частина GNU Coreutils. Його початковий код написаний на мові C. Його можна знайти в офіційних репозиторіях GNU або на GitHub.
Підсумок: Якщо потрібно зрозуміти, "що всередині" — використовуйте `strings`. Якщо потрібно побачити, "як це працює" — використовуйте `objdump` або `strace`.

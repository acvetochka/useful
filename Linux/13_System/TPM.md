# TPM

`TPM` (Tmux Plugin Manager) — це менеджер плагінів для термінального мультиплексора tmux, який спрощує встановлення, оновлення та керування розширеннями. Він автоматизує завантаження плагінів із репозиторіїв Git і інтегрує їх безпосередньо у конфігурацію tmux, що робить налаштування більш гнучким і відтворюваним.

## Ключові факти
- Розробник: Gpakosz (спільнота tmux)

- Тип ліцензії: MIT License

- Основна функція: Керування плагінами tmux через `.tmux.conf`

- Мова реалізації: Shell (bash)

- Репозиторій: GitHub — [`tmux-plugins/tpm`](https://github.com/tmux-plugins/tpm)

## Як працює
TPM завантажує плагіни зі списку, визначеного у файлі конфігурації `.tmux.conf`. Користувач додає блок із шляхом до плагіна, після чого TPM виконує клонування репозиторію у локальну директорію `~/.tmux/plugins`. Керування здійснюється гарячими клавішами tmux — зазвичай `prefix + I` для встановлення і `prefix + U` для оновлення плагінів.

## Переваги
TPM стандартизує спосіб роботи з плагінами tmux, замінюючи ручне копіювання скриптів автоматизованою системою. Це полегшує поширення конфігурацій між системами, зменшує ризик помилок у налаштуванні й підтримує повторюваність робочого середовища для розробників і системних адміністраторів.

## Поширені плагіни
Через TPM користувачі часто встановлюють популярні доповнення, як-от:

- tmux-sensible — покращені стандартні налаштування
- tmux-resurrect — збереження та відновлення сесій
- tmux-continuum — автоматичне збереження сесій
- tmux-yank — копіювання тексту в системний буфер

## Поточний статус
TPM є найпоширенішим менеджером плагінів у спільноті tmux і залишається активно підтримуваним проєктом із відкритим кодом. Він сумісний із сучасними версіями tmux і підтримується в більшості UNIX-подібних систем.

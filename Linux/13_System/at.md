# at
## Що таке at

`at` — це утиліта для одноразового (one-time) запуску команд у майбутньому.

📌 Складається з:
- `at` — команда для створення задач
- `atd` — демон, який їх виконує

👉 простіше:
- cron = регулярні задачі
- at = разова задача в конкретний момент

## Для чого використовується
Типові кейси:
- виконати щось пізніше (через 10 хв, завтра, о 18:00)
- відкладені дії (shutdown, restart)
- тестові задачі
- автоматизація без постійного cron

## Як працює at
1. ти задаєш час
2. пишеш команду
3. задача зберігається
4. демон atd запускає її у потрібний момент

## Базове використання
🔸 Запуск
```bash
at 18:00
```
👉 відкриється prompt:
```bash
at> echo "Hello"
at> <Ctrl+D>
```

🔸 Через here-string
```bash
echo "echo hello" | at now + 1 minute
```

🔸 Через here-doc
```bash
at now + 5 minutes <<EOF
echo "test"
touch /tmp/file
EOF
```

## Формати часу
🔸 Конкретний 
```bash
at 15:
```

🔸 Дата + час
```bash
at 15:30 10.04.2026
```

🔸 Відносний час
```bash
at now + 10 minutes
at now + 2 hours
at now + 3 days
```

🔸 Інші формати
```bash
at midnight
at noon
at teatime
```

## Управління задачами
🔸 Список задач
```bash
atq
```
🔸 Видалити задачу
```bash
atrm 5
```
🔸 Подивитися задачу
```bash
at -c 5
```

## Як виглядає задача всередині

👉 at зберігає:
- команди
- змінні середовища
- поточну директорію

## Середовище (ВАЖЛИВО)

👉 на відміну від cron:

✅ at зберігає твоє середовище
- PATH
- змінні
- cwd

📌 Але:  
👉 краще все одно використовувати абсолютні шляхи

## Логування і output

👉 результат команди:
> надсилається в mail користувачу

🔸 Приклад
```bash
echo "ls" | at now + 1 minute
```
👉 результат прийде в mail

🔸 Перенаправлення в файл
```bash
echo "ls > /tmp/out.txt 2>&1" | at now + 1 minute
```

## Права доступу

Контролюються файлами:
- `/etc/at.allow`
- `/etc/at.deny`
  
🔸 Логіка
- якщо є at.allow → тільки ці користувачі
- інакше → всі, крім тих, хто в at.deny

## Демон atd

👉 повинен бути запущений:
```bash
systemctl status atd
```
🔸 Запуск
```bash
sudo systemctl start atd
sudo systemctl enable atd
```

# Практичні приклади
🔸 Shutdown через 1 годину
```bash
echo "shutdown -h now" | at now + 1 hour
```

🔸 Видалити файл пізніше
```bash
echo "rm /tmp/file" | at now + 30 minutes
```

🔸 Запустити backup
```bash
at 02:00 <<EOF
/home/user/backup.sh
EOF
```

🔸 Нагадування
```bash
echo 'notify-send "Time to rest"' | at now + 1 hour
```

## Просунуті можливості
🔸 Виконання кількох команд
```bash
at now + 1 minute <<EOF
cd /project
git pull
npm install
EOF
```

🔸 Затримка виконання
```bash
sleep 10 && command
```

## Відмінності від cron
| Функція        | `at` | `cron`  |
| -------------- | ---- | ------- |
| Повторюваність | ❌    | ✔       |
| Простота       | ✔    | ❌       |
| Середовище     | ✔    | ❌       |
| Гнучкість часу | ✔    | середня |


## Типові помилки
❗ 1. atd не запущений

👉 нічого не виконується

❗ 2. Неправильний час
```bash
at 25:00 ❌
```

❗ 3. Немає прав  

❗ 4. Немає mail → не бачиш output


## Best Practices

✔ Використовуй для разових задач  
✔ Перевіряй atq  
✔ Логуй output  
✔ Перевіряй atd  
✔ Не замінюй cron  

## Мінімальний шаблон
```bash
echo "/script.sh >> /log.txt 2>&1" | at now + 10 minutes
```

## Висновок

👉 at — це:

- простий планувальник
- для разових задач
- дуже зручний для швидких дій

## Коли використовувати

👉 Використовуй at, якщо:

- треба "запустити один раз пізніше"
- не хочеш писати cron
- потрібно швидко відкласти дію

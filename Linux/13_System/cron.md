# cron
## Що таке cron

`cron` — це системний планувальник задач у Linux/Unix, який дозволяє:

👉 автоматично запускати команди або скрипти за розкладом

📌 Складається з двох частин:
- cron — демон (служба), який виконує задачі
- crontab — файл із розкладом задач

## Для чого використовується
Типові кейси:
- резервні копії (backup)
- очищення логів
- запуск скриптів
- синхронізація даних
- відправка email
- оновлення системи
- cron для веб-додатків (jobs, queues)

## Як працює cron

👉 Демон постійно працює у фоні і:
1. читає crontab
2. кожну хвилину перевіряє час
3. якщо є збіг — запускає команду

## Формат cron (основа)
```bash
* * * * * command
│ │ │ │ │
│ │ │ │ └── день тижня (0–7) (0/7 = неділя)
│ │ │ └──── місяць (1–12)
│ │ └────── день місяця (1–31)
│ └──────── година (0–23)
└────────── хвилина (0–59)
```

🔸 Приклад
```bash
0 2 * * * /home/user/backup.sh
```
👉 запуск кожен день о 02:00

## Спеціальні символи
- `*` — будь-яке значення
  ```bash
  * * * * * → кожну хвилину
  ```

- `,` — список
  ```bash
  0 9,18 * * *
  ```
  👉 о 9:00 і 18:00

- `-` — діапазон
  ```bash
  0 9-17 * * *
  ```
  👉 щогодини з 9 до 17

- `/` — крок
  ```bash
  */5 * * * *
  ```
  👉 кожні 5 хвилин

## Спеціальні рядки (shortcuts)
```bash
@reboot     # при запуску системи
@daily      # раз на день
@hourly     # щогодини
@weekly     # раз на тиждень
@monthly    # раз на місяць
@yearly     # раз на рік
```

## Робота з crontab
🔸 Відкрити редактор
```bash
crontab -e
```
🔸 Подивитися
```bash
crontab -l
```
🔸 Видалити
```bash
crontab -r
```
🔸 Для іншого користувача
```bash
sudo crontab -u username -e
```
## Системні cron

Є також:
- `/etc/crontab`
- `/etc/cron.d/`
- `/etc/cron.daily/`
- `/etc/cron.hourly/`


🔸 Важлива різниця

У `/etc/crontab` є ще поле:
```bash
* * * * * user command
```

## Середовище (ДУЖЕ ВАЖЛИВО)

👉 cron НЕ запускається як твій shell

Це означає:
- немає PATH як у тебе
- немає alias
- немає змінних середовища

🔸 Рішення
1. Вказувати повні шляхи
    ```bash
    /usr/bin/node script.js
    ```
2. Задати PATH
    ```bash
    PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
    ```
3. Використовувати bash
    ```bash
    bash -c "your command"
    ```

## Логування

👉 за замовчуванням cron мовчить

🔸 Запис у файл
```bash
* * * * * /script.sh >> /var/log/script.log 2>&1
```
🔸 Що означає
```
2>&1
```
👉 stderr → stdout

## Перевірка роботи
🔸 Логи cron

У системі:
```bash
journalctl -u cron
```
або:
```bash
/var/log/syslog
```

## Типові помилки
❗ 1. Не працює команда

👉 причина:
- немає PATH
- неправильний shell

❗ 2. Відносні шляхи
```bash
script.sh ❌
/home/user/script.sh ✅
```

❗ 3. Права доступу
```bash
chmod +x script.sh
```

❗ 4. Неправильний час

👉 перевір:
```bash
timedatectl
```

## Практичні приклади
🔸 Backup щодня
```bash
0 3 * * * /home/user/backup.sh
```

🔸 Очистка логів
```bash
0 0 * * 0 find /logs -type f -mtime +7 -delete
```
🔸 Кожні 10 хв
```bash
*/10 * * * * /script.sh
```
🔸 Після reboot
```bash
@reboot /start.sh
```

## Просунуті можливості
🔸 Lock (щоб не запускалось двічі)
```bash
flock -n /tmp/script.lock /script.sh
```
🔸 Timeout
```bash
timeout 30 /script.sh
```
🔸 Перевірка перед запуском
```bash
[ -f /tmp/ok ] && /script.sh
```

## Альтернатива cron

👉 сучасний варіант:
```bash
systemd timers
```
Переваги:
- логування
- залежності
- гнучкість

## Best Practices

✔ Завжди використовуй абсолютні шляхи  
✔ Логуй усе  
✔ Перевіряй вручну команду  
✔ Використовуй flock  
✔ Не запускай важкі задачі кожну хвилину  

## Мінімальний "production" шаблон
```bash
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

# backup
0 2 * * * /home/user/backup.sh >> /var/log/backup.log 2>&1
```

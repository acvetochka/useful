# usermod

Нижче — повна довідка по команді usermod у форматі, який зручно використати як окремий файл у твоїй документації. Я додав: опис, принцип роботи, усі основні ключі, приклади, типові сценарії та важливі нюанси безпеки.

## Команда usermod

`usermod` — це системна утиліта Linux для зміни параметрів існуючого користувача.
Вона дозволяє змінювати:
- ім'я користувача
- UID
- групи
- домашній каталог
- shell
- термін дії облікового запису
- парольний статус
- блокування / розблокування

Команда є частиною пакету `shadow-utils`.

Синтаксис
```bash
usermod [options] LOGIN
```
де

- `LOGIN` — ім’я користувача, якого потрібно змінити.

## Що змінює usermod у системі

Команда редагує системні файли:
```
/etc/passwd
/etc/shadow
/etc/group
/etc/gshadow
```
Саме там зберігається інформація про користувачів.

## Основні параметри (ключі)
- `-c COMMENT`

  Змінює поле опису користувача (GECOS).
  ```bash
  usermod -c "Backend developer" user1
  ```
  У файлі `/etc/passwd`:
  ```accesslog
  user1:x:1001:1001:Backend developer:/home/user1:/bin/bash
  ```

- `-d HOME_DIR`

  Змінює домашній каталог.
  ```bash
  usermod -d /data/user1 user1
  ```
  ⚠️ Сам каталог не переміщується, лише змінюється запис у `/etc/passwd`.

- `-m`

  Використовується разом з `-d`.

  Переміщує домашній каталог.
  ```bash
  usermod -d /data/user1 -m user1
  ```
  Файли з:
  ```
  /home/user1
  ```
  перемістяться у
  ```
  /data/user1
  ```
- `-e EXPIRE_DATE`

  Встановлює дату закінчення дії акаунта.

  Формат:
  ```
  YYYY-MM-DD
  ```
  Приклад:
  ```bash
  usermod -e 2026-12-31 user1
  ```
  Після цієї дати користувач не зможе входити у систему.

- `-f INACTIVE`

  Визначає кількість днів після закінчення дії пароля, коли акаунт буде деактивований.
  ```bash
  usermod -f 30 user1
  ```
  Через 30 днів після прострочення пароля акаунт блокується.

- `-g GROUP`

  Змінює primary group користувача.
  ```bash
  usermod -g developers user1
  ```

- `-G GROUPS`

  Задає додаткові групи.
  ```bash
  usermod -G docker,developers user1
  ```
  > [!IMPORTANT]
  >  ця команда перезаписує список груп.  
  >  Тобто користувач буде лише у цих групах.

- `-a`

  Використовується разом з `-G`.

  Додає групу без видалення інших.
  ```bash
  usermod -aG docker user1
  ```
  Це правильний спосіб додавання користувача до групи.

  > ⚠️ Типова помилка  
  > `usermod -G docker user1`  
  > У цьому випадку користувач втратить всі інші групи.

- `-l NEW_LOGIN`

  Змінює ім’я користувача.
  ```bash
  usermod -l newname oldname
  ```
  ⚠️ Домашній каталог не змінюється автоматично.

- `-L`

  Блокує користувача (Lock).
  ```bash
  usermod -L user1
  ```
  **Що відбувається технічно**

  Команда додає символ `!` у файл `/etc/shadow`

  До блокування
  ```accesslog
  user1:$6$hash....
  ```

  Після блокування
  ```accesslog
  user1:!$6$hash....
  ```
  Це означає:
  - пароль не може використовуватися для входу.

  Важливий нюанс

  `usermod -L`:
  - блокує лише пароль
  - не завершує активні сесії
  - не блокує SSH ключі

  Користувач може залишатися активним, якщо:
  - використовує SSH key authentication
  - має запущені процеси
  - має cron jobs

- `-U`

  Розблоковує користувача (Unlock).
  ```bash
  usermod -U user1
  ```
  Технічно:

  символ `!` видаляється з `/etc/shadow`.

  До
  ```accesslog
  user1:!$6$hash....
  ```
  Після
  ```accesslog
  user1:$6$hash....
  ```

- `-s SHELL`

  Змінює login shell.
  ```bash
  usermod -s /bin/zsh user1
  ```
  Щоб заборонити логін:
  ```bash
  usermod -s /usr/sbin/nologin user1
  ```
  або
  ```bash
  usermod -s /bin/false user1
  ```

- `-u UID`

  Змінює UID користувача.
  ```accesslog
  usermod -u 2000 user1
  ```
  > [!IMPORTANT]
  > файли користувача не змінять UID автоматично.

  Їх потрібно виправити:
  ```bash
  find / -uid OLD_UID -exec chown user1 {} \;
  ```
- `-o`

  Дозволяє неунікальний UID.
  ```accesslog
  usermod -u 1000 -o user1
  ```
  Використовується рідко.

- `-p PASSWORD`

  Задає пароль у вигляді зашифрованого хешу.
  ```bash
  usermod -p HASH user1
  ```
  Зазвичай використовують `passwd`, а не цей параметр.

## Перевірка стану користувача

Команда:
```bash
passwd -S user1
```
Приклад:
```accesslog
user1 L 2025-01-01 0 99999 7 -1
```

- `L` означає locked.

## Повне блокування користувача

`usermod -L` часто використовують як швидку реакцію на компрометацію пароля, але важливо розуміти що саме воно захищає, а що — ні.

Коли використовують usermod -L

Типовий сценарій:
1) є підозра, що пароль користувача став відомий стороннім
2) потрібно миттєво припинити логін по паролю

Тоді адміністратор виконує:
```bash
sudo usermod -L username
```

Що відбувається технічно

У файлі:
```bash
/etc/shadow
```
перед хешем пароля додається символ `!` 

Було:
```accesslog
user:$6$hash....
```
Стає:
```accesslog
user:!$6$hash....
```
Це означає:
> ➡ жоден пароль не зможе співпасти з хешем

Що це блокує

Після usermod -L користувач не зможе входити по паролю через:
- login
- ssh з паролем
- su
- sudo з паролем

> [!IMPORTANT]
> Але є важливий нюанс  
> `usermod -L` не блокує доступ через SSH-ключі.

Наприклад, якщо користувач входить так:
  ```bash
  ssh user@server
  ```
  і використовує
  ```bash
  ~/.ssh/authorized_keys
  ```
  то доступ може залишитися.

Тому при реальному інциденті роблять більше

Типова реакція адміністратора виглядає так:

1️⃣ блокують пароль
  ```bash
  usermod -L username
  ```
2️⃣ завершують активні сесії
```bash
pkill -u username
```
або
```bash
loginctl terminate-user username
```
3️⃣ перевіряють SSH ключі
```bash
~/.ssh/authorized_keys
```
4️⃣ змінюють пароль
```bash
passwd username
```
5️⃣ розблоковують користувача
```bash
usermod -U username
```
***
**Коли ще використовують usermod -L**
- співробітник звільнився  
  акаунт не видаляють одразу, а блокують.

- тимчасове блокування

  наприклад:
  -  відпустка
  -    перевірка доступів

- системні акаунти

  іноді блокують пароль, щоб користувач не міг логінитись взагалі.
***

**Ще один нюанс**
> usermod -L не змінює shell користувача.

Тому іноді додатково роблять:
```bash
usermod -s /usr/sbin/nologin username
```
щоб повністю заборонити логін.


## Типові сценарії використання
- Додати користувача до групи
  ```bash
  usermod -aG docker user1
  ```
- Змінити shell
  ```bash
  usermod -s /bin/zsh user1
  ```
- Заблокувати акаунт
  ```bash
  usermod -L user1
  ```

- Розблокувати акаунт
  ```bash
  usermod -U user1
  ```
- Перемістити домашній каталог
  ```bash
  usermod -d /data/user1 -m user1
  ```
- Встановити дату закінчення акаунта
  ```bash
  usermod -e 2027-01-01 user1
  ```

| параметр | значення                   |
| -------- | -------------------------- |
| -c       | змінити опис               |
| -d       | змінити home directory     |

| -e       | дата закінчення акаунта    |
| -f       | неактивність після пароля  |
| -g       | primary group              |
| -G       | secondary groups           |
| -a       | додати групу               |
| -l       | змінити login              |
| -L       | заблокувати пароль         |
| -U       | розблокувати пароль        |
| -s       | змінити shell              |
| -u       | змінити UID                |
| -o       | дозволити неунікальний UID |
| -p       | встановити пароль (hash)   |

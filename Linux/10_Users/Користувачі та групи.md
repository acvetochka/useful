## 1. Облікові записи користувачів

Linux — багатокористувацька операційна система.

Це означає, що:
- одночасно можуть працювати багато користувачів
- кожен користувач має власний набір файлів
- доступ до ресурсів контролюється системою прав

Кожен користувач має:

| Параметр       | Опис                     |
| -------------- | ------------------------ |
| username       | ім'я користувача         |
| UID            | унікальний ідентифікатор |
| GID            | ідентифікатор групи      |
| home directory | домашній каталог         |
| shell          | командна оболонка        |

Основні принципи:
- Користувач (user) — обліковий запис.
- Група (group) — об'єднання користувачів.
- Власник (owner) — користувач, якому належить файл.
- Права доступу (permissions) — визначають, що можна робити з файлом.

### UID (User ID)

Це числовий ідентифікатор користувача.

Типові діапазони:
| UID   | Тип                  |
| ----- | -------------------- |
| 0     | root                 |
| 1–999 | системні користувачі |
| 1000+ | звичайні користувачі |

Перевірити UID:
```bash
id
```
Приклад:
```bash
uid=1000(user) gid=1000(user) groups=1000(user),27(sudo)
```

### Типи користувачів:
1. Root   
  UID = `0`  
  Має повний контроль над системою.  
  Може:
     - читати всі файли
     - змінювати права
     - керувати користувачами
     - запускати будь-які процеси

1. Звичайні користувачі  
  UID: `1000+`
  Використовуються для роботи людей.

3. Системні користувачі  
   UID: `1–999`  
   Використовуються службами:
   - www-data
   - mysql
   - postfix
   - nginx  
   - Вони не входять у систему.

## 2. Спеціальні файли користувачів

Інформація про користувачів зберігається у файлах:
```
/etc/passwd
/etc/shadow
/etc/group
/etc/gshadow
```

- `/etc/passwd`

  Основний файл з інформацією про користувачів.

  Формат:
  ```
  username:password:UID:GID:comment:home:shell
  ```
  Поля:
  | Поле     | Значення                     |
  | -------- | ---------------------------- |
  | username | ім'я                         |
  | password | x (реальний пароль у shadow) |
  | UID      | user id                      |
  | GID      | primary group                |
  | comment  | опис                         |
  | home     | домашній каталог             |
  | shell    | оболонка                     |

  Перегляд:
  ```bash
  cat /etc/passwd
  ```

    Приклад рядка:
  ```
  user:x:1000:1000:User Description:/home/user:/bin/bash
  ```

- `/etc/shadow`

  Зберігає зашифровані паролі.


  Формат:
  ```bash
  username:hash:last_change:min:max:warn:disable:expire
  ```
  Пояснення:
  | Поле        | Значення        |
  | ----------- | --------------- |
  | hash        | хеш пароля      |
  | last_change | дата зміни      |
  | min         | мінімальний час |
  | max         | максимальний    |
  | warn        | попередження    |

  Перегляд (root):
  ```bash
  sudo cat /etc/shadow
  ```

  Приклад:
  ```bash
  user:$6$hash:19700:0:99999:7:::
  ```

- `/etc/group`

  Файл груп.


  Формат:
  ```bash
  group_name:password:GID:members
  ```

  Приклад:
  ```bash
  developers:x:1001:user1,user2
  ```

- `/etc/gshadow`

  Аналог shadow для груп.

## 4. Утиліти для роботи з користувачами
- `useradd`

  Створення користувача.
  ```bash
  useradd username
  ```
  Важливі ключі
  | Ключ | Значення        |
  | ---- | --------------- |
  | -m   | створити home   |
  | -d   | задати home     |
  | -s   | shell           |
  | -u   | UID             |
  | -g   | primary group   |
  | -G   | додаткові групи |
  | -c   | коментар        |

  Приклад
  ```bash
  useradd -m -s /bin/bash -c "Developer" user1
  ```

- `adduser`

  Дружня оболонка над useradd.
  ```bash
  adduser username
  ```
  Вона:
  - створює home
  - створює групу
  - просить пароль

- `passwd`

  Встановлення пароля.
  ```bash
  passwd username
  ```
  Ключі
  | Ключ | Значення        |
  | ---- | --------------- |
  | -l   | lock            |
  | -u   | unlock          |
  | -d   | delete password |
  | -e   | force change    |

  Приклад:
  ```bash
  passwd -l user1
  ```
- `usermod`

  Зміна параметрів користувача.

  Ключі
  | Ключ | Значення      |
  | ---- | ------------- |
  | -aG  | додати групу  |
  | -G   | задати групи  |
  | -l   | змінити ім'я  |
  | -d   | змінити home  |
  | -s   | змінити shell |
  | -u   | змінити UID   |

  Приклад:
  ```bash
  usermod -aG sudo user1
  ```

- `userdel`

  Видалення користувача.
  ```bash
  userdel username
  ```
  Ключ
  - `-r` -  Видалити разом з home.
  - `-f` - видалити користувача навіть якщо він знаходиться в системі
  
  ```bash
  userdel -r user1
  ```

## 5. Групи

Групи дозволяють керувати доступом.
- `groupadd`

  Створення групи.
  ```bash
  groupadd developers
  ```

  Ключі
  | Ключ | Значення   |
  | ---- | ---------- |
  | -g   | задати GID |

- `groupdel`
  ```bash
  groupdel developers
  ```

- `groupmod`

  Зміна групи.

  ```bash
  groupmod -n newname oldname
  ```

## 6. Команди перегляду користувачів

- `who` - Хто у системі.
  ```bash
  who
  ```

- `whoami` - Поточний користувач.
  ```bash
  whoami
  ```

- `id` - Показує UID і групи.
  ```bash
  id username
  ```

- `groups`
  ```bash
  groups username
  ```

- `getent` - Показує записи з системних баз.
  ```bash
  getent passwd
  getent group
  ```

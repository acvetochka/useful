# Стандартні потоки введення-виведення (Standrt Streams)

У Linux під терміном «потоки» найчастіше мають на увазі стандартні потоки введення-виведення (Standard Streams). Це канали зв'язку між програмою та її середовищем. 

Кожна команда або процес у Linux за замовчуванням має три такі потоки: 

## 1. stdin (Standard Input) — Стандартне введення

- Дескриптор файлу: 0.
- Призначення: Потік, з якого програма читає вхідні дані.
- За замовчуванням: Ваша клавіатура.
- Символ перенаправлення: < (наприклад, command < file.txt змусить команду читати дані з файлу замість клавіатури). 

## 2. stdout (Standard Output) — Стандартне виведення
- Дескриптор файлу: 1.
- Призначення: Потік, куди програма відправляє результати своєї успішної роботи.
- За замовчуванням: Екран термінала.
- Символ перенаправлення: > або 1> (наприклад, ls > list.txt збереже список файлів у документ). 

## 3. stderr (Standard Error) — Стандартні помилки
- Дескриптор файлу: 2.
- Призначення: Окремий потік спеціально для діагностичних повідомлень та помилок.
- За замовчуванням: Також екран термінала.
- Символ перенаправлення: 2> (наприклад, command 2> errors.log запише лише помилки у файл). 

## Чому це важливо?
Розділення на stdout та stderr дозволяє користувачеві бачити результат роботи на екрані, одночасно зберігаючи помилки в окремий лог-файл, або навпаки — ігнорувати помилки, спрямовуючи їх у порожнечу. 

Також існує поняття конвеєра (pipe) — символ |, який з'єднує stdout першої команди зі stdin другої, дозволяючи будувати складні ланцюжки обробки даних.

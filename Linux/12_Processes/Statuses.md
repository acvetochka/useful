## Загальний принцип

У полі STAT або S виводиться:
> process state (стан процесу)
або точніше:
> process state code (код стану процесу)

Зазвичай він складається з:
- main state (основний стан)
- additional flags (додаткові прапорці)

Наприклад:
- `S` = Interruptible Sleep (переривний сон / очікування)
- `Sl` = Interruptible Sleep, multithreaded (переривний сон, багатопотоковий процес)
- `S+` = Interruptible Sleep, foreground process group (переривний сон, процес у foreground-групі термінала)

## 1. Основні стани процесу
### `R` — Running / Runnable (виконується / готовий до виконання)
**Що означає**

Процес:
- або running (виконується) прямо зараз на CPU
- або runnable (готовий до виконання) і чекає, коли планувальник дасть йому процесорний час

**Важливий нюанс**

У Linux `R` не завжди означає, що процес уже прямо зараз використовує ядро CPU.

Часто це ширше поняття:
> running or runnable (виконується або готовий виконуватися)

**Коли це нормально**
- активна команда в терміналі
- компіляція
- обчислення
- скрипт, який зараз працює
- серверний процес під навантаженням

**Коли це може бути проблемою**

Якщо багато процесів мають стан R, це може означати:
- CPU contention (конкуренція за CPU)
- high load average (високий load average)
- CPU bottleneck (вузьке місце в процесорі)

### `S` — Interruptible Sleep (переривний сон / очікування події)
**Що означає**

Процес зараз не виконується, а чекає подію.

Це найпоширеніший і зазвичай нормальний стан.

Чому `interruptible`

Тому що такий процес можна розбудити:
- сигналом
- появою даних
- завершенням таймера
- приходом мережевого пакета
- іншою очікуваною подією

**Типові приклади**
- bash чекає введення команди
- sshd чекає нове підключення
- вебсервер чекає request
- процес чекає дані із сокета

**Висновок**

`S` зазвичай означає:

idle waiting in normal mode (нормальне очікування в простої)

### `D` — Uninterruptible Sleep (непереривний сон / непереривне очікування в ядрі)
**Що означає**

Процес чекає на подію в ядрі, найчастіше пов’язану з:
- disk I/O (дисковим введенням-виведенням)
- block device I/O (операціями з блоковим пристроєм)
- filesystem activity (активністю файлової системи)
- NFS / network storage (мережевим сховищем)

**Чому це важливо**

Це один із найважливіших станів для troubleshooting.

Процес у стані `D` часто виглядає як завислий, але насправді він чекає завершення низькорівневої операції.

**Чому `kill -9` часто не допомагає**

Бо процес не просто “спить”, а знаходиться в:

> kernel wait (очікуванні в ядрі)

Поки ядро не завершить або не зірве цю операцію, процес може не зникати.

**Типові причини**
- повільний диск
- завислий диск
- проблеми з RAID/SAN/NFS
- збій драйвера
- зависле монтування

**Висновок**

`D` часто означає:

I/O wait problem (проблема очікування введення-виведення)

### `T` — Stopped (зупинений)
**Що означає**

Процес зупинений, але не завершений.

Зазвичай це стається через:
- job control signal (сигнал керування задачами)
- Ctrl+Z
- SIGSTOP
- SIGTSTP

**Що це означає на практиці**

Процес призупинений і не виконується, але його можна:
- продовжити через `fg`
- продовжити через `bg`
- завершити

**Висновок**

`T` = stopped by user or shell control (зупинений користувачем або механізмом керування задачами shell)

### `t` — Tracing Stop (зупинений трасуванням / відладкою)
**Що означає**

Процес зупинено через:
- debugging (налагодження)
- tracing (трасування)
- `gdb`
- `strace`
- `ptrace`

### Різниця між `T` і `t`

- `T` = Stopped (зупинений) звичайним stop-сигналом або job control

- `t` = Tracing Stop (зупинений трасуванням) відладчиком або інструментом трасування

### `Z` — Zombie (зомбі-процес)
**Що означає**

Процес уже завершився, але його запис ще залишається в таблиці процесів.

Іншими словами:
- код уже не виконується
- майже всі ресурси вже звільнені
- але parent process (батьківський процес) ще не забрав exit status (код завершення) через wait()

**Як це виглядає**

Часто видно так:

> defunct process (дефектний / “мертвий” запис процесу)

наприклад:
```bash
[myproc] <defunct>
```

**Чому це виникає**

Коли дочірній процес завершується, ядро тимчасово зберігає його мінімальну інформацію, щоб батьківський процес міг її прочитати.

**Чи небезпечно це**

Окремі zombie-процеси — не страшно.
Але якщо їх багато, це означає:
- bug у parent process
- parent не викликає `wait()`
- процеси не прибираються належним чином

**Висновок**

`Z` = terminated but not reaped (завершений, але ще не “прибраний” батьківським процесом)

### `I` — Idle Kernel Thread (неактивний потік ядра)
**Що означає**

Це потік ядра, який зараз нічого не робить і чекає роботу.

**Де зустрічається**

Переважно для:
> kernel threads (потоків ядра)

**Висновок**

`I` зазвичай нормальний і не є проблемою.

### `X` — Dead (мертвий)
**Що означає**

Дуже рідкісний стан, який означає:

> dead process (мертвий процес)

У нормальній повсякденній роботі майже не зустрічається.

### `W` — Paging (пейджинг / сторінковий обмін пам’яті)
**Що означає**

Історичний стан:

> paging (обмін сторінками пам’яті)

У сучасних Linux майже не використовується і вважається застарілим.

## 2. Додаткові прапорці
### `<` — High Priority (високий пріоритет)
Що означає

Процес має:

> high scheduling priority (високий пріоритет планування)

Іноді це описують як:

> not nice to other users (не надто “ввічливий” до інших процесів)

**Практично**

Процес отримує кращий доступ до CPU.

### `N` — Low Priority (низький пріоритет)
**Що означає**

Процес має:

> low scheduling priority (низький пріоритет планування)

Часто це результат `nice` або `renice`.

**Практично**

Такий процес більш “ввічливий” і легше поступається CPU іншим.

### `L` — Locked Pages (заблоковані сторінки пам’яті)
**Що означає**

Процес має сторінки пам’яті, які:

> locked into memory (заблоковані в оперативній пам’яті)

Тобто вони не повинні бути витіснені в `swap`.

**Де трапляється**
- realtime applications (системи реального часу)
- low-latency applications (системи з низькою затримкою)
- спеціалізовані сервіси

### `s` — Session Leader (лідер сесії)
**Що означає**

Процес є:

> session leader (лідером сесії)

**Простими словами**

Це головний процес певної сесії термінала або запуску.

Часто це:
- shell
- login shell
- деякі демони

### `l` — Multithreaded (багатопотоковий)
**Що означає**

Процес має:

> multiple threads (кілька потоків виконання)

**Де часто зустрічається**
- браузери
- Java applications (Java-застосунки)
- сучасні сервери
- GUI applications (графічні програми)

### `+` — Foreground Process Group (foreground-група процесів)
**Що означає**

Процес належить до:

> foreground process group (активної групи процесів термінала)

**Простими словами**

Це процес, який зараз “володіє” терміналом і отримує сигнали типу:
- Ctrl+C
- Ctrl+Z

### 3. Розбір популярних комбінацій
- `S` — Interruptible Sleep (переривний сон / очікування події)

  Процес спить і чекає подію. Це нормально.

- `Sl` — Interruptible Sleep, multithreaded (переривний сон, багатопотоковий)
  
  Процес спить, чекає подію і має кілька потоків.

- `S+` — Interruptible Sleep, foreground process group (переривний сон, foreground-процес термінала)
  
  Процес спить, але належить активній групі процесів поточного термінала.

- `Ss` — Interruptible Sleep, session leader (переривний сон, лідер сесії)
  
  Процес чекає подію і є лідером сесії.

- `Ssl` — Interruptible Sleep, session leader, multithreaded (переривний сон, лідер сесії, багатопотоковий)
  
  Комбінований статус для багатопотокового процесу-лідера сесії, який зараз чекає подію.

- `R` — Running / Runnable (виконується / готовий до виконання)
  
  Процес активний або готовий стати активним.

- `R+` — Running / Runnable, foreground process group (виконується / готовий до виконання, foreground-процес)
  
  Активний процес у foreground термінала.

- `Rl` — Running / Runnable, multithreaded (виконується / готовий до виконання, багатопотоковий)
  
  Багатопотоковий активний процес.

- `T` — Stopped (зупинений)
  
  Процес призупинений.

- `T+` — Stopped, foreground process group (зупинений foreground-процес)
  
  Зупинений процес, що був активним у терміналі.

- `t` — Tracing Stop (зупинений трасуванням / відладкою)
  
  Процес зупинений дебагером або інструментом трасування.

- `Z` — Zombie (зомбі-процес)
  
  Процес завершився, але ще не прибраний parent process.

- `Z+` — Zombie, foreground process group (зомбі-процес foreground-групи)
  
  Zombie-процес, пов’язаний із foreground-групою термінала.

- `D` — Uninterruptible Sleep (непереривний сон / очікування в ядрі)
  
  Процес чекає I/O або подію в ядрі. Це важливий діагностичний сигнал.

- `Dl` — Uninterruptible Sleep, multithreaded (непереривний сон, багатопотоковий)
  
  Багатопотоковий процес очікує завершення kernel/I/O-операції.

- `SN` — Interruptible Sleep, low priority (переривний сон, низький пріоритет)
  
  Процес спить і має знижений пріоритет.

- `R<` — Running / Runnable, high priority (виконується / готовий до виконання, високий пріоритет)
  
  Активний процес із підвищеним пріоритетом
  .

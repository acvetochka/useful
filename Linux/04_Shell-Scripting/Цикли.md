# Цикли

У мові Bash цикли використовуються для багаторазового виконання блоків коду. Основними типами є цикли `for` та `while`.

## Цикл for
Цей цикл призначений для перебору послідовностей значень. У кожній ітерації змінна циклу отримує наступне значення зі списку.

**Базова структура:**
- **Джерела значень**: Списком можуть бути прості слова, цілі фрази в лапках або результат виконання команди (наприклад, `$(cat file)`).
- **Обхід файлів**: Один із найпоширеніших способів використання `for` — перебір файлів у директорії за допомогою шаблонів (наприклад, `for file in /home/user/*`).
- **Цикл у стилі C**: Bash підтримує синтаксис, схожий на мову C, що зручно для числових ітерацій: `for (( i=1; i <= 10; i++ ))`.
- **Особливість**: Змінна, яка використовувалася в циклі, зберігає своє останнє значення навіть після виходу з нього.


## Цикл while
Цикл while виконує команди доти, доки задана умова залишається істинною.
- **Принцип роботи**: Умова перевіряється на вході в кожну ітерацію. Якщо вона повертає істину, виконується тіло циклу.
- **Приклад використання**: Найчастіше застосовується з лічильниками або для читання даних, поки вони не закінчаться.
- **Обробка файлів**: За допомогою конвеєра та команди read, цикл while може обробляти файл рядок за рядком: cat file | while read line.

## Керування циклами
Для зміни стандартної поведінки циклів існують дві спеціальні команди:
- `break`: дозволяє негайно перервати виконання циклу (і for, і while), не чекаючи виконання умови завершення.
- `continue`: завершує лише поточну ітерацію достроково і переходить до наступної, не виходячи з циклу повністю.

## Вкладені цикли та роль IFS
Ви можете запускати одні цикли всередині інших (вкладені цикли). Це часто використовується для обробки структурованих даних, наприклад, коли зовнішній цикл проходить по рядках файлу, а внутрішній — по окремих полях кожного рядка.
Важливу роль тут відіграє змінна IFS (Internal Field Separator). Вона визначає символи-розділювачі (за замовчуванням: пробіл, табуляція, новий рядок). Наприклад, для коректної обробки файлу /etc/passwd, де дані розділені двокрапками, IFS налаштовують як `IFS=:`

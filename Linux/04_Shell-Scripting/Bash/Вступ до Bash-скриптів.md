#  Вступ до Bash-скриптів
## Що таке Bash

`Bash` (Bourne Again Shell) — це командна оболонка та мова скриптів для Unix/Linux.

Він дозволяє:
- виконувати команди
- автоматизувати задачі
- писати сценарії (scripts)

👉 Bash — це і інтерпретатор, і мова програмування.

## Для чого потрібні Bash-скрипти

Основні застосування:

- автоматизація (backup, деплой)
- обробка файлів
- запуск послідовності команд
- адміністрування системи
- DevOps / CI/CD

📌 Приклад: замість 10 команд вручну → один скрипт

## Створення першого скрипта
1. Створюємо файл
   ```bash
   nano script.sh
   ```
2. Додаємо код
   ```bash
   #!/bin/bash
   echo "Hello, world!"
   ```

## Shebang (#!)

Перший рядок:
```bash
#!/bin/bash
```
Це означає:
> використовувати Bash для виконання скрипта

📌 Альтернатива (більш універсальна):
```bash
#!/usr/bin/env bash
```
✔ краще для переносимості (portable)

## Запуск скрипта
Варіант 1 — через bash:
```bash
bash script.sh
```
Варіант 2 — як виконуваний файл:
```bash
chmod +x script.sh
./script.sh
```

## Права доступу (chmod)

Щоб зробити файл виконуваним:
```bash
chmod +x script.sh
```
Права:

- `r` — read
- `w` — write
- `x` — execute

Перевірити:
```bash
ls -l script.sh
```

## Як Bash виконує скрипт
1. Читає файл зверху вниз
2. Інтерпретує кожну команду
3. Виконує послідовно

📌 Це не компіляція, а інтерпретація

## Коментарі
```bash
# Це коментар
echo "Hello"  # теж коментар
```

## Мінімальний шаблон скрипта
```bash
#!/usr/bin/env bash

# Exit on error
set -e

# Основний код
echo "Start script"
```

## Типові помилки початківців

❌ Забули shebang  
❌ Не дали права (chmod +x)  
❌ Запускають ./script.sh без ./  
❌ Пробіли навколо = у змінних  
❌ Використання Windows line endings (CRLF)  

## Перевірка Bash-версії
```bash
bash --version
```

## Важливі поради

- Завжди використовуй:
  ```bash
  #!/usr/bin/env bash
  ```
- Пиши зрозумілі коментарі
- Тестуй скрипти маленькими частинами
- Використовуй `set -e` для безпеки

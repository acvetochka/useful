# Антипатерни Bash (частина до Розділу 2)

## ❌ 1. Неекрановані змінні
```bash
rm $file
```
🔥 Проблема:
- пробіли → розбивається на аргументи
- `*` може розгорнутись (globbing)
- потенційно небезпечно

✔ Правильно:
```bash
rm "$file"
```

## ❌ 2. Використання `$*` замість `"$@"`
```bash
for arg in $*; do
  echo "$arg"
done
```

🔥 Проблема:
- аргументи з пробілами ламаються

✔ Правильно:
```bash
for arg in "$@"; do
  echo "$arg"
done
```

## ❌ 3. Старий стиль command substitution
```bash
result=`pwd`
```
🔥 Проблема:
- погано читається
- вкладені команди — біль

✔ Правильно:
```bash
result=$(pwd)
```

## ❌ 4. Пробіли в оголошенні змінних
```bash
var = 5
```
🔥 Проблема:
- Bash думає, що var — команда

✔ Правильно:
```bash
var=5
```

## ❌ 5. Відсутність лапок у test / if
```bash
if [ $name = Alice ]; then
```

🔥 Проблема:
- якщо `name=""` → синтаксична помилка

✔ Правильно:
```bash
if [ "$name" = "Alice" ]; then
```
або 
```bash
if [[ $name = Alice ]]; then
```


## ❌ 6. Використання [ ] замість [[ ]] (коли не треба POSIX)
```bash
if [ "$a" == "$b" ]; then
```

🔥 Проблема:
- [ ] менш безпечний
- менше можливостей

✔ Краще:
```bash
if [[ $a == $b ]]; then
```

## ❌ 7. Ігнорування exit code
```bash
cp file.txt /backup/
echo "Done"
```

🔥 Проблема:
- навіть якщо cp впав → скрипт продовжиться

✔ Краще:
```bash
set -e
```
або:
```bash
cp file.txt /backup/ || exit 1
```

## ❌ 8. Магічні числа замість $?
```bash
if [ $? -eq 0 ]; then
```

🔥 Проблема:
- читається гірше
- легко зробити помилку

✔ Краще:
```bash
if command; then
  echo "OK"
fi
```

## ❌ 9. Не використовувати local у функціях
```bash
my_func() {
  var="test"
}
```
🔥 Проблема:
- змінна "витікає" назовні

✔ Правильно:
```bash
my_func() {
  local var="test"
}
```

## ❌ 10. Захардкоджені шляхи
```bash
#!/bin/bash
```
або:
```bash
LOG="/home/user/project/log.txt"
```

🔥 Проблема:
- не переносимо

✔ Краще:
```bash
#!/usr/bin/env bash
```

## ❌ 11. Відсутність set -u
```bash
echo "$UNDEFINED_VAR"
```

🔥 Проблема:
- тихі помилки

✔ Краще:
```bash
set -u
```

## ❌ 12. Глобінг замість find
```bash
for f in *.txt; do
```

🔥 Проблема:
- якщо файлів нема → цикл зламається

✔ Краще:
```bash
shopt -s nullglob
```
або:
```bash
find . -name "*.txt"
```

## Міні-висновок

Якщо запам’ятати 5 речей:

- `"${var}"` завжди
- `"$@"` замість $*
- `$(...)` замість ``...``
- `[[ ]]` замість `[ ]`
- `set -euo pipefail`

👉 ти вже вище 80% Bash-скриптів у продакшені 😄

# Змінні та типи даних у Bash

## 1. Оголошення змінних

У Bash не потрібно вказувати тип змінної.
```bash
name="Alice"
age=25
```
📌 Важливо:
> НЕ можна ставити пробіли навколо =

❌ Неправильно:
```bash
name = "Alice"
```
✔ Правильно:
```bash
name="Alice"
```
## 2. Використання змінних

Щоб отримати значення — використовуємо `$`:
```bash
echo $name
```
🔸 Рекомендований синтаксис
```bash
echo "${name}"
```
📌 Чому краще:

- безпечніше
- працює з рядками поруч
- echo "Hello ${name}123"

❌ Без {}:
```bash
echo "Hello $name123"   # Bash шукає змінну name123
```

## 3. Типи даних (умовно)

У Bash все — рядки, але можна працювати як з:
- рядками
- числами
- масивами (окремо буде розділ)

## 4. Арифметика
```bash
a=5
b=3

result=$((a + b))
echo $result
```

✔ Підтримується:
- `+`
- `-`
- `*`
- `/`
- `%`

## 5. Командна підстановка (Command substitution)

Записуємо результат команди у змінну:
```bash
current_dir=$(pwd)
echo $current_dir
```
📌 Старий стиль (не рекомендується):
```bash
current_dir=`pwd`
```

## 6. Read (ввід користувача)
```bash
read name
echo "Hello $name"
```

🔸 З підказкою
```bash
read -p "Enter your name: " name
```

🔸 Без відображення (пароль)
```bash
read -s password
```

## 7. Спеціальні змінні

Це дуже важливо 👇

🔸 Аргументи скрипта
```bash
echo $0   # ім’я скрипта
echo $1   # перший аргумент
echo $2   # другий аргумент
```

🔸 Всі аргументи
```bash
echo "$@"
echo "$*"
```
📌 Різниця:
- `"$@"` — кожен аргумент окремо (правильно)
- `"$*"` — всі в один рядок
  
🔸 Кількість аргументів
```bash
echo $#
```

🔸 Код завершення
```bash
echo $?
```

✔ 0 → успіх  
❌ ≠0 → помилка

🔸 PID процесу
```bash
echo $$
```

## 8. Environment variables (змінні оточення)

Приклади:
```bash
echo $HOME
echo $PATH
echo $USER
```

🔸 Створення змінної оточення
```bash
export MY_VAR="hello"
```

📌 Тепер доступна у дочірніх процесах


## 9. Readonly змінні
```bash
readonly PI=3.14
```

❌ змінити вже не можна:
```bash
PI=3.1415   # помилка
```

## 10. Видалення змінної
```bash
unset name
```

## 11. Локальні змінні (у функціях)
```bash
my_func() {
  local var="hello"
}
```

📌 local — тільки всередині функції


## 12. Default values (дуже важливо 🔥)
```bash
echo "${name:-Guest}"
```
➡️ Якщо name не задана → "Guest"

🔸 Присвоєння за замовчуванням
```bash
name=${name:-Guest}
```

## 13. Перевірка змінної
```bash
echo "${name:?Variable not set}"
```
➡️ Якщо не задана → скрипт впаде


## 14. Кавички (ДУЖЕ ВАЖЛИВО ⚠️)
🔸 Подвійні `" "` — дозволяють змінні
```bash
echo "Hello $name"
```
🔸 Одинарні `' '` — ні
```bash
echo 'Hello $name'
```

➡️ Виведе буквально $name


## 15. Типові помилки

❌ Забули $:
```bash
echo name
```
❌ Пробіли:
```bash
var = 5
```
❌ Неекрановані змінні:
```bash
rm $file   # небезпечно!
```
✔ Правильно:
```bash
rm "$file"
```

## 16. Best practices

✔ Завжди використовуй:
```bash
"${var}"
```
✔ Перевіряй змінні:
```bash
set -u
```
✔ Імена:
```bash
USER_NAME="Alice"
```

## 17. Міні-приклад
```bash
#!/usr/bin/env bash

read -p "Enter your name: " name

name=${name:-Guest}

echo "Hello, ${name}!"
```

## Висновок
- У Bash все — рядки
- `${var}`  — кращий / безпечніший варіант, ніж `$var`, `"${var}"` — золотий стандарт
- Кавички — критично важливі
- `$@` майже завжди краще за `$*`

#


Створення Bash-скриптів — це процес об'єднання команд, які зазвичай вводяться вручну в терміналі, у текстові файли для автоматизації певних завдань.

## Основи створення Bash-скриптів
Процес розробки починається зі створення та наповнення файлу командами.

**Основні правила синтаксису включають:**
- **Запис команд**: Команди можна писати як у різних рядках, так і в одному, розділяючи їх крапкою з комою (;), проте написання в окремих рядках робить файл легшим для читання.
- **Коментарі**: Символ # використовується для створення коментарів, які оболонка ігнорує під час виконання.
- **Змінні**: У Bash всі змінні зазвичай є рядковими, але можуть оброблятися як числа. Для присвоєння значення використовується оператор =, а для звернення до змінної — символ $ (наприклад, $var_name).
- **Логічні конструкції**: Скрипти можуть містити розгалуження (if-then, case) та цикли (for, while) для керування потоком виконання.

## Роль шебангу
`Шебанг` — це послідовність символів #!/bin/bash, яка розміщується виключно у першому рядку файлу сценарію.

Його головні функції:
- **Визначення інтерпретатора**: Він вказує системі точний шлях до виконуваного файлу оболонки (в даному випадку Bash), яка повинна обробити цей скрипт.
- **Спеціальне призначення**: Використання символу `#` у шебангу відрізняється від його ролі в інших частинах скрипта, де він позначає звичайний коментар. Якщо не вказати шебанг, система може спробувати виконати скрипт за допомогою оболонки за замовчуванням, що може призвести до помилок, якщо скрипт використовує специфічні функції Bash.

Для розширеного керування поведінкою скрипта (наприклад, зупинки при помилках) часто використовується команда set відразу після шебангу

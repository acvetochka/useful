# Створення файлів з правами

у Linux не всі команди створення файлів дозволяють одразу задати права, і на результат все одно впливає umask. Розглянемо всі основні способи.

## 1. Створення каталогу з правами — mkdir -m

Команда mkdir має спеціальний параметр:
```bash
mkdir -m MODE directory
```
Приклад
```bash
mkdir -m 700 private_folder
```
Отримаємо:
> drwx------ private_folder

тобто
> 700

Інший приклад
```bash
mkdir -m 755 project
```
результат:
> rwxr-xr-x

> [!IMPORTANT] 
> `mkdir -m` ігнорує `umask` і встановлює права саме ті, які вказані.

## 2. Створення файлу з правами — install

Найправильніший спосіб створити файл одразу з правами:
```bash
install -m MODE source destination
```
Приклад
```bash
install -m 600 config.txt /tmp/config.txt
```
Файл одразу матиме:
> 600
***

**Створення порожнього файлу**
```bash
install -m 600 /dev/null secret.txt
```

## 3. Через touch + chmod

Команда touch не має параметра для прав, тому роблять:
```bash
touch file.txt
chmod 600 file.txt
```
Це найпоширеніший варіант.

## 4. Через umask

Іноді права задають опосередковано через umask.

Наприклад:
```bash
umask 077
touch file.txt
```
Файл буде:
> 600

## 5. Через shell redirection

Створення файлу:
```bash
echo "test" > file.txt
```
Права визначаються:
```
default permissions - umask
```

## 6. Через cp

Можна копіювати файл одразу з потрібними правами.
```bash
cp --preserve=mode source file
```

## 7. Через install (найчастіше у скриптах)

Наприклад, для скриптів:
```bash
install -m 755 script.sh /usr/local/bin/script
```

## 8. Через mktemp

Створює тимчасові файли з безпечними правами.
```bash
mktemp
```
Права:
> 600

## 9. Приклад створення приватного каталогу
```bash
mkdir -m 700 ~/.ssh
```

## 10. Типові практики
| задача                     | команда                         |
| -------------------------- | ------------------------------- |
| створити приватний каталог | mkdir -m 700 dir                |
| створити приватний файл    | install -m 600 /dev/null file   |
| створити скрипт            | install -m 755 script.sh script |
| створити файл через shell  | touch + chmod                   |

## 11. Підсумок
| команда     | можна задати права |
| ----------- | ------------------ |
| mkdir       | ✅ (-m)             |
| install     | ✅ (-m)             |
| touch       | ❌                  |
| echo > file | ❌                  |
| cp          | частково           |

> [!NOTE]
> 💡 Цікавий факт:  
> у Linux kernel при створенні файлу використовує системний виклик open(), де задається режим створення, але umask завжди застосовується до нього. Тому навіть якщо програма задає 777, kernel може зробити 755.

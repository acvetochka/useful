## 1. Базова модель прав у Linux

Linux використовує 3 біти прав:

право	число	значення
r	4	read
w	2	write
x	1	execute

| Право | число | Значення            |
| ----- | ----- | ------------------- |
| r     | 4     | читати (read)       |
| w     | 2     | змінювати (write)   |
| x     | 1     | запускати (execute) |

Комбінація:
```
4 + 2 + 1 = 7
```
Отже:
> 7 = rwx

Тому максимальні права:
```
777
```
тобто
> rwxrwxrwx


## 2. Чому файли не створюються з правами 777

Тому що виконуваний файл — це небезпечно.

Якщо кожен новий файл був би:
```
777
```
то:
- будь-який файл був би виконуваний
- можна випадково виконати небезпечний код
- це створює проблеми безпеки

Тому Linux використовує правило:
| тип       | максимальні права |
| --------- | ----------------- |
| file      | 666               |
| directory | 777               |

## 3. Чому файли отримують саме 666

Файл не повинен автоматично бути `executable`.

Тому система створює файл з:
> rw-rw-rw-

тобто
```
666
```
Пояснення:
- r	- читати
- w -	змінювати
- x -	запускати

Але файл:
- не завжди програма
- може бути текст
- конфігурація
- документ

Тому `execute` не додається автоматично.

## 4. Чому каталоги мають 777

`Каталог` — це контейнер файлів.

Щоб працювати з каталогом потрібно:

| право | значення           |
| ----- | ------------------ |
| r     | бачити список      |
| w     | створювати файли   |
| x     | заходити у каталог |

Тому каталоги отримують:
> rwxrwxrwx

тобто
```
777
```

## 5. Що таке umask

`umask` = user file creation mask

Це маска прав, яка забирає частину прав при створенні файлів.

Тобто:
```
final permissions =
default permissions − umask
```

6. Алгоритм роботи umask

Кроки:

1️⃣ система визначає базові права  
  ```bash
  file → 666
  directory → 777
  ```
2️⃣ береться umask

3️⃣ права віднімаються

## 7. Приклад: umask 022

Перевірити:
```bash
umask
```
результат:
> 022

- Для файлів
  ```
  666
  -022
  ----
  644
  ```
  результат:  
    > rw-r--r--

- Для каталогів
  ```bash
  777
  -022
  ----
  755
  ```

  результат:
  >  rwxr-xr-x


## 8. Що означає 022

Розшифруємо:
```
0 2 2
```
| позиція | значення |
| ------- | -------- |
| owner   | 0        |
| group   | 2        |
| others  | 2        |

2 означає:
- `write` забирається

Тобто:
| користувач | права |
| ---------- | ----- |
| owner      | rw    |
| group      | r     |
| others     | r     |

## 9. Інший приклад: umask `077`
```bash
077
```
- Файл
  ```
  666
  -077
  ----
  600
  ```
  > rw-------

- Каталог
  ```
  777
  -077
  ----
  700
  ```
  > rwx------

Це дуже безпечна конфігурація.

Файли доступні тільки власнику.

## 10. Чому використовується саме маска

umask — це зручний спосіб контролювати безпеку.

Без неї кожна програма повинна була б сама вирішувати:
- які права ставити
- для кого

А так:
> kernel застосовує umask автоматично

## 11. Де задається umask

umask можна задати:
- для shell
```shell
umask 022
```
- у bash конфігурації
  ```bash
  ~/.bashrc
  ```

  або
    ```bash
    /etc/profile
    ```
    системно
    ```bash
    /etc/login.defs
    ```

## 12.  Символьний формат umask

Можна задати так:
```bash
umask u=rwx,g=rx,o=rx
```

## 13. Побачити права без маски

Команда:
```bash
umask -S
```
показує:
```
u=rwx,g=rx,o=rx
```

## 14. Важлива деталь (яку мало хто пояснює)

umask не віднімається арифметично!

Вона працює як `bit mask`.

Але для простоти її часто пояснюють як віднімання.

Реально:
> permission = default & ~umask

## 15. Приклад реального процесу

Створюємо файл:
```bash
touch test.txt
```
kernel робить:
```
base = 666
umask = 022
```
результат:
> 644

Перевірка:
```bash
ls -l test.txt
```
> -rw-r--r--


## 16. Типові значення umask

| umask | файл | каталог | де використовують  |
| ----- | ---- | ------- | ------------------ |
| 022   | 644  | 755     | стандарт Linux     |
| 027   | 640  | 750     | сервери            |
| 077   | 600  | 700     | приватні системи   |
| 002   | 664  | 775     | shared development |

## 17. Типові сценарії
- сервер
  ```bash
  umask 027
  ```
  > група може читати.

- приватна система
  ```bash
  umask 077
  ```
  > файли приватні.

- Дуже важливий dev-сценарій

  У командах розробників часто використовують:
  ```
  umask 002
  ```
  Тоді:

  > file → 664  
  > dir → 775

  Це означає:
  - група може редагувати файли
  - зручно для shared project directories


## 18. Приклад реального сценарію

Уявімо:

ти працюєш з:
- приватними ключами
- токенами
- конфігами

Тоді зручно зробити:
```bash
umask 077
```
щоб всі нові файли були автоматично приватні.

## 19. Чи можна поставити umask 000?

Можна:
```bash
umask 000
```
Тоді:

| тип     | права |
| ------- | ----- |
| файл    | 666   |
| каталог | 777   |

Але це дуже небезпечно.

Файли будуть:
```
rw-rw-rw-
```
будь-який користувач зможе їх змінювати.

**Коли це використовують**

umask 000 іноді використовують:
- у shared directory
- у docker контейнерах
- у CI/CD

Але у звичайній системі це погана практика.

## 20. Для чого реально використовується umask

Основна ідея:
> не міняти права кожного файлу вручну.

Наприклад, якщо створюється багато файлів:
```bash
touch file1
touch file2
touch file3
touch file4
```
Замість:
```bash
chmod 600 file1
chmod 600 file2
chmod 600 file3
```
ти просто робиш:
```bash
umask 077
```
і всі нові файли одразу правильні.

## 21. Дуже важливий момент

umask впливає тільки на нові файли.

Вона не змінює існуючі.

Наприклад:
```
file.txt → 644
```
після:
```bash
umask 077
```
файл не стане 600.

## 22. umask діє тільки для процесу

Якщо ти виконаєш:
```bash
umask 077
```
вона буде діяти:
- у цьому shell
- для процесів, які він запускає

Після нового login:

umask може знову бути:

> 022

## 23. Де задається umask системно

umask може бути визначена у:
```
/etc/profile
/etc/bash.bashrc
/etc/login.defs
~/.bashrc
```

## Підсумок

umask — це глобальна маска прав для нових файлів.

Вона дозволяє:
- не робити chmod постійно
- задати політику доступу
- автоматично контролювати безпеку

Алгоритм:

- file default = 666
- dir default = 777
- final permissions =
default − umask

Приклад:
```bash
umask 022
file → 644
dir → 755
```

**Найкорисніше правило практики**
| задача            | umask |
| ----------------- | ----- |
| звичайна робота   | 022   |
| командна розробка | 002   |
| приватні файли    | 077   |

## 1. Базова модель прав у Linux

Linux використовує 3 біти прав:

право	число	значення
r	4	read
w	2	write
x	1	execute

| Право | число | Значення            |
| ----- | ----- | ------------------- |
| r     | 4     | читати (read)       |
| w     | 2     | змінювати (write)   |
| x     | 1     | запускати (execute) |

Комбінація:
```
4 + 2 + 1 = 7
```
Отже:
> 7 = rwx

Тому максимальні права:
```
777
```
тобто
> rwxrwxrwx

## 2. Чому файли не створюються з правами 777

Тому що виконуваний файл — це небезпечно.

Якщо кожен новий файл був би:
```
777
```
то:
- будь-який файл був би виконуваний
- можна випадково виконати небезпечний код
- це створює проблеми безпеки

Тому Linux використовує правило:
| тип       | максимальні права |
| --------- | ----------------- |
| file      | 666               |
| directory | 777               |

## 3. Чому файли отримують саме 666

Файл не повинен автоматично бути `executable`.

Тому система створює файл з:
> rw-rw-rw-

тобто
```
666
```
Пояснення:
- r	- читати
- w -	змінювати
- x -	запускати

Але файл:
- не завжди програма
- може бути текст
- конфігурація
- документ

Тому `execute` не додається автоматично.

## 4. Чому каталоги мають 777

`Каталог` — це контейнер файлів.

Щоб працювати з каталогом потрібно:

| право | значення           |
| ----- | ------------------ |
| r     | бачити список      |
| w     | створювати файли   |
| x     | заходити у каталог |

Тому каталоги отримують:
> rwxrwxrwx

тобто
```
777
```

## 5. Що таке umask

`umask` = user file creation mask

Це маска прав, яка забирає частину прав при створенні файлів.

Тобто:
```
final permissions =
default permissions − umask
```

## 6. Алгоритм роботи umask

Кроки:

1️⃣ система визначає базові права  
  ```bash

  directory → 777
  ```
2️⃣ береться umask

## 8. Що означає 022

Розшифруємо:
```
0 2 2
```
| позиція | значення |
| ------- | -------- |
| owner   | 0        |
| group   | 2        |
| others  | 2        |

2 означає:
- `write` забирається

Тобто:
| користувач | права |
| ---------- | ----- |
| owner      | rw    |
| group      | r     |
| others     | r     |

## 16. Типові значення umask

| umask | файл | каталог | права файлу (опис)                                                                    | права каталогу (опис)                                                                                                    | де використовують  |
| ----- | ---- | ------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ------------------ |
| 022   | 644  | 755     | **власник:** читати + змінювати; <br>**група:** читати; <br> **інші:** читати         | **власник:** читати, створювати, заходити; <br> **група:** переглядати і заходити; <br> **інші:** переглядати і заходити | стандарт Linux     |
| 027   | 640  | 750     | **власник:** читати + змінювати; <br> **група:** читати; <br> **інші:** немає доступу | **власник:** повний доступ; <br> **група:** переглядати і заходити; <br> **інші:** немає доступу                         | сервери            |
| 077   | 600  | 700     | тільки **власник** може читати і змінювати                                            | тільки **власник** має повний доступ                                                                                     | приватні системи   |

## 17. Типові сценарії
- сервер
  ```bash
  umask 027
  ```
  > група може читати.

- приватна система
  ```bash
  umask 077

  ```

  > file → 664  
  > dir → 775

  Це означає:
  - група може редагувати файли
  - зручно для shared project directories

## 19. Чи можна поставити umask 000?

Можна:
```bash
umask 000
```
Тоді:

| тип     | права |
| ------- | ----- |
| файл    | 666   |
| каталог | 777   |

Але це дуже небезпечно.

Файли будуть:
```
rw-rw-rw-
```
будь-який користувач зможе їх змінювати.

**Коли це використовують**

umask 000 іноді використовують:
- у shared directory
- у docker контейнерах
- у CI/CD

Але у звичайній системі це погана практика.

## 20. Для чого реально використовується umask

Основна ідея:
> не міняти права кожного файлу вручну.

Наприклад, якщо створюється багато файлів:
```bash
touch file1
touch file2
touch file3
touch file4
```
Замість:
```bash
chmod 600 file1
chmod 600 file2
chmod 600 file3
```
ти просто робиш:
```bash
umask 077
```
і всі нові файли одразу правильні.

## 24. Як подивитись права у цифровому (octal) вигляді
в Linux є кілька способів подивитися права у цифровому (octal) вигляді, тобто так, як вони використовуються в chmod (наприклад 755, 644, 600). Розглянемо найзручніші.

### 1. Найзручніший спосіб — команда stat
```bash
stat file.txt
```
Приклад виводу:
```
File: file.txt
Size: 0
Access: (0644/-rw-r--r--)  Uid: (1000/alona)   Gid: (1000/alona)
```
Тут видно:
```
0644
```
- `0` — спеціальні біти (suid/sgid/sticky)
- `644` — стандартні права
***

**Показати тільки число**
```bash
stat -c "%a" file.txt
```
Результат:
```
644
```
***
**Для каталогу**
```bash
stat -c "%a" folder
```
Приклад:
```
755
```
***

### 2. Показати разом з назвою файлу
```bash
stat -c "%a %n" *
```
Приклад:
```
644 file1.txt
755 folder
600 secret.txt
```
***
### 3. Використання find

Можна знайти файли з певними правами.

Наприклад:
```bash
find . -perm 644
```
Або показати права:
```bash
find . -printf "%m %p\n"
```
Результат:

> 644 ./file.txt  
> 755 ./folder

***

### 4. Через ls (не прямо)

Команда:
```bash
ls -l
```
показує символьний формат:
> -rw-r--r--

Його треба перевести у числа:

| права | число |
| ----- | ----- |
| rwx   | 7     |
| rw-   | 6     |
| r-x   | 5     |
| r--   | 4     |

Наприклад:
> -rw-r--r--

це
> 644
***

### 5. Показати права для багатьох файлів

Зручно:
```bash
stat -c "%a %n" *
```
Приклад:
```
644 file.txt
600 password.txt
755 script.sh
```

### 6. Показати права для каталогу і всіх файлів
```bash
stat -c "%a %n" directory/*
```

### 7. Побачити спеціальні біти (suid/sgid/sticky)
```bash
stat file
```
Наприклад:
```
Access: (4755/-rwsr-xr-x)
```
Тут:
> 4755

- `4` = SUID

### 8. Приклад з усіма спеціальними бітами
| число | значення |
| ----- | -------- |
| 4xxx  | SUID     |
| 2xxx  | SGID     |
| 1xxx  | Sticky   |

Наприклад:
> 1777

каталог `/tmp`.

### 9. Найкорисніша команда для адміністратора

Зазвичай використовують:
```bash
stat -c "%a %n" *
```
або
```bash
stat -c "%a %n" file
```

### 10. Маленький лайфхак

Можна зробити alias:
```bash
alias perms='stat -c "%a %n"'
```
Тоді:
```bash
perms file.txt
```

## 25. Комбінації прав

Є дуже зручний спосіб миттєво переводити rwx → числа, не рахуючи кожного разу 4+2+1. Достатньо запам’ятати 8 комбінацій прав, бо кожна позиція (owner / group / others) може мати тільки ці варіанти.

### 1. Основна таблиця rwx → числа
| права | число | пояснення              |
| ----- | ----- | ---------------------- |
| ---   | 0     | нічого                 |
| --x   | 1     | execute                |
| -w-   | 2     | write                  |
| -wx   | 3     | write + execute        |
| r--   | 4     | read                   |
| r-x   | 5     | read + execute         |
| rw-   | 6     | read + write           |
| rwx   | 7     | read + write + execute |

Це всі можливі комбінації.

### 2. Як переводити права у число

Наприклад:
```
-rw-r--r--
```
Розбиваємо на три блоки:
```
rw-  r--  r--
```
Тепер дивимось у таблицю:
- `rw-` = 6
- `r--` = 4
- `r--` = 4

Отже:
> 644

### 3. Ще приклади
- Приклад 1
  ```
  rwxr-xr-x
  ```
  - `rwx` = 7
  - `r-x` = 5
  - `r-x` = 5
  > 755

- Приклад 2
  ```
  rw-rw-r--
  ```

  - `rw-` = 6
  - `rw-` = 6
  - `r--` = 4
  > 664


- Приклад 3
  ```
  rwx------
  ```
  - `rwx` = 7
  - `---` = 0
  - `---` = 0
  > 700

### 4. Як переводити число назад у rwx

Наприклад:
```
754
```
Розбиваємо:
- 7 → rwx
- 5 → r-x
- 4 → r--

Результат:
> rwxr-xr--

### 5. Дуже корисний трюк для швидкого запам’ятовування

Запам’ятай тільки три числа:
- 4 = read
- 2 = write
- 1 = execute

І тоді:

| число | формула | права |
| ----- | ------- | ----- |
| 7     | 4+2+1   | rwx   |
| 6     | 4+2     | rw-   |
| 5     | 4+1     | r-x   |
| 4     | 4       | r--   |
| 3     | 2+1     | -wx   |
| 2     | 2       | -w-   |
| 1     | 1       | --x   |
| 0     | 0       | ---   |

### 6. Як адміністратори читають права дуже швидко

Досвідчені адміни дивляться на букви r w x:
```
rwxr-xr-x
```
і одразу бачать:
```
7 5 5
```
бо:
- rwx → 7
- r-x → 5
- r-x → 5

Це приходить з практикою.

### 7. Найпоширеніші права у Linux
| число | права     | де використовується |
| ----- | --------- | ------------------- |
| 644   | rw-r--r-- | звичайні файли      |
| 755   | rwxr-xr-x | програми / каталоги |
| 600   | rw------- | секретні файли      |
| 700   | rwx------ | приватні каталоги   |
| 664   | rw-rw-r-- | shared файли        |
| 775   | rwxrwxr-x | shared каталоги     |

### 8. Ще один лайфхак

Коли бачиш:
```bash
chmod 755
```
можна одразу читати:
- owner → rwx
- group → r-x
- others → r-x

### 9. Тренування (дуже корисно)

Спробуй перевести:
```
rwxrwx---
```
Правильна відповідь:
> 770

І ще:
```
rw-r-----
```
Відповідь:
> 640

## Підсумок

umask — це глобальна маска прав для нових файлів.

Вона дозволяє:
- не робити chmod постійно
- задати політику доступу
- автоматично контролювати безпеку

Алгоритм:

- file default = 666
- dir default = 777
- final permissions =
default − umask

Приклад:
```bash
umask 022
file → 644
dir → 755
```

**Найкорисніше правило практики**
| задача            | umask | що блокується                                      |
| ----------------- | ----- | -------------------------------------------------- |
| звичайна робота   | 022   | забороняє запис для group і others                 |
| командна розробка | 002   | дозволяє групі редагувати файли                    |
| приватні файли    | 077   | забороняє доступ для всіх, крім власника           |
| сервери           | 027   | забороняє запис для group і повністю доступ others |

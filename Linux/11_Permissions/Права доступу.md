## 1. Основи прав доступу

Linux використовує модель:

**Три категорії**
| Категорія | Опис     |
| --------- | -------- |
| owner     | власник  |
| group     | група    |
| others    | всі інші |

**Три права:**
| Право | Значення            |
| ----- | ------------------- |
| r     | читати (read)       |
| w     | змінювати (write)   |
| x     | запускати (execute) |


**Приклад:**
```bash
-rwxr-xr--
```

**Розшифровка:**
| Частина | Значення |
| ------- | -------- |
| rwx     | owner    |
| r-x     | group    |
| r--     | others   |

**Права каталогів**

Права для директорій мають інший зміст.

| Право | Значення            |
| ----- | ------------------- |
| r     | перегляд списку     |
| w     | створення/видалення |
| x     | доступ до каталогу  |

## 2. Числові права
| Право | Число |
| ----- | ----- |
| r     | 4     |
| w     | 2     |
| x     | 1     |

| Комбінація | Значення |
| ---------- | -------- |
| 7          | rwx      |
| 6          | rw-      |
| 5          | r-x      |
| 4          | r--      |

Приклад:
```bash
chmod 755 script.sh
```

## 3. chmod

Зміна прав.

Символьний режим
```bash
chmod u+x file
chmod g-w file
chmod o+r file
```

Числовий режим
```bash
chmod 644 file
chmod 755 script
```

Ключі
| Ключ | Значення   |
| ---- | ---------- |
| -R   | рекурсивно |
| -v   | verbose    |

Приклад:
```bash
chmod -R 755 project
```

## 4. chown

Зміна власника.
```bash
chown user file
```
Зміна групи:
```bash
chown user:group file
```
Ключі
| Ключ | Значення   |
| ---- | ---------- |
| -R   | рекурсивно |

Приклад:
```bash
chown -R user:developers project
```

## 5. chgrp

Зміна групи.
```bash
chgrp developers file
```


## 6. Спеціальні дозволи

- `SUID` - Set User ID.

  Файл виконується від імені власника.

  Приклад:
  ```bash
  -rwsr-xr-x
  ```
  Встановити:
  ```bash
  chmod u+s file
  ```

- `SGID` - Set Group ID.
  ```bash
  chmod g+s file
  ```
  На каталогах:
  - нові файли отримують групу каталогу.

- `Sticky bit`

  Використовується для каталогів.

  Sticky означає:
  видаляти файл може тільки
  - власник
  - root

  Приклад:
  ```bash
  /tmp
  ```
  ```bash
  drwxrwxrwt
  ```
  Встановити:
  ```bash
  chmod +t dir
  ```

## 7. Права за замовчуванням (umask)

`umask` визначає які права будуть відняті.

Перевірити:
```bash
umask
```
Приклад:
```
022
```
Розрахунок:

  Файли:
  ```
  666 - umask
  ```
  Каталоги:
  ```
  777 - umask
  ```

  ```bash
  666 - 022 = 644
  777 - 022 = 755
  ```

## 8. Атрибути файлів

Перегляд:
```bash
lsattr
```
Встановити:
```bash
chattr +i file
```
Основні атрибути:

| Атрибут | Значення    |
| ------- | ----------- |
| i       | immutable   |
| a       | append only |
| d       | no dump     |
| A       | no atime    |

`immutable`

Файл не можна:
- змінити
- видалити
- перейменувати

Навіть root не може без зняття атрибуту.

## 9. ACL

ACL дозволяє задавати права для конкретних користувачів.

Перегляд:
```bash
getfacl file
```
Додати:
```bash
setfacl -m u:user1:rwx file
```
Видалити:
```bash
setfacl -x u:user1 file
```

## 10. sudo

`sudo` дозволяє виконувати команди як root.

Файл конфігурації:
```bash
/etc/sudoers
```
Редагувати:
```bash
visudo
```
Приклад:
```
user ALL=(ALL) ALL
```

## 11. Безпека користувачів

Основні принципи:
- мінімальні права (least privilege)
- розділення користувачів
- використання sudo
- контроль груп

## 12. Типові сценарії
Дати доступ групі
```bash
chown :developers project
# або
chmod 770 project
```
Спільний каталог
```bash
chmod 2775 project
```
SGID забезпечить одну групу.

Захист файлу
```bash
chattr +i config
```
